The System

5 Agents, 3 Levels

Every agent is a Claude Code instance running in its own tmux session. The hierarchy decides who owns which scope of decisions.

Missionary
Writes high-level methodology. Evaluates results. Decides when to stop.
Strategizer
Owns one strategy. Decides what to explore next. Runs 10-16 explorations.
Explorer
Researches one question deeply
Librarian
Searches the shared knowledge base
Curator
Processes findings into the library
OUTER LOOP
Missionary writes a strategy document (methodology), launches a strategizer to execute it, reads the final report, decides whether to write another strategy or declare the mission complete.
INNER LOOP
Strategizer queries the librarian for context, designs an exploration goal, launches an explorer in tmux, reads its report, launches the curator to file findings. Repeat until the strategy is exhausted.

The System

How Information Flows

Four directions of knowledge movement keep the system coherent without any agent needing to hold everything in context.

Downward: Goals
Missionary writes strategy docs. Strategizer writes GOAL.md for each exploration. Each level narrows scope for the level below.
Upward: Reports
Explorer writes REPORT.md. Strategizer writes strategy summary. Each level compresses findings for the level above.
Lateral: Shared Library
Curator files every finding into a 95+ entry factual library. Librarian retrieves relevant entries on demand. All agents share this knowledge base.
Meta: Process Lessons
A meta-inbox captures operational lessons (what went wrong, what to do differently). These survive context resets and inform future behavior.

Surviving Context Resets

THE STOP HOOK
When the strategizer runs out of context, a shell hook detects the exit, matches the transcript path to the strategy directory, and feeds a meta-prompt back in. The strategizer picks up where it left off with no manual intervention. This kept the inner loop alive across unlimited context resets.
WHY THIS MATTERS
Without this, the strategizer would die after ~7 explorations. With it, Strategy 2 ran 16 explorations autonomously. The strategizer was only at 12% context usage after 7 explorations thanks to the loop resetting state while preserving knowledge on disk.

How the agents collaborated

Strategizer surveyed the landscape
First explorations mapped all known constraints in the problem space. Each explorer wrote a report; the curator filed findings into the library. The strategizer queried the librarian before each new exploration to avoid redundant work.
Explorations built on each other
Later explorers received librarian-retrieved context from earlier ones. Exploration 5 used findings from Explorations 1-4 to narrow the search. The library acted as shared memory across agents that never directly communicated.
Converged on a single answer
By Exploration 9, the strategizer concluded that all constraints pointed to one theory. It wrote a final report to the missionary with the result and a validation summary.
Key system behavior
The strategizer autonomously decided the exploration order, adapted based on findings, and identified when it had enough evidence to stop. No human guided the research direction after the initial strategy was written.

What we learned about the system

Mid-run pivoting works
The missionary evaluated Strategy 1's output, identified a gap (uniqueness wasn't proven), and designed Strategy 2 specifically to close it. The system self-corrected without human intervention.
Library compounding
Strategy 2 explorers had access to all of Strategy 1's findings via the librarian. They didn't re-derive what was already known. The shared library meant knowledge persisted across strategies.
Systematic elimination
The strategizer tested 5 categories of alternative approaches. Each exploration tried to find a viable alternative. Every path either failed on its own terms or converged back to the same answer.
Stop-hook resilience
16 explorations exceeded a single context window. The stop-hook loop kept the strategizer alive across resets, with on-disk state (reports, library, goals) as the persistent memory.
Why this matters
The system didn't just find an answer — it challenged its own answer. The missionary acted as a skeptical reviewer, refusing to accept Strategy 1's result at face value. This is the behavior you want from an autonomous research system.

Takeaways

What Worked & What Broke

3 test runs before the full mission. Each fixed real problems.

Worked
Stop hook loop kept the strategizer alive across context resets. Transcript path matching was simple and reliable after the initial bug was fixed.
Curator maintained a 95+ entry library with hierarchical indexing. Deduplication and conflict checking worked autonomously. The library became the system's long-term memory.
Librarian queries improved decision quality. Context from previous explorations prevented redundant work and enabled the strategizer to build on prior findings.
Context efficiency was excellent. Strategizer at 12% context usage after 7 explorations. Could easily handle 16+.
Fixed Mid-Run
Foreground explorers blocked everything. Run 1 used foreground sub-agents that couldn't be monitored or timed out. Switched to tmux-based explorers with progress polling.
Manual curation didn't scale. Curator was initially manual. Added fire-and-forget tmux launch after each exploration.
Explorer writing was bursty. Wrote 8 lines early, then nothing for 10 min, then 521 at end. Improved to 33 lines early by run 3 via prompt changes.
Known Limitations
One manual step remains: the settings.json permission prompt requires a human "2" approval per run. Could be pre-registered.
No auto-retry on connection failures. Strategizer got stuck on ConnectionRefused for 33 min. Recovered when nudged manually.
