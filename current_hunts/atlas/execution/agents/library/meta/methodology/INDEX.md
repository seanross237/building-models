# Methodology — Meta Library

Lessons about what strategy structures work: exploration sequencing, phased approaches, when to pivot, and proven patterns for specific task types.

## Entries

- **divergent-survey-pattern.md** — The four-part pattern (survey + deep dive + synthesis + novelty assessment) for Phase 1 landscape mapping.
- **devils-advocate-after-construction.md** — Attack immediately after building; give the attacker the full document, not a summary.
- **predict-attack-synthesize-sequence.md** — The proven strategy cycle: extract predictions, then attack, then synthesize into final document.
- **adversarial-explorations-need-null-hypothesis.md** — Every devil's advocate must test whether the null hypothesis (no framework needed) explains the same observations.
- **repair-pattern.md** — One repair per exploration, structured as: what changed, what survived, what's fixed, what's broken, before/after scorecard.
- **benchmark-comparison-early.md** — Compare new frameworks to established benchmarks early to reveal the gap before over-investing.
- **verification-catches-library-errors.md** — Verification explorations don't just confirm; they catch wrong premises and misleading claims in the knowledge base.
