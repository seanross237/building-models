## Lesson

Devil's advocate explorations should always include a null hypothesis test: "Can the observations/predictions be explained without this framework?" Without it, you can attack individual predictions without asking the most important question -- whether the framework adds anything beyond its components.

## Evidence

- **strategy-003 exploration s3-008** — Found that the "compatible-but-separate" interpretation is a simpler alternative the framework hadn't addressed. This was the most damaging finding.
- **strategy-003 exploration s3-009** — Developing the null hypothesis revealed that most "predictions" were consistency checks (explainable by H0), not discriminators. Only one prediction was genuinely discriminating.
- **strategy-004 exploration s4-006** — Null hypothesis test showed that H0 explains everything the framework claims. This is the sharpest possible critique.

## When to apply

Every devil's advocate exploration. Also useful when constructing frameworks -- develop the null hypothesis ALONGSIDE the framework. If you can't distinguish your framework from the null, it's not a scientific theory yet.
