# Route Map

## Route Families Considered

1. `pure canonical packet model`
   - Define packet variables and leakage bookkeeping first, then search for an
     exact-NS no-near-closed-circuit theorem.

2. `critical-element packet leakage rigidity`
   - Assume a minimal singular object exists in a critical space, extract a
     canonical packet graph from that object, and prove exact NS forces packet
     leakage or angular spreading incompatible with Tao-like gate isolation.

3. `direct microlocal regularity route`
   - Push a microlocal phase-space regularity program directly as a new proof
     lane, without using Tao-like packet circuitry as the comparison object.

4. `reopened partial-regularity bootstrap with new language`
   - Try to revive the epsilon-regularity family using modern decomposition
     language while keeping the core covering architecture.

## Proposed Route In One Line

Use critical-element/profile-decomposition machinery to canonically select a
hypothetical minimal blowup object, define packet variables on that object via a
microlocal or phase-space extraction rule, and then prove a positive lower bound
on exact-NS spectator leakage or angular spreading that rules out Tao-like
near-closed delayed-threshold packet circuits.
