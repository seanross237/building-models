# Meta Note - Step 008 Exploration 001

Framework-freezing for compactness-rigidity branches must separate genuine
solution frameworks from host-space language before any host-space audit.

Reusable lesson:

- suitable-weak / Leray-Hopf plus LEI can be a real inherited framework;
- `L^3` and `\dot H^{1/2}` can be serious critical-space languages without
  thereby becoming admitted frameworks;
- and any proposed bridge architecture between critical-space language and the
  LEI floor must be rejected unless the repository supplies one explicit
  compatibility package.

That rule blocks a common failure mode: rescuing a dead branch by silently
switching framework, endpoint, or temporal packaging under the banner of
compactness prestige.
