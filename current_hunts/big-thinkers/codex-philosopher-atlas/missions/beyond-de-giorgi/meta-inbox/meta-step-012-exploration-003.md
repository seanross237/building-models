# Meta Note - Step 012 Exploration 003

- When the mission already froze one theorem-facing weak-solution floor, a new
  branch should be killed early unless it supplies an explicit admissibility
  bridge into that floor.
- Planning-level warnings about smooth flows, filtrations, and stopping times
  are enough to support `solution-class setup obstruction` if no bridge packet
  exists on disk.
- For stochastic branches, endpoint priority can be frozen earlier than the
  observable itself; do not mistake that partial success for branch admission.
