# Chain 05 - Geometry Reboot Under Open Package Change

## Central Premise

The prior geometry branch did not fail universally; it failed on one explicitly audited neutral package. If geometry still carries the missing NS-specific signal, the only honest way to revisit it is as a new branch that changes the package openly and charges every adaptation cost explicitly.

## Verifiable Result Target

Either:

- one adapted-package geometric or hybrid observable that survives cost accounting and controls the full stretching mechanism in a fixed representation; or
- a bounded negative result showing that geometry needs hidden package adaptation, stronger derivative input, or tuned normalization to look operational.

## Why This Chain Is Meaningfully Different

This is the only chain whose main object is the localization package itself. It does not treat Lagrangian or tube-adapted coordinates as harmless setup; it audits whether the package change is the source of the apparent gain.

## Ordered Steps

### Step 1 - Justify the package change and fix the new claim boundary

- Depends on: none.
- Action: state exactly why the neutral Eulerian package from `step-004` may have been too weak and what new package is being proposed, such as Lagrangian, tube-following, or another sharply defined adapted localization.
- Action: choose one comparator package, normally the killed neutral Eulerian package, so any gain can be measured honestly.
- Expected output: a package memo naming the new localization, its claimed advantage, and the precise boundary of what this branch is allowed to claim.
- Kill condition: if the proposed package change cannot name a specific mathematical burden it reduces, or if its only benefit is vaguer geometry language, stop immediately.

### Step 2 - Fix one observable family and charge all adaptation costs up front

- Depends on: Step 1.
- Action: choose one primary observable or hybrid observable for the new package.
- Action: write down the full bookkeeping terms introduced by adaptation: moving-cutoff terms, Jacobian or relabeling effects, matching rules, and any derivative burden hidden in the package.
- Action: fix one full-stretching representation before any positive claims are allowed.
- Expected output: an observable-and-cost table listing the candidate observable, the full stretching target, and every package-induced error term.
- Kill condition: if the package only becomes favorable after hiding matching costs or importing stronger derivative control than the branch admits, close the chain negatively.

### Step 3 - Re-run the dynamic persistence screen on the new package

- Depends on: Step 2.
- Action: test whether the primary observable is genuinely more dynamically plausible on the adapted package than it was on the neutral Eulerian package.
- Action: compare the persistence gain directly against the newly introduced package costs.
- Expected output: a persistence memo rating the candidate as dynamically plausible, informative but weak, or package-dependent.
- Kill condition: if the dynamic improvement is fully explained by the adaptation overhead or by a tuned threshold/window choice, stop and record that failure.

### Step 4 - Test control of the full stretching mechanism with costs included

- Depends on: Step 3.
- Action: evaluate whether the observable controls the full stretching mechanism in the fixed representation after all package-induced error terms are charged.
- Action: keep the comparator package in view to detect fake gains caused only by relocalization.
- Expected output: a lemma sheet or obstruction memo stating exactly what part of stretching is controlled and what the package change really bought.
- Kill condition: if the apparent gain disappears once adaptation costs are paid, or if only a local/self-induced piece improves, close the route negatively.

### Step 5 - Close with a package-level verdict

- Depends on: Step 4, or directly on Step 3 if the chain dies there.
- Action: state whether the geometry reboot found a genuinely new route or only proved that geometry needs illicit package adaptation to look live.
- Expected output: either a concrete adapted-package geometry route with a named next theorem target, or a sharp memo explaining why the geometry signal does not survive honest package accounting.
- Kill condition: if the final claim says more than the audited package, observable family, and stretching representation support, narrow it until it is fully earned.
