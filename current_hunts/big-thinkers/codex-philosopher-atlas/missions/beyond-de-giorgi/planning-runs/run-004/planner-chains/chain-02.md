# Chain 02 - Critical Element With Averaging-Sensitive Rigidity

## Central Premise

If all local estimate cleanups are exhausted, the remaining path may be to assume blowup, extract a minimal counterexample in a critical phase space, and rule it out using a rigidity mechanism that depends on exact NS structure unavailable to Tao's averaged model.

## Verifiable Result Target

Either:

- a concrete critical-element program with one named phase space, one named rigidity lever, and one explicit first missing lemma; or
- an obstruction map showing exactly why compactness extraction or NS-specific rigidity still fails.

## Why This Chain Is Meaningfully Different

This is the only chain that starts from a hypothetical singular solution and tries to win globally through concentration-compactness and rigidity rather than through a new local estimate.

## Ordered Steps

### Step 1 - Choose the phase space before discussing rigidity

- Depends on: none.
- Action: rank realistic critical or near-critical spaces such as `L^3`, `\dot H^{1/2}`, and `BMO^{-1}` by extraction feasibility, known theorem support, and compatibility with a suitable weak or mild-solution framework.
- Expected output: a phase-space memo naming the primary extraction setting and the runner-up.
- Kill condition: if no candidate space supports a credible minimal-element package, stop and record a compactness-floor obstruction.

### Step 2 - Fix one candidate rigidity lever up front

- Depends on: Step 1.
- Action: choose one exact NS structure that could exclude the critical element, such as a tensor-sign law, Lagrangian/topological invariant, or a pressure-Hessian feature.
- Action: reject generic levers that Tao's averaged model plausibly preserves, such as bare backward uniqueness plus harmonic analysis.
- Expected output: a rigidity-screen memo naming one promoted lever and one backup.
- Kill condition: if no Tao-sensitive rigidity lever can be named before the extraction package is built, terminate the branch negatively.

### Step 3 - Build the minimal-counterexample dossier

- Depends on: Step 2.
- Action: formulate the hypothetical minimal blowup object, record the symmetry normalization, and state exactly what compactness or almost-periodicity statements are required.
- Expected output: a dossier of assumptions, normalization choices, known ingredients, and missing ingredients.
- Kill condition: if the minimal object cannot be formulated cleanly in the chosen phase space, stop and write that failure precisely.

### Step 4 - Stress-test the rigidity route on the dossier

- Depends on: Step 3.
- Action: run the promoted rigidity lever against the hypothetical critical element and identify the exact contradiction pathway.
- Action: separate what is already in the literature from the first missing NS-specific lemma.
- Expected output: a contradiction blueprint or a memo saying the lever collapses back to Tao-preserved structure.
- Kill condition: if the first missing lemma is no sharper than the original regularity problem, downgrade to an obstruction result.

### Step 5 - Close with the narrowest earned claim

- Depends on: Step 4.
- Action: state either the best live critical-element program or the exact reason this route is still blocked.
- Expected output: either a compactness-plus-rigidity roadmap or a calibrated negative memo with precise failure points.
- Kill condition: if the final claim blurs multiple phase spaces or multiple rigidity levers that were not actually audited, narrow it.
