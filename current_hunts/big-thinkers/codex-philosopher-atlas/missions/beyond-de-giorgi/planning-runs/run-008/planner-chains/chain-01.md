# Chain 01 - Exact-NS Property Atlas And Tao Differential Diagnosis

## Central Premise

The mission may be closer to a publishable structural map than to a single winning proof attack. The first live branch is therefore to compare exact NS against Tao's averaged model directly and promote only those exact properties that are both genuinely destroyed by averaging and operational inside one named proof architecture.

## Verifiable Result Target

Either:

- one ranked property atlas listing 3-5 exact NS structures, with a Tao verdict, an architecture attachment, and a first missing lemma for each survivor; or
- one obstruction memo showing that every currently visible exact NS property is either Tao-robust, prior-art-collapsed, or too weak to reach a theorem endpoint.

## Why This Chain Is Meaningfully Different

This chain is comparison-first rather than attack-first. It does not commit to one proof architecture at the start, and its natural positive output is a landscape map rather than a conditional theorem statement.

## Ordered Steps

### Step 1 - Freeze the comparison granularity and output schema

- Depends on: none.
- Action: define exactly what counts as "preserved by Tao averaging" and "destroyed by Tao averaging" for this mission.
- Action: fix one comparison schema for every candidate:
  exact property,
  averaged-model verdict,
  nearest prior-art family,
  possible proof architecture,
  and first theorem-facing burden.
- Expected output: a comparison protocol memo with one pass example and one fail example.
- Kill condition: if the Tao screen cannot be made concrete at one comparison level, stop and write a calibration obstruction.

### Step 2 - Build the candidate property menu from mission-live sources only

- Depends on: Step 1.
- Action: shortlist at most five candidate property families, for example:
  commutator-ready scalarization structure,
  trajectory-level deformation constraints,
  stochastic circulation transport,
  vorticity-direction persistence beyond prior-art criteria,
  or one exact pressure/stress identity that is not already closed.
- Action: reject any candidate that is already killed by Step 010, Step 011, or the earlier fixed-geometry audit.
- Expected output: a candidate table with one-sentence novelty and prior-art notes.
- Kill condition: if the candidate list is mostly renamed dead branches, terminate with a recycled-portfolio obstruction.

### Step 3 - Attach each survivor to one proof architecture or one bounded negative

- Depends on: Step 2.
- Action: for each candidate, name one architecture where it would have to act:
  local criterion,
  transport-continuation criterion,
  concentration exclusion,
  stochastic contradiction,
  or critical-element rigidity.
- Action: state one exact burden the property would need to reduce.
- Expected output: an architecture-attachment sheet.
- Kill condition: if a candidate cannot be attached to one theorem-facing burden, demote it immediately.

### Step 4 - Rank by operationality, not by slogan-level novelty

- Depends on: Step 3.
- Action: rank survivors by four axes:
  Tao sensitivity,
  endpoint closeness,
  prior-art distance,
  and narrowness of the first missing lemma.
- Action: keep only candidates that clear all four axes at a nontrivial level.
- Expected output: a final ranked atlas with one primary, one secondary, and one bounded-negative branch.
- Kill condition: if no candidate survives the four-axis screen, close with a mission-wide obstruction map.

### Step 5 - Convert the ranking into execution-ready admissions rules

- Depends on: Step 4.
- Action: for each surviving branch, write the exact admission rule and the first kill condition that the later run must enforce.
- Action: record which closed branches are no longer allowed to re-enter under new names.
- Expected output: an execution memo that feeds later selector work directly.
- Kill condition: if the ranking cannot be translated into concrete admission rules, downgrade the branch to a descriptive map only.
