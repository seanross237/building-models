# Chain 01 - Tao Differential Diagnosis And Attackability Ranking

## Central Premise

Before spending most of the run on any one mechanism, freeze the actual comparison target: the exact 3D Navier-Stokes equations versus Tao's averaged model. The branch is live only if it identifies structures genuinely destroyed by averaging and then ranks them by theorem-floor admissibility, endpoint contact, and novelty ceiling. If no such survivor exists, the correct output is a bounded obstruction atlas rather than another speculative branch.

## Why This Chain Is Meaningfully Different

This is the only comparison-first chain. It does not try to prove regularity directly. Its job is to prevent budget from being wasted on structures that Tao averaging preserves anyway, or on structures that are exact but have no credible route to a theorem-facing endpoint.

## Ordered Steps

### Step 1 - Freeze the comparison contract and scoring currency

- Depends on: none.
- Action: write the exact comparison target, namely which features of Tao's averaged model count as preserved and which candidate NS structures would count as destroyed.
- Action: freeze one scoring currency for the whole chain: `Tao destruction + theorem-floor admissibility + endpoint contact`.
- Action: set the acceptable outputs in advance: ranked shortlist, bounded obstruction atlas, or one reserve-only candidate.
- Expected output: a comparison-contract memo with a fixed scoring rubric.
- Kill condition: if the chain cannot say exactly what counts as "destroyed by averaging" beyond general rhetoric, stop with a comparator-definition obstruction.

### Step 2 - Build the candidate property register from established mission context

- Depends on: Step 1.
- Action: list only serious candidate structures already on the board: strain-vorticity alignment, local Beltrami deficit, tube geometry, deterministic deformation/stretching, far-field pressure harmonicity, exact helical interaction geometry, and any other exact-NS feature that is not just generic harmonic analysis.
- Action: mark which candidates were already downgraded in prior runs and why.
- Action: reject candidates that merely rename energy identity, Calderon-Zygmund, Littlewood-Paley, or Sobolev machinery.
- Expected output: a candidate-property register with prior-run status tags.
- Kill condition: if the register is dominated by Tao-compatible harmonic-analysis structure, stop with a no-live-candidate obstruction.

### Step 3 - Run the Tao differential test property by property

- Depends on: Step 2.
- Action: for each candidate, state the exact object or identity that the true NS system has and ask whether Tao averaging preserves, weakens, or erases it.
- Action: require one concrete witness of destruction: a lost sign pattern, lost triad geometry, lost transport identity, lost harmonic continuation law, or lost strain-geometry coupling.
- Action: demote any candidate whose claimed distinctiveness survives only as vague "more structure" language.
- Expected output: a Tao-differential table with `destroyed`, `unclear`, and `preserved` classifications.
- Kill condition: if no candidate survives this gate with a concrete destroyed feature, stop with a Tao-filter obstruction.

### Step 4 - Attach one proof architecture and one theorem-floor admission test to each survivor

- Depends on: Step 3.
- Action: for each surviving property, assign one concrete attack architecture only: criterion-scale geometric route, deterministic stretching route, local harmonic-tail route, packet/helical route, or compactness reserve.
- Action: state the admissible floor for that route: direct suitable-weak / LEI by default, otherwise one named proxy class with explicit bridge debt.
- Action: freeze one endpoint per survivor, not a menu of hopes.
- Expected output: a property-to-architecture map with theorem-floor admission notes.
- Kill condition: if a survivor has no theorem-facing endpoint or needs a bridge already as hard as the full theorem, demote it immediately.

### Step 5 - Rank by attackability and collapse the dead families into sharp negatives

- Depends on: Step 4.
- Action: rank the survivors by presentable-result odds, novelty ceiling, and debt size in the fixed scoring currency.
- Action: compress the dead families into one-page obstruction notes so later chains do not reopen them by accident.
- Action: nominate at most two primary chains and one reserve chain.
- Expected output: a ranked shortlist plus an explicit no-reopen list.
- Kill condition: if the ranking cannot distinguish between survivors because all endpoints are equally vague, stop with an endpoint-contact obstruction.

### Step 6 - Produce the canonical atlas artifact

- Depends on: Step 5.
- Action: write the final atlas naming the 2-3 live exact-NS structural leads, the most plausible architecture for each, the first missing lemma, and the dead branches that should stay closed.
- Action: record whether exact helical structure stays demoted, returns as reserve, or re-enters the live set.
- Expected output: a reusable mission-wide property atlas and attackability ranking.
- Kill condition: if the final atlas still reads like brainstorm notes rather than a filter with enforcement power, downgrade it to obstruction and stop.
