# Chain 02 - Deterministic Material Stretching Route With Admissibility, Stability, And Interface Gates

## Central Premise

The deterministic transport-side branch remains live only if one material observable can be shown to do more than tell a formal stretching story. The observable must survive three distinct gates:
- admissibility on a theorem-credible floor or across an explicitly paid bridge,
- stability under approximation, localization, and nonlocal reinsertion,
- theorem-facing leverage either on the inherited LEI / CKN interface or on one clearly specified alternative interface.

If the branch fails, it should fail with a high-resolution obstruction rather than a generic "transport does not help" verdict.

## Why This Chain Is Meaningfully Different

This is still the deterministic transport-first branch. It studies deformation, cumulative positive strain, or a similarly specific material observable rather than packet geometry, scalarized criteria, or contradiction-by-compactness. The refinement is methodological: the branch now distinguishes mechanism failure from interface mismatch and demands sharper learning from every stop condition.

## Ordered Steps

### Step 1 - Freeze one observable, one native framework, one theorem interface, and one novelty bet

- Depends on: none.
- Action: choose exactly one material observable.
- Action: name the weakest framework on which that observable is supposed to exist. If a stronger class is needed, state the bridge burden explicitly rather than hiding it inside "admits."
- Action: choose one primary theorem interface. This may be the inherited LEI / CKN concentration route or one alternative interface that is better matched to semi-global or flow-map-native structure.
- Action: state the novelty class being tested: new admissible observable, new sign structure, new localization device, or new theorem interface.
- Action: build an admissibility matrix with separate entries for measurable definition, renormalized or weak definition, approximation stability, and LEI-compatible compactness.
- Expected output: a scope-lock memo with one observable, one native framework, one primary interface, one novelty bet, and a four-part admissibility matrix.
- Kill condition: if there is no defensible definition even after an explicitly acceptable bridge, stop and record which admissibility gate failed.

### Step 2 - Derive the strongest admissible identity and name the actual obstruction schema

- Depends on: Step 1.
- Action: write the strongest evolution statement honestly available for the observable, separating four levels: exact, mollified, renormalized, and purely formal.
- Action: identify where the viscous term enters and whether it is sign-definite, absorbable, commutator-sized, or dominant.
- Action: identify one explicit nonlocal obstruction schema rather than listing "pressure / strain" as generic debt.
- Action: state whether the nonlocal term is external bookkeeping or structurally part of the candidate mechanism.
- Expected output: an evolution sheet with four identity levels, a viscous-defect ledger, and one named nonlocal obstruction schema.
- Kill condition: if the observable only has a smooth-flow-map identity, or if the admissible defect / commutator ledger already dominates the principal term, stop and classify the exact failure.

### Step 3 - Rank the burdens before testing leverage

- Depends on: Step 2.
- Action: build a ranked burden ledger for the chosen theorem interface, naming the dominant bottleneck rather than an arbitrary "bad term."
- Action: test whether the observable controls the dominant bottleneck directly, controls a genuine feeder quantity, or only controls a secondary burden.
- Action: if the gain is only intermediate, state the exact compound-mechanism claim required to make it relevant.
- Expected output: a ranked burden ledger plus a leverage memo marking the observable as direct, intermediate, secondary, or null.
- Kill condition: if the observable reaches only secondary burdens and there is no credible compound path to the dominant bottleneck, stop and record a non-coercivity obstruction.

### Step 4 - Stress-test insertion on the primary interface without confusing architecture failure with mechanism failure

- Depends on: Step 3.
- Action: attempt insertion of the promoted gain into the chosen primary interface and charge cutoff, localization, temporal-window, viscosity, and nonlocal reinsertion costs explicitly.
- Action: determine whether admissible localization destroys the gain itself or only prevents entry into the chosen proof architecture.
- Action: if the primary interface fails for architectural reasons while a stable semi-global or trajectory-level gain survives, record that as interface mismatch rather than immediate mechanism death.
- Expected output: an insertion memo with one of three outcomes: survives on interface, destroyed by admissible localization, or survives but mismatches interface.
- Kill condition: if admissible localization erases the gain and no theorem-facing form survives, stop and record a localization / nonlocal destruction obstruction.

### Step 5 - Produce one falsifiable artifact and run one concrete robustness check

- Depends on: Step 4.
- Action: produce one hard artifact with an explicit evidence tier.
- Action: Tier A is a prototype estimate with stated hypotheses and conclusion.
- Action: Tier B is a failed lemma whose failure mechanism is explicit and informative.
- Action: Tier C is a toy scenario, allowed only if it isolates one exact debt term that appears in the real ledger.
- Action: after the artifact exists, run one concrete robustness check matched to it: averaging sensitivity, delocalization under pressure, coarse-graining collapse, or profile-instability under the chosen interface.
- Expected output: an evidence packet with one artifact tier and one concrete robustness verdict.
- Kill condition: if there is no falsifiable intermediate claim, or only a suggestive toy picture with no transfer to the real ledger, stop and record a documentary obstruction.

### Step 6 - State the narrowest earned claim and the next branch implication

- Depends on: Step 5.
- Action: if positive, write one conditional claim naming the observable, framework, interface, insertion point, and first missing lemma.
- Action: if negative, classify failure precisely as undefinable observable, definable but approximation-unstable, admissible identity too weak, viscous defect dominance, nonlocal obstruction dominance, non-coercive leverage, localization destruction, interface mismatch, topology or exponent mismatch, or renamed full-regularity mirage.
- Action: state what the obstruction says about neighboring branches: rule-out, redirect, or hybridization opportunity.
- Expected output: a final-comparison-ready conditional route or high-resolution obstruction note.
- Kill condition: if the only remaining "missing lemma" is effectively full regularity restated in transport language, downgrade immediately to named obstruction.
