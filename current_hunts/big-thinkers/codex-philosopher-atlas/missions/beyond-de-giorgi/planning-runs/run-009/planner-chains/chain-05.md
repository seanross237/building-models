# Chain 05 - Critical Element Program With Pre-Admitted Exact-NS Firewall

## Central Premise

Compactness-rigidity is no longer an admissible primary branch. It is live only if an earlier chain first produces one exact-NS firewall quantity that is theorem-facing, definable on the compactness class, and plausibly narrower than the full regularity problem. Without that firewall, the right output is an immediate no-entry obstruction.

## Why This Chain Is Meaningfully Different

This is the only global contradiction architecture in the portfolio. It does not search for a new mechanism inside the PDE first; it consumes one already-admitted mechanism and asks whether it can survive extraction, almost periodicity, and contradiction.

## Ordered Steps

### Step 1 - Enforce the firewall admission gate before any host-space discussion

- Depends on: one upstream chain having produced a concrete firewall candidate.
- Action: write the exact firewall quantity, the theorem endpoint it supports, and why it is genuinely exact-NS rather than generic harmonic-analysis structure.
- Action: require that the quantity be definable on the intended compactness class and stable enough to survive compactness limits.
- Expected output: either a firewall-admission memo or an immediate no-entry verdict.
- Kill condition: if no concrete firewall packet exists, stop immediately and write a firewall-missing obstruction.

### Step 2 - Audit one host space and one framework against the admitted firewall

- Depends on: Step 1.
- Action: choose one host space only, such as `L^3` or `\dot H^{1/2}`, and one solution framework.
- Action: audit local theory, stability, extraction, normalization, pressure compatibility, and compatibility of the firewall quantity with compactness.
- Expected output: one host-space package audit.
- Kill condition: if the chosen host space still lacks a coherent extraction package or the firewall quantity is not definable there, stop and write a host-space obstruction.

### Step 3 - Define the critical element in theorem-like form

- Depends on: Step 2.
- Action: state the minimizing quantity, symmetry normalization, compactness statement, and exact class of the hypothetical minimal blowup object.
- Action: ensure the firewall quantity survives in the critical-element class with no hidden extra assumptions.
- Expected output: a critical-object dossier.
- Kill condition: if the critical element cannot be stated without importing new diffuse assumptions, stop and write a formulation obstruction.

### Step 4 - Run the contradiction pathway line by line

- Depends on: Step 3.
- Action: insert the firewall quantity into the almost-periodic or compactness-normalized setting.
- Action: identify exactly where contradiction should arise and what remaining lemma is needed.
- Action: reject any route whose contradiction requires restating the full regularity problem.
- Expected output: one contradiction-path memo plus a first-missing-lemma statement.
- Kill condition: if the first missing lemma is equivalent to full regularity, stop and write a no-complexity-drop obstruction.

### Step 5 - Use Tao sensitivity only as secondary evidence

- Depends on: Step 4.
- Action: ask whether the firewall mechanism would survive Tao-style averaging, but only after definability and contradiction power are established.
- Action: keep Tao sensitivity supportive rather than agenda-setting.
- Expected output: a short Tao-comparator note.
- Kill condition: if Tao sensitivity is the main reason the route looks interesting, downgrade to obstruction.

### Step 6 - State the narrowest earned claim

- Depends on: Step 5.
- Action: if positive, write one theorem-like conditional blueprint naming the host space, framework, critical-element class, firewall quantity, contradiction line, and first narrower missing lemma.
- Action: if negative, classify failure as missing firewall, host-space incoherence, critical-object formulation failure, firewall non-definability, or no genuine complexity drop.
- Expected output: a final-comparison-ready compactness route or obstruction memo.
- Kill condition: if the final claim cannot be written without broad Kenig-Merle rhetoric replacing actual package details, narrow it until the unpaid debt is explicit.
