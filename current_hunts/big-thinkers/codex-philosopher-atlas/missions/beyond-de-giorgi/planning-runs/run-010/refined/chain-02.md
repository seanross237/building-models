# Refined Chain 02 - Deterministic Stretching Route Only If One Observable Survives Theorem-Floor and Endpoint-Interface Gates

## Central Premise

The transport branch is live only if one deterministic stretching observable can be defined directly on the suitable-weak / LEI theorem floor, or on one proxy class whose descent back to that floor is quantitatively cheaper than the target theorem, and if that observable yields a gain inside a localized theorem-facing ledger rather than only in transport-native language.

## Why This Chain Is Still Meaningfully Different

This remains the only transport-first branch in the portfolio, but it is no longer a rescue operation for transport rhetoric. Its job is narrower: test whether any deterministic stretching observable changes the inherited burden ledger at a theorem-facing bottleneck after admissibility, prior-art, localization, and transfer costs are all charged early.

## Ordered Steps

### Step 1 - Freeze one observable, one endpoint ledger, and one admissibility standard

- Depends on: none.
- Action: choose one observable family only and state its exact object class: direct theorem-floor Eulerian quantity, coarse-grained Eulerian surrogate, or genuinely path-based quantity.
- Action: freeze one theorem-facing endpoint ledger before any gain claims, with local concentration exclusion as default and continuation-scale control allowed only if the observable already lives in that currency.
- Action: if a proxy class is used, write the exact bridge burden back to suitable-weak / LEI data and require that burden to be demonstrably cheaper than the target theorem.
- Action: name the nearest prior-art analogue and one claimed mathematical delta.
- Expected output: an admissibility-and-ledger memo naming one observable, one endpoint currency, one prior-art comparator, and any proxy bridge burden.
- Kill condition: if the observable is not directly definable on the theorem floor and the only honest bridge is theorem-scale, or if the claimed delta over prior art is only packaging, stop with a theorem-floor or prior-art obstruction.

### Step 2 - Derive the exact localized evolution and the same-currency defect ledger

- Depends on: Step 1.
- Action: derive the evolution identity in the form actually relevant to the frozen endpoint, including localization, pressure entry, cutoff terms, and temporal-window structure.
- Action: separate exact terms from terms requiring extra regularity and mark any place where smooth-flow or hidden trajectory machinery enters.
- Action: write all defects in the same ledger as the frozen endpoint burden rather than in transport-native units.
- Expected output: a localized evolution sheet plus a same-currency defect ledger.
- Kill condition: if the key production term is exact only off the theorem floor, or if commutator, pressure, cutoff, temporal-window, and nonlocal strain terms are not credibly lower order in the frozen ledger, stop with an exactness or non-separation obstruction.

### Step 3 - Run the anti-repackaging and novelty gate

- Depends on: Step 2.
- Action: classify the candidate as genuine mechanism change, ambiguous reformulation, or known strain/deformation repackaging.
- Action: reject candidates whose gain is only a renaming of known deformation, strain-alignment, or coarse-grained transport criteria without a changed endpoint ledger.
- Action: keep an ambiguous candidate alive only if it names one exact place in the frozen ledger where the burden could change.
- Expected output: a novelty screen with one surviving candidate at most.
- Kill condition: if the candidate has no explicit ledger-changing delta over prior art, stop with a repackaging obstruction.

### Step 4 - Freeze one theorem-facing bottleneck and one minimum gain margin inside localization

- Depends on: Step 3.
- Action: choose one exact bottleneck only inside the localized endpoint interface: a concentration feed line, positive stretching contribution after localization, or another single burden term already present in the frozen ledger.
- Action: specify one minimum meaningful gain margin in that same currency and at that same scale.
- Action: reject candidates that merely track the dangerous mechanism rather than control or exclude it.
- Expected output: a bottleneck-and-margin memo.
- Kill condition: if no theorem-facing bottleneck can be named, or if the observable only correlates with the bad mechanism and does not produce one-sided leverage on it after localization, stop with a non-operative-gain obstruction.

### Step 5 - Demand one hard artifact before any late comparison

- Depends on: Step 4.
- Action: produce one hard artifact immediately: either a prototype localized inequality in the live ledger, a reduced deterministic scenario with an explicit transfer map back to the live ledger, or a failed lemma isolating the exact obstruction.
- Action: allow descriptive toy models to count only if the transfer map and hidden regularity assumptions are explicit.
- Action: treat a failed lemma as valid evidence for obstruction but not for positive promotion.
- Expected output: a tiered artifact packet with hidden assumptions exposed.
- Kill condition: if the branch still has no theorem-facing artifact, or if the artifact lives only on a toy floor with no credible transfer map, stop with a documentary obstruction.

### Step 6 - Run full gain-versus-debt insertion on the inherited interface

- Depends on: Step 5.
- Action: insert the artifact into one mini proof skeleton containing local energy, pressure handling, cutoff debt, nonlocal strain freedom, and any proxy-to-floor transfer step.
- Action: budget gains and debts in the same frozen ledger and reject any uncaptured cross-currency step unless it is backed by an explicit transfer lemma.
- Action: distinguish exactly three outcomes: survives the inherited interface, loses net gain after full budgeting, or preserves a gain that is real but interface-mismatched.
- Expected output: an endpoint-budget memo showing net gain, net loss, or interface mismatch.
- Kill condition: if the gain is liquidated by full endpoint budgeting, or survives only in transport-native currency, stop with an endpoint-transfer obstruction.

### Step 7 - Use Tao sensitivity only as a secondary discriminator and close narrowly

- Depends on: Step 6.
- Action: ask whether the surviving feature would plausibly be erased by Tao-style averaging, but only after theorem-facing survival is already established.
- Action: if positive, write one conditional route naming the observable, theorem floor or proxy bridge, endpoint ledger, bottleneck term, artifact tier, and first genuinely narrower missing lemma.
- Action: if negative, classify failure as theorem-floor mismatch, proxy-bridge inflation, exactness failure, non-separation of defects, repackaging, non-operative gain, documentary failure, or endpoint-transfer failure.
- Action: downgrade immediately if the remaining "missing lemma" is just full regularity restated in transport language.
- Expected output: a final-comparison-ready route or a sharply typed obstruction memo.
- Kill condition: if the conclusion outruns the artifact or the full endpoint budget, narrow it until it does not.

## Probability Assessment

**Probability that this refined chain yields a presentable result: 0.74**

The most plausible presentable output is a bounded obstruction memo rather than a positive transport route. That is still mission-relevant if it shows, in a theorem-facing ledger, exactly where deterministic stretching observables fail: theorem-floor admissibility, proxy inflation, non-separation of localized defects, or endpoint-transfer collapse.
