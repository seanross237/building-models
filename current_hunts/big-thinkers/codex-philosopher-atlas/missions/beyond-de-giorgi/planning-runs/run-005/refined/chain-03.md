# Refined Chain 03 - Tensor Stretching Route With Early Overlap Audit And Quantitative Admission

## Central Premise

This branch is live only if one tensor package

- a stretching representation,
- a candidate observable,
- and an exact accounting of local and nonlocal pressure-Hessian debt

supports a quantitative statement about the full `S omega . omega` mechanism that is robust across natural reformulations. If no such package survives, the correct output is an obstruction showing why local tensor geometry is informative but non-coercive in the actual Navier-Stokes system.

## Verifiable Result Target

Either:

- one conditional tensor criterion, control lemma, or reduced proposition whose missing theorem is explicitly narrower than full regularity and still acts on the full stretching mechanism; or
- a calibrated obstruction memo locating the first fatal break as one of:
  - prior-art collapse,
  - representation dependence,
  - eigenvalue-degeneracy collapse,
  - failure of quantitative leverage on full stretching,
  - or pressure-Hessian debt remaining same-order and nonclosing.

## Why This Chain Is Meaningfully Different

This version keeps the tensor route but removes most of the rhetorical slack. It does not ask whether a tensor observable sounds interesting. It asks whether a tightly defined tensor package survives early overlap screening, survives a cross-representation check, and yields a quantitative control target before the chain invests in detailed estimates.

## Scope Lock

Before substantive execution begins, fix all of the following:

- one primary stretching representation and one mandatory alternate natural representation for robustness checking;
- the overlap-audit sources:
  prior mission findings, internal run history, and the nearest tensor-alignment / pressure-Hessian literature family;
- the admission standard for a promoted candidate:
  it must point to one explicit inequality, sign mechanism, or coercivity statement on the full `S omega . omega` term after pressure debt is written down;
- the policy that Tao-sensitivity is secondary evidence only, not the main gate.

If these items cannot be fixed concretely, stop and write a setup obstruction.

## Ordered Steps

### Step 1 - Run an early overlap and representation audit

- Depends on: none.
- Action: assemble the exact identities linking `S`, `omega`, and `nabla^2 p`, including the pressure split, but pair this immediately with an overlap audit against the nearest alignment, strain-eigenvalue, and pressure-Hessian criteria.
- Action: record which apparent mechanisms survive in both the primary and alternate representations and which ones look representation-specific.
- Action: reject any candidate direction that is already prior art in substance or that looks promising only in one coordinate packaging.
- Expected output: an overlap-and-ledger table with exact identity, nearest prior-art overlap, representation sensitivity, and open leverage target.
- Kill condition: if every candidate either collapses to known literature or loses its appeal under the alternate-representation check, stop and write an overlap/representation obstruction.

### Step 2 - Admit at most two tensor packages by a quantitative gate

- Depends on: Step 1.
- Action: shortlist at most two tensor packages, each containing one observable plus the exact representation and debt split it needs.
- Action: for each package, state one explicit control target on the full stretching term:
  sign control, coercive bound, or a decomposition in which the good part and the same-scaling debt are both written explicitly.
- Action: record scaling, definability, nearest overlap, and why the package is not just a renamed local fragment or a generic bounded geometric diagnostic.
- Expected output: a quantitative admission sheet with one explicit target inequality or mechanism for each surviving package.
- Kill condition: if no package can state a non-proxy quantitative target on the full `S omega . omega` term, stop and write a quantitative-admission obstruction.

### Step 3 - Stress-test definability, degeneracy, and Tao-sensitivity in the right order

- Depends on: Step 2.
- Action: check whether each admitted package is well-defined near repeated eigenvalues or can be rewritten invariantly without losing the mechanism it claimed in Step 2.
- Action: verify stability under the regularity level actually available in the intended argument, so the package is not meaningful only on smoother flows than the problem grants.
- Action: only after those tests, state what feature Tao's averaged model should destroy and why that feature is the same one used in the candidate control target.
- Expected output: a definability-and-robustness memo licensing at most one lead package and at most one backup.
- Kill condition: if invariant reformulation destroys the mechanism, if the package is undefined on the intended class, or if Tao-sensitivity can be stated only post hoc, stop and write a robustness obstruction.

### Step 4 - Attempt full stretching control immediately and name the first missing lemma

- Depends on: Step 3.
- Action: write the control attempt on the exact full `S omega . omega` term in the primary representation and verify that the same mechanism survives the alternate representation.
- Action: charge all pressure-Hessian debt explicitly, including the harmonic far-field contribution, and classify whether the candidate gives control, only correlation, or no usable reduction.
- Action: identify the first missing lemma and require it to be strictly narrower than arbitrary finite-energy regularity, with the chosen tensor package used essentially.
- Expected output: a control-or-failure table with the first missing lemma named precisely.
- Kill condition: if the debt remains same-order without complexity drop, if the result is only correlation rather than control, or if the first missing lemma is effectively the full regularity problem, stop and write a nonclosure obstruction.

### Step 5 - State the narrowest earned claim only

- Depends on: Step 4.
- Action: write the final result as either:
  - a conditional tensor proposition with exact assumptions, exact control target, exact known inputs, and the first missing theorem; or
  - an obstruction statement naming the first fatal break and why later tensor refinements cannot evade it.
- Action: benchmark the final claim against the nearest literature family before declaring novelty.
- Action: remove all unevaluated alternatives and keep only the single earned package, or the single obstruction type.
- Expected output: a final-comparison-ready theorem blueprint or obstruction memo.
- Kill condition: if the claim cannot be written in theorem-like or obstruction-like form without vague placeholders, the branch has not earned a presentable result.
