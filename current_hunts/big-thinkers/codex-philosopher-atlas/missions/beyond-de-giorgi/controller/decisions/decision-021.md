# Decision Memo - beyond-de-giorgi / decision-021

## Decision

`proceed`

## Mission-Control Verdict

This is a bootstrap decision after planning run `run-009`, not a review of a
completed step inside the current branch. The question is therefore whether the
mission should now enter the newly selected chain or reject it before any new
strategizer work begins.

The answer is yes: proceed.

Run `run-009` already performed the branch comparison that mission control
needed. Its final decider selected the reformulation-audit chain
`Quantified Reformulation Audit With Anti-Repackaging, Early Endpoint Exchange,
And Interface-Aware Budgeting`, and the repository's active `CHAIN.md` now
matches that winner exactly. The planning artifacts are aligned on the key
points:

- the prior stochastic-transport branch was invalidated at `step-012`;
- replanning compared materially different successor branches rather than
  silently repairing the failed one;
- the new winner was chosen because it gives the strongest theorem-facing floor
  while forcing early honesty about endpoint exchange, prior-art overlap, and
  anti-repackaging risk; and
- the next logical move is Chain Step 1 calibration, not another branch
  comparison.

That means the chain is active, not weakened. Mission control should now let
the strategizer test whether the reformulation audit can freeze a stable
endpoint window, a theorem-facing burden ledger, and an enforceable
anti-repackaging protocol before any candidate reformulation is promoted.

## Why `replan` Is Wrong

Replanning again would duplicate work that `run-009` has already finished. The
latest planning run did the required comparison, chose a canonical winner, and
updated the active chain accordingly. There is no intervening step result
showing that the new branch failed its setup gate, collapsed into prior-art
equivalence, or otherwise needs immediate replacement.

Conservative control here means entering the new chain at its first gate, not
reopening portfolio selection before that gate has even been tested.

## Why `terminate` Is Wrong

The mission-level question remains open. The repository has earned a new
theorem-facing branch and a stronger ordering of alternatives, not a terminal
positive or terminal negative answer about which exact Navier-Stokes structural
property could break the De Giorgi barrier.

This new branch may still fail quickly, and a bounded obstruction memo may
still be its likeliest success payload. But that has not happened yet.
Termination would therefore be premature.

## Required Next Controller Move

Proceed into `step-013` as the bootstrap execution for Chain Step 1 of the
current reformulation-audit branch.

That step should remain a strict calibration gate. It must freeze:

- one primary endpoint family and one backup endpoint family on the inherited
  LEI / CKN floor;
- one theorem-facing baseline burden and one admissible gain notion in a typed
  same-currency ledger;
- one explicit comparability rule naming when a transfer lemma is required;
- one anti-repackaging classification rule with the categories
  `equivalent redistribution`, `ambiguous redistribution`, and
  `potentially live mechanism change`;
- one solution class and localization regime; and
- one prior-art audit format that later candidate packets must satisfy.

If those items cannot be fixed concretely, the branch should stop at Chain Step
1 with an under-specified-ledger obstruction rather than drifting into
candidate reformulation work.
