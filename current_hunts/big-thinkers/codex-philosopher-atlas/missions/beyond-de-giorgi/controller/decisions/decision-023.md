# Decision Memo - beyond-de-giorgi / decision-023

## Decision

`replan`

## Mission-Control Verdict

This is a post-step review of `step-014`, which was supposed to execute Chain
Step 2 of the active reformulation-audit branch. The relevant question is not
whether the step found a positive candidate. It is whether the latest result
keeps the current chain alive.

It does not.

`step-014` reached the exact kill condition written into `CHAIN.md` for Step 2:
if every active reformulation is either recycled, identity-level, or lacks a
credible exchange path back to the frozen theorem-facing burden currency, the
branch must stop before the anti-repackaging gate. That is the outcome the step
reported. The two active candidates
`Lamb-vector / Helmholtz-projected form` and
`vorticity transport / Biot-Savart form`
both restore the same localized LEI cutoff-flux burden once pressure re-entry,
cutoff commutators, reconstruction, admissibility debt, and locality loss are
charged. The step also found that the needed transfer lemmas are absent on the
repository record and that the nearest prior-art packets already absorb the
identity-level content.

That is a branch invalidation, not a soft warning. Proceeding to Chain Step 3
would contradict the current chain's own admission rules.

## Why `proceed` Is Wrong

`proceed` would only be correct if `step-014` had preserved at least one live
candidate or one narrowly unresolved exchange path that still made the current
ordered chain the best next move. The step says the opposite. Its explicit
verdict is:
`exchange-failure obstruction, branch should stop before anti-repackaging gate`.

Because the chain was designed to kill decorative reformulations early, mission
control should respect that kill rather than advance to the formal
anti-repackaging screen out of sequence.

## Why `terminate` Is Wrong

The mission-level question is broader than exact reformulations of the
quadratic burden. `step-014` earned a bounded negative result about the current
reformulation-audit branch, but it did not establish that all remaining
Navier-Stokes-specific structural routes are exhausted or non-operational.

The repository still contains materially different non-reformulation directions
that earlier planning treated as serious reserves, including transport-side and
geometry-facing branches. The right next move is therefore a fresh branch
comparison informed by this newly earned obstruction, not mission completion.

## Required Next Controller Move

Replan the mission.

The replanning run should treat the following as established:

- the exact-reformulation audit has now produced its bounded negative payload;
- exact Lamb/projected and vorticity/Biot-Savart rewrites did not survive even
  the early endpoint-exchange screen on the inherited LEI / CKN ledger; and
- any successor branch should be materially different from this failed chain,
  not a cosmetic repair of the same reformulation family.

The new planner should compare only branches that use a different mechanism
class from exact reformulation auditing and should explicitly explain why the
new branch is not already ruled out by the Step-014 exchange-failure result.
