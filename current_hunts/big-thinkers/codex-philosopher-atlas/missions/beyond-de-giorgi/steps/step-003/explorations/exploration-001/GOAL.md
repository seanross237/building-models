# Exploration 001 Goal — Choose The Singular Scenario Class

## Objective

Choose one primary singular-scenario class, and at most one secondary
comparator, for Step 3 of the geometry route.

The choice must be made specifically for the narrowed primary survivor:

`direction coherence + tube persistence`

The exploration should compare the most plausible scenario classes still
supported by the local repository, including at least:

- filament or tube concentration
- sheet or pancake concentration

If another concrete scenario class appears in the local record, it may be
included, but only if it is supported by named repository sources.

The report must decide:

- which scenario is the fairest test bed for the primary hybrid after Step 2
- whether a comparator scenario should also be retained
- why rejected scenarios are inferior, overfitted, or too weakly tied to the
  survivor set
- and one explicit failure mode for each chosen scenario

## Success Criteria

- Produces a source-based decision memo, not decorative vortex imagery.
- Uses `[VERIFIED]`, `[INFERRED]`, and `[PROPOSED]` labels consistently.
- Names exact source files for every major scenario choice.
- Explains the scenario choice in terms of the live full-stretching target,
  not merely because the geometry sounds natural.
- Ends with:
  - one primary scenario class
  - optional one comparator scenario
  - one failure mode for each chosen scenario
  - and a short note on what this choice implies for the next localization step

## Failure Criteria

- Treats tube language alone as enough reason to choose a tube scenario.
- Chooses a scenario without saying what aspect of the hybrid it is meant to
  stress or expose.
- Leaves sheet/pancake concentration unassessed as a comparator.
- Smuggles in a localization protocol instead of limiting this exploration to
  scenario choice and scenario-level failure modes.
