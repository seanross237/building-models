stochastic branch, then state whether the setup already looks too smooth-flow
dependent to justify Step 2.

## Required Questions

Answer all of the following from local repository materials:

1. Which endpoint family is the correct first priority for this branch:
   local concentration exclusion,

   or another exact contradiction line?

   mission, and how does the stochastic branch sit relative to it?
3. What do the local sources say about strong filtrations, optional stopping,
   localization, and smooth stochastic-flow dependence as setup risks?

   inherited suitable-weak / Leray-Hopf plus LEI floor?
5. Given the mechanism, comparator, and setup burdens, should Step 12 classify
   the branch as ready for exact-identity expansion or as already obstructed?

- `local concentration exclusion` is the only honest default endpoint on the
  run-008 record; continuation should be rejected unless the observable already
  behaves like near-coercive norm control.
- The inherited suitable-weak / Leray-Hopf plus LEI floor is the only locally
  frozen theorem-facing solution class, and stochastic-flow objects likely sit
  above it rather than inside it.
- The repository supports a strong solution-class warning but not a positive
  admissibility bridge from that weak-solution floor to the stochastic
  observables on offer.

Confirm, refine, or reject these hypotheses from source-backed evidence.

## Success Criteria

- Produce one admission table with columns:
  `item`, `frozen choice`, `status`, `support`, `main burden`.
- Label major claims `[VERIFIED]`, `[INFERRED]`, or `[PROPOSED]`.
- End with one explicit conclusion:
  `setup passes for Step 2`, or
  `solution-class / setup obstruction already active`.

obstruction.
