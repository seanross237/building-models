# Exploration 001 Goal

## Objective

Freeze the smallest serious setup menu for `step-008`.

This is not a host-space ranking exercise. The point is to decide, from local
repository materials, which primary solution frameworks and theorem-level
endpoints are actually live after `step-007`, `decision-011`, and the `run-005`
replan, and which options are only cosmetic repackagings or unsupported
retrofits.

## Required Questions

Answer all of the following from local repository materials:

1. What candidate primary solution frameworks are still serious for this chain?
   Distinguish clearly between:
   - suitable-weak / Leray-Hopf with local energy inequality,
   - mild / maximal-lifespan critical-space evolutions,
   - and any claimed mixed bridge architecture.
2. Which of those are actually supported as frameworks, rather than merely
   appearing as host-space or compactness language?
3. What theorem-level contradiction endpoints or cash-out lines are locally
   supported for each serious framework candidate?
4. What exact inherited baseline does the repository already freeze through the
   suitable-weak / local-energy architecture?
5. Which options should be rejected as cosmetic variants, unsupported bridge
   fantasies, or packaging drift?

## Working Hypotheses To Test, Not Assume

- The only fully supported architecture is the inherited suitable-weak /
  Leray-Hopf plus LEI floor.
- `L^3` and `\dot H^{1/2}` are serious only as ambiguous critical-space
  languages, not as frozen frameworks.
- No mixed bridge architecture is actually supported locally.
- The strongest locally supported contradiction line is still LEI-adjacent:
  epsilon-regularity / endpoint-regularity exclusion.
- Backward uniqueness, Liouville-type exclusion, ancient-solution exclusion,
  or frequency-cascade routes may appear as candidate endpoint families, but
  not as package-cleared live endpoints.

Confirm, refine, or reject those hypotheses from source-backed evidence.

## Success Criteria

- Produce a compact framework-and-endpoint menu with columns:
  `candidate`, `actual local support`, `theorem endpoint`,
  `baseline compatibility`, `rejection or survival note`.
- Label major claims `[VERIFIED]`, `[INFERRED]`, or `[PROPOSED]`.
- State the inherited local-energy baseline explicitly enough that later work
  cannot silently change it.
- End with one narrow menu:
  the exact candidates still serious for `step-008`,
  and the ones rejected before any coherence stress test.

## Failure Criteria

- The report treats host-space labels as if they were themselves frameworks.
- The report allows the theorem endpoint to drift with the candidate.
- The report invents a mixed bridge or temporal package that is not supported
  by local sources.

If failure occurs, say so directly and recommend a setup obstruction.
