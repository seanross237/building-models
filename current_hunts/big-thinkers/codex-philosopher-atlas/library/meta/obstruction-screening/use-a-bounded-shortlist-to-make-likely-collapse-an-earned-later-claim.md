# Use A Bounded Shortlist To Make Likely Collapse An Earned Later Claim

If the local record already suggests a negative baseline, still freeze a
narrow candidate family before declaring the branch dead.

A bounded shortlist turns "this probably collapses" into a later earned
statement because each candidate has to fail on the same fixed ledger. Without
that shortlist, the branch is being dismissed by prejudice rather than by
audit.

If that shortlist is already frozen on disk, do not reopen branch-adjacent
families just to make the slate look richer. Keep at least one reserve
comparator, but spend active slots only on packages that still carry a
nontrivial claimed route back to the same frozen burden. A family whose most
honest localization immediately returns to the same burden with no sharper
candidate-specific structure should stay reserve-only or be rejected rather
than consume an active audit slot.
