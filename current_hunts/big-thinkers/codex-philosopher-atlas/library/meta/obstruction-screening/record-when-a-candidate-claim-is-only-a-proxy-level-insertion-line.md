# Record When A Candidate Claim Is Only A Proxy-Level Insertion Line

When a branch can write a theorem-facing candidate sentence but the local
record still supports only a package-free insertion into an existing
criterion, record that status explicitly instead of treating the sentence as
provisional theorem success.

Operational rule:

- if the claim only says some new ingredient should insert into an existing
  criterion once the right package, observable class, or proof mechanism is
  supplied, file it as an insertion line;
- if the supporting audit still leaves the ingredient dynamically weak,
  package-dependent, or proxy-level, keep the branch verdict negative or
  provisional rather than upgrading it to an earned theorem statement;
- and treat this narrowing as useful output, because it preserves the honest
  boundary between theorem-facing language and actual theorem delivery.
