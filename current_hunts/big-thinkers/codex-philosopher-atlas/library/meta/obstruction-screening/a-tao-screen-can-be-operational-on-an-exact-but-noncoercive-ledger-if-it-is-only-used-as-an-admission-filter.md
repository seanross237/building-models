# A Tao Screen Can Be Operational On An Exact-But-Noncoercive Ledger If It Is Only Used As An Admission Filter

When a branch has not yet earned a coercive invariant package, Tao sensitivity
can still become operational, but only in a narrower role.

First separate exact structure from bookkeeping. If the local record only
supports an exact triad-level core, treat any dyadic shell ledger as an
aggregation of that core rather than as a stronger new invariant.

Then use the Tao screen only as an admission filter: ask whether a candidate
feature survives canonicalization, genuinely depends on exact-NS packet
constraints, and disappears only when those constraints are replaced by
averaged-circuit freedoms.

Do not overread that result. A passing Tao discriminator does not by itself
produce a one-sided gain on the theorem-facing bad term, and it does not
rescue an exact-but-noncoercive ledger from noncoercivity.
