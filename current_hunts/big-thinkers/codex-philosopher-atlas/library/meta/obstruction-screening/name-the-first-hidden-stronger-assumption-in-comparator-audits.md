# Name The First Hidden Stronger Assumption In Comparator Audits

Comparator audits should default to naming the first hidden stronger
assumption that would be needed to keep the route alive.

That hidden upgrade is often the real collapse point. In this mission family it
showed up as stronger `nabla xi`-type control on the direction side and
tube-adapted relocalization on the tube side. Once that upgrade is explicit,
the audit can classify the route honestly: if the comparator only survives by
importing the stronger assumption, treat that as a negative finding rather than
as an implicit rescue.

For proxy-based geometry routes, make the admission version of that screen
equally explicit. Hidden Eulerian component matching across the full
diffusion-scale window and any forbidden relocalization are not minor
presentation choices; they are part of the actual bridge burden back to the
theorem floor. If the proxy route only cashes out after those upgrades, the
branch has already lost at admission.
