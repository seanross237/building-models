# When All Audited Candidates Are Only Dynamically Weak, Stop Before Representation Work

In a wide-funnel geometry branch, fragility screens are not just leftover
comparators. They help state the earliest honest stop condition once the live
candidate fails to become dynamically plausible on the audited package.

The reusable rule is simple:

- if every audited candidate is only `informative but dynamically weak` or
  `static-only`
- and none of them names a credible full-stretching gain that survives the
  fixed localization and time window

then the chain should stop before any further kernel-level representation work,
decomposition design, or other downstream formalization.

At that point the branch does not need a new rewrite. It needs a genuinely new
ingredient.
