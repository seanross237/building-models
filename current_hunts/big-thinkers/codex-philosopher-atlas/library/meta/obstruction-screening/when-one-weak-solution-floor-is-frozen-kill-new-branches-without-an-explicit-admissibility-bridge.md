# When One Weak-Solution Floor Is Frozen, Kill New Branches Without An Explicit Admissibility Bridge

Once a mission has already frozen one theorem-facing weak-solution floor, do
not treat a new branch as admitted just because its proposed objects sound
plausible or exact in principle.

Ask one admission question immediately:
does the local record already contain an explicit bridge from that inherited
floor into the new branch's operative objects?

If not, the branch is still at setup obstruction.

Planning-level warnings about smooth flows, filtrations, optional stopping,
localization, or reconstruction are already enough to support that negative
label when no bridge packet exists on disk, because those warnings identify
the stronger structure the branch would need before its objects can count near
the singular regime.

If the only live content is a dynamic proxy object rather than a theorem-floor
native one, run that admission check before package selection. Proxy-based
language is not a free extension of the frozen floor: the branch still has to
write the exact bridge burden back to the inherited endpoint object and ask
whether that descent is actually cheaper than the target theorem.

For geometry branches, hidden Eulerian component matching, forbidden
relocalization, and imported `nabla xi` control are already part of that
bridge burden. If they are needed just to descend from the proxy class to the
theorem floor, record `theorem-floor obstruction` rather than treating them as
later technical cleanup.
