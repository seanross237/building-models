# Build A Dedicated Prior-Art Packet Before Benchmarking A Literature-Heavy Branch

When a branch sits next to a large prior-art literature, build a small factual
packet for the benchmark target before later screening steps depend on planning
artifacts.

That packet does not need to be exhaustive. It only needs enough theorem-level
detail to make later novelty checks sharper than headline-only comparisons and
to keep benchmark work from leaning too hard on mission-planning prose.

When the candidate sits close to endpoint or critical-regularity theorems, log
that adjacency as an overlap burden that raises the admission bar. Name
proximity is pressure against novelty until the packet shows a distinct
theorem-level route.

Do not count one named literature neighborhood as a completed comparator
packet. The packet should force one nearest comparator packet with these exact
entries:

- one exact shared structure already present in that comparator;
- one exact mathematical delta beyond it;
- one already-standard gain the local record is willing to concede; and
- one same-currency cash-out on the frozen theorem burden, or one explicit
  transfer lemma that gets back there.

If the step cannot write that packet concretely, stop the branch before formal
identity or packaging work starts pretending the comparator burden has already
been discharged.
