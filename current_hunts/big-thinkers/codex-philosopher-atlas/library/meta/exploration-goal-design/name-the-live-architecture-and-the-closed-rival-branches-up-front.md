# Name The Live Architecture And The Closed Rival Branches Up Front

When a mission step must choose one inherited architecture from a constrained
menu, the goal should name both the live target architecture and the already
closed rival branches explicitly.

Treat prior negatives as hard constraints rather than background color.
Otherwise the exploration drifts toward the most familiar or best-supported
option even when that option is already closed by the local record.

This framing makes the real question sharper: not just which architecture is
best supported, but which one is both supported and least already closed. It
also reduces redundant exploration time in mission families where several rival
branches have already been killed for different reasons.
