# Meta Library Index

Methodology, goal-design patterns, and system behavior lessons for Philosopher Atlas.

## Topics

- `exploration-goal-design/`: reusable prompt-design lessons for formula-first
  obstruction reports, killed-mode accounting, notation normalization, source
  scoping, prior-art packet design, earlier fixation of the operative control
  representation, anti-overfit localization design, architecture-choice
  prompts that distinguish live from already-closed rival branches, and
  diffusion-scale Eulerian tests for candidates whose only live content is
  dynamic persistence, plus bounded procedural novelty claims for steps that
  only freeze an auditable candidate table, and role-ledger-first exact-object
  reconstruction that separates inherited mechanism from branch-level policy
  choices such as leakage rules. The same topic now also records a sharper
  prior-art packet rule: naming one literature neighborhood is not enough;
  later steps should demand one nearest comparator with one exact shared
  structure, one exact mathematical delta, and one same-currency cash-out or
  explicit transfer lemma before identity work proceeds.
- `obstruction-screening/`: reusable rules for deciding whether a candidate
  actually acts on the live obstruction instead of merely restating it, with
  an added geometry-specific screen against static or Lamb-vector-only claims,
  plus a localization-exactness screen for rewrite audits, an explicit
  cutoff-and-ledger question for estimate-level gains, a hidden-assumption
  screen for comparator audits, an architecture-drift screen for candidates
  that only survive by becoming a different branch, a stop rule for branches
  whose full audited candidate set is only dynamically weak, a shortlist rule
  for branches whose baseline already looks negative, a continue-or-kill
  rule for branches already frozen to one bad term and one gain currency, and
  host-space audit rules distinguishing genuine framework-plus-endpoint
  packages from mere compactness prestige, scale-invariant appearance, or
  silent framework drift inside an inherited architecture. The same topic also
  records that an organized burden ledger is still not a live setup until one
  temporal architecture and one bridge are actually delivered, and that
  endpoint-first branches should separate actual theorem endpoints from
  mechanism language at the first gate, freeze admission rules on the exact
  endpoint object, freeze one-floor windows as one primary endpoint plus at
  most one stricter reserve together with solution class and localization, and
  treat gate clearance as narrower than tensor admission.
  It now also records that once one ledger is frozen, prior-art overlap should
  collapse family labels before packaging work begins. It also records a sharper
  threshold rule: once some modes are already killed by the test structure,
  only a non-cosmetic improvement on the same live inequality after later debt
  is restored counts as progress. It also records a support-stability rule:
  when object identity is central, separate true symmetries, fixed
  canonicalization, and genuinely different object changes before calling a
  representation move cosmetic. The same topic now also records a scope-lock
  rule for early setup gates: do not count the lock as real until both the
  baseline cash-out and the minimum gain threshold or admissible gain notion
  are frozen in the same burden currency that the later coercivity test must
  beat. It also records a transfer rule: if a claim is stated in a different
  burden currency, demand an explicit transfer lemma back to the same
  theorem-facing burden and reject debt repaid in some different ledger as
  narrative bookkeeping. It also records a Tao-screen rule for exact-but-
  noncoercive branches: separate exact structure from bookkeeping first, and
  use Tao sensitivity only as an admission filter unless it cashes out as a
  real gain on the frozen bad term. It now also records that for
  reformulation-first audits the endpoint-exchange question is often the
  earliest live gate, so cleaner native-variable language without an explicit
  same-currency return should be labeled exchange failure before later
  anti-repackaging or overlap screens. It now also records a label-order rule
  for prior-art screens: when one narrow
  discriminator survives but setup or calibration already fails, keep
  prior-art overlap secondary and let the earliest honest obstruction label
  stay primary. It now also records a stochastic-transport setup rule: one
  burden plus one comparator neighborhood is still only menu trimming until
  the branch freezes one theorem-facing observable and one sentence of
  leverage, with `local concentration exclusion first` as the sharper default
  filter before continuation language. The same topic now also records the
  solution-class admission rule for that branch family: once one theorem-facing
  weak-solution floor is frozen, kill a new branch unless an explicit
  admissibility bridge is on disk, and do not mistake endpoint-ordering
  success for branch admission. It now also records an anti-repackaging
  rule-writing lesson: state the failure buckets in the branch's own burden
  currency so expectation-only, filtration-only, localization-only, or
  parasitic-framework-only gains can be rejected mechanically. The same topic
  now also records that classification should be made on the restored full
  theorem-facing burden, and that any `ambiguous` middle bucket should be kept
  only for one exact missing transfer step. It also records a slate-discipline
  rule for frozen shortlists: do not reopen branch-adjacent families to make a
  menu look richer, keep at least one reserve comparator, and use active slots
  only for candidates that still name a nontrivial route back to the same
  frozen burden. It also records a sharper rewrite-audit distinction: derive
  the native exact equation first, charge debt at the first localization
  step, and separate candidates that act on the frozen burden natively from
  those that only touch it after reconstruction. It now also records a
  theorem-floor admission refinement for proxy-based branches: run the bridge
  check before package selection, count hidden Eulerian matching,
  relocalization, and imported stronger control as bridge debt rather than
  later cleanup, and keep `theorem-floor obstruction` primary when that debt
  is already theorem-scale. It now also records a criterion-first geometry
  refinement: freeze the endpoint to the most concrete named criterion on
  disk, force the gain currency to be `criterion input` when that endpoint is
  a criterion family, and label package-free insertion sentences as narrower
  than earned theorem statements.
- `workflow-monitoring/`: reusable session-monitoring lessons for telling apart
  created scaffolds, partial artifacts, and genuinely completed exploration
  outputs, including stronger completion checks than sentinel existence alone.
  The same topic now also records that launcher-side stalls should be logged
  separately from research-side inconclusiveness.
