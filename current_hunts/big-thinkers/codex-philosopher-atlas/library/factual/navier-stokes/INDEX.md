# Navier-Stokes

Reusable findings on exact support objects and representation policy for
Fourier/helical packet branches.

## Entries

- `support-means-a-role-labeled-helical-packet-with-mandatory-conjugate-completion.md`:
  in this mission family, `support` is a finite role-labeled helical packet
  together with the forced conjugate completion needed for a real-valued
  velocity field.
- `conjugate-completion-is-mandatory-canonicalization-not-primary-packet-identity.md`:
  conjugate completion must be enforced in the canonical realization, but it is
  bookkeeping for a real-valued packet rather than the packet's primary
  identity.
- `helical-signs-amplitude-normalization-and-phase-anchor-belong-to-the-frozen-packet-sheet.md`:
  later candidate packets must declare one sign sheet, one amplitude
  normalization, and one phase anchor instead of leaving those choices implicit.
- `packet-audits-must-separate-true-symmetries-canonicalization-and-substantive-model-changes.md`:
  packet audits should distinguish genuine invariances from fixed
  canonicalization choices and from changes that create a different packet
  model.
