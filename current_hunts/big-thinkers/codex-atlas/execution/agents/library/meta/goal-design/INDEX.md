## Entries

- **one-task-per-exploration.md** — Never combine multiple cognitive tasks in one exploration; multi-task goals time out or produce shallow results. Corollaries: split conceptual mapping + computation across explorer types; depth beats breadth for generating insight; **exception: two genuinely parallel computations with shared setup can combine** (neither depends on the other's result).
- **specify-failure-paths.md** — Include "if this fails, explain why and propose alternatives" to turn negative results into positive leads; includes variants for physical implication explorations, domain constraints, per-assumption honesty, no-go framing, **formalism applicability pre-screening** (ask "can this formalism yield the modification in principle?" before committing), **explicit skip permission for ambiguous sub-tasks** (say "skip if normalization is unclear; note what you'd need" — prevents silent omission), **request structural explanation when construction may fail** (RH s002-E006 — "if the construction fails, determine if the failure is empirical or structural; if structural, provide the proof" → elicits algebraic impossibility proofs that close entire construction classes, far more valuable than numerical negatives alone), and **quantitative falsification criterion for hypothesis-testing computations** (vasseur-pressure E004 — state the falsification criterion explicitly: "if X is constant in Y, hypothesis is falsified"; seed alternative paths).
- **preload-context-from-prior-work.md** — Paste relevant prior findings directly into goals; explorers have no memory and will re-research without it. *Updated: added variant for pre-loading exact formulas/numerical values (SED E004 cut exploration time from 36→14 min); added **"Verify Formulas Before Preloading"** variant (RH E004); added **"Verify Code Before Preloading"** variant (SED s002 E005 — GOAL.md code had ω_max declared but not applied in PSD body; explorer caught + fixed the bug, but 20 min lost; checklist: every declared param must appear in body; test against known result before pasting); added **closure-stack preload** variant (anatomy meta-002 — list already-closed routes explicitly so follow-on work does not re-open dead leads).*
- **use-line-count-targets.md** — Set explicit line count targets (150-800 depending on task type) to prevent both under-writing and scope creep.
- **name-specific-authors-and-papers.md** — Name target authors/papers for literature searches; combine with "and any others you find" for thoroughness. **NEW variant: "also survey related approaches by same group/school"** [vasseur-pressure E003 — most valuable finding was incidental Choi-Vasseur discovery, not the targeted Tran-Yu assessment].
- **use-classification-schemes.md** — Provide explicit severity/type classifications (FATAL/SERIOUS/MODERATE, DISCRIMINATING/INHERITED, etc.) to force honest assessment.
- **use-absolute-file-paths.md** — Always use full absolute paths in goals; relative paths cause explorers to write to wrong directories.
- **specify-temporal-recency.md** — Specify explicit time windows (e.g., "focus on 2024-2026") to get genuinely current results instead of established older work.
- **request-equations-for-construction.md** — For construction/synthesis tasks, explicitly ask for equations and derivations; without this, output is conceptually compelling but mathematically schematic.
- **require-claim-attribution.md** — Ask explorers to distinguish which claims are sourced from papers vs. their own reasoning; prevents blending literature and original analysis.
- **specify-rigor-level.md** — Explicitly request the rigor level (e.g., "theorem-level precision," "paper-by-paper inventory") to get detailed technical tracking rather than adequate-but-shallow summaries.
- **specify-computation-parameters.md** — For numerical simulation goals, specify exact parameters; includes variants for: Python code templates (strongest form; SED E005 confirms — fastest exploration when used correctly), **sequence computations fast→slow to enable early filtering** (SED E005), **specify integration region explicitly** (SED E005), **startup diagnostics for fragile imports** (RH s002-E006 — `python3 -c "import scipy.optimize"` before computation; pair with explicit fallback formula), **specify expected output file keys** (RH s002-E006 — provide exact key names for .npz/.pkl extraction so explorer reads saved data, not re-computes), threshold-finding scan questions, naming exact formula names, requesting comparative ratios, exact code for baseline comparison values, physics sanity checks, formula disambiguation, print-first, cross-check-first, parameter bins, normalization matching, sign+magnitude; **NEW: hierarchy-of-approximations for theory/synthesis explorations** (SED s003-meta-001 — for theory goals, provide the reference values at each approximation level so the explorer can construct the full picture, not just verify the formula).
- **prioritize-novelty-assessment.md** — When a specific theory/formula is identified, make novelty assessment Task 1 before validation or extension; includes variants for universal constants (literal string search), embedding "verdict on novelty" as mandatory output section, and **"find the closest prior work" framing** (sharper than "has this been published?" — forces equation-level comparison).
- **distinguish-identity-from-mechanism.md** — Require explorers to label whether a mathematical relationship is an algebraic identity (trivially true) vs. a physical derivation (requires mechanism); prevents conflating elegant algebra with physics.
- **require-mechanism-layer-maps.md** — **NEW (anatomy-of-averaged-ns-blowup-firewall meta-001), UPDATED (meta-002, exact-ns-no-near-closed-tao-circuit meta-001).** When a paper embeds a toy mechanism into a PDE, require explicit separation of: (1) exact operator/PDE, (2) reduced shell/circuit system, (3) final induction/theorem. Also require a variable-role table (carrier, clock, trigger, conduit, output). **Updates:** for firewall/intervention work, add a literal channel/intervention table keyed to the reduced mechanism's actual transfers; if the reduced mechanism is quadratic, encode it as a monomial hypergraph rather than just a node graph. Prevents attacking the wrong object in later work and keeps comparisons mechanism-level.
- **build-adversarial-suppression-into-first-audit.md** — **[NEW, exact-ns-no-near-closed-tao-circuit meta-001]** If the exploration may conclude "no low-leakage configuration exists," require at least one configuration chosen specifically to suppress the suspected leakage before accepting that conclusion. Prevents naive first-ansatz failures from masquerading as structural impossibility.
- **instruct-incremental-writing.md** — Include "write section by section, not all at once" in GOAL.md to prevent multi-minute stalls. **For multi-test/staged explorations, use section-specific checkpoints: "after Test N, write Test N section before proceeding to Test N+1."** **[SECTION COMPLETE] marker variant**; **research-buffering stall is systematically unpreventable** — fix is kill + extract + manual write or split-search-from-synthesis. **Adversarial per-claim checkpoint**. NEW: **write-during-long-computation variant** [RH s003-meta-009] — for computations >15 min, submit as background job and write intermediate results first; decouples writing from computation to prevent context-pressure loss of all results. *12 confirmations across 7 missions.*
- **require-quantification-in-stress-tests.md** — In stress-test/adversarial explorations, ask explorers to compute quantitative bounds for each objection (not just argue it); a computed catch-22 is a finding, an argued one is a concern. **Extended to model comparisons:** also require χ²/dof for rotation-curve/observational fits — "the curve looks similar" is not a finding.
- **provide-specific-tools-for-reformulation.md** — For reformulation/extension explorations, specify the exact mathematical tools to use (e.g., "use RT formula + HQEC"); don't make the explorer survey approaches.
- **require-independent-verification-baseline.md** — For complex recursive/algorithmic computations (e.g., BCFW at 6-point), provide a non-recursive independent formula as the ground truth target; two implementations of the same algorithm cannot cross-validate each other.
- **require-gap-analysis-in-formal-mappings.md** — In formal mapping explorations, require "at least 3 gaps/limitations" analysis; without this, explorers underreport scope limitations and overstate the mapping's validity. Distinct from failure-paths: this is about honest scoping of a successful result.
- **allow-explorer-synthesis.md** — Don't over-specify conclusions; specify what to investigate and what format to produce, but leave synthesis conclusions open. Explorers can discover structural patterns (e.g., two-tier tree-vs-loop insight; N²/p ≈ 250-310 ratio; underexploited constraints insight; unexpected no-go theorems) the strategizer didn't anticipate — often the most valuable outputs. **NEW variant: "What would be a breakthrough?" primes explorer for high-value output** without specifying what the answer should be. **NEW variant: "Explain WHY this agrees with [prior result]" elicits novel bonus derivations** — the mechanism explanation is more valuable than the consistency check itself (SED s003-meta-001: symmetry argument from P&C consistency check).
- **separate-numerical-from-physics-parameters.md** — When designing parameter scans, explicitly fix numerical method parameters (dt, precision) separately from physics parameters (ω_max, coupling); "natural" co-variation choices (e.g., dt=π/ω_max) confound analysis and cause instabilities at scan extremes.
- **require-baseline-adjusted-significance.md** — For trend detection vs. a control parameter, require both raw significance and baseline-adjusted significance; raw comparison can overstate trend significance by ~2× when a β=0 offset exists.
- **require-baryonic-completeness.md** — For galaxy rotation curve fits, require total baryonic mass (stars + gas + He correction), not just stellar mass; use BTFR as sanity check; gas can dominate over stars in dwarf/irregular galaxies (NGC 2403: gas = 85% of total mass).
- **specify-modified-inertia-vs-gravity.md** — In any modified-dynamics exploration (MOND-like theories), require the explorer to specify upfront whether the modification is to inertia (F = m_i a) or gravity (Poisson equation). Not a labeling choice — has decisive observational consequences: modified inertia cannot explain cluster gravitational lensing (Poisson equation unchanged → photons unaffected → lensing mass = baryonic mass only). Failure to specify costs 5–7 explorations before the fatal test is run.
- **require-matrix-sanity-checks.md** — For matrix construction + spectral analysis goals, add an explicit pre-analysis sanity check step: verify the matrix is Hermitian and non-degenerate (no all-zero eigenvalue sets) and the spacing distribution is roughly bell-shaped before running statistics. Without this, explorers proceed through full analysis on degenerate matrices (e.g., odd Dirichlet characters → antisymmetric → zero matrix), producing meaningless outputs that look like findings.
- **ask-forward-looking-literature-questions.md** — Always include "does this combination/result appear in the literature?" even when confident about the state of the art. Literature in active fields can advance faster than the library; a single forward-looking question is the cheapest way to discover this. Yang-mills s002 E001: found CNS 2025 (area law at β < 1/24, doubling SZZ threshold) as the most important finding, from exactly this type of question.
- **adversarial-synthesis-goal-structure.md** — NEW (SED s003-meta-004): 4-part output structure for finishing-strategy adversarial synthesis goals: (1) adversarial review per claim with verdict+novelty rating, (2) prior art search for the conclusion, (3) grand synthesis, (4) consolidated claims table. Pre-load all claims with evidence summaries, prior art candidates, and strongest objections directly into GOAL.md. Long goals (200-300 lines) are unavoidable and worth it. Tier 5 requires concrete falsifiable paths per claim, not just literature checks. *Updated (thermal-time s003-meta-002): prior art discovery can be the most important outcome.* **UPDATED (vasseur-pressure E009): evidence grading (A/B/C/D/F per claim) makes strength immediately visible; "weakest link" deliverable forces rank-ordering; novel claims list by adversarial reviewer (not original explorer) produces more conservative assessments; require fix specification per weakness to enable action rather than paralysis.**
- **require-gaussian-control-for-non-gaussian-states.md** — When computing with non-Gaussian states using Gaussian approximation (e.g., quadratic modular Hamiltonian from covariance matrices), always request a parallel computation with a Gaussian control state (coherent state) to isolate approximation artifacts from physical effects. **CONFIRMED (thermal-time s003-meta-001):** two-tier pattern (coherent=easy validation + squeezed=hard test) executed successfully; Gaussian caveat on excited-state modular flow finding now closed.
- **scope-computations-to-minimum-diagnostic.md** — For computationally intensive explorations, specify the MINIMUM computation needed. "Compute ratio at K=N=500,1000,2000,3000,5000 and extrapolate" is better than "validate with 5000 zeros and larger GUE" — the explorer will run all available approaches instead of the one diagnostic test. Distinct from one-task-per-exploration (which is about task types); this is about scoping within a single computational task.
- **characterize-maximizers-not-just-bounds.md** — **[NEW, YM s003-meta-002]** When a proof strategy involves bounding λ_max, frame goals as "characterize the set of maximizers" — the structural characterization (e.g., "F=4d ↔ pure gauge") reveals geometry and suggests proof routes. Distinct from allow-explorer-synthesis (question structure, not conclusion openness).
- **verify-goal-claims-before-delegating.md** — **[NEW, YM s003-meta-005/s003-007]** Before writing GOAL.md citing a prior result (e.g., "M(Q) ≼ M(I) confirmed by E004"), verify the cited claim actually matches what the exploration found. Two-level fix: (1) strategizer re-reads REPORT.md before citing; (2) GOAL.md includes "first compute X for 3-5 random examples — if it fails, pivot." Prevented two wasted explorations in yang-mills s003. **NEW variant: "verify paper existence before citing"** [vasseur-pressure E003 — "Tran-Yu (2014, AIHP)" doesn't exist; actual program is 5 papers 2015-2021]. Distinct from preload-context (providing findings) and include-trivial-control-checks (validating code).
- **include-predecessor-comparison-for-reformulations.md** — **[NEW, barandes-stochastic E004]** When evaluating a reformulation's value, provide a successful predecessor (e.g., amplituhedron) as structured comparison framework. Produces multi-dimensional evaluation table far clearer than standalone philosophical assessment. Variants: "one paragraph answer to the key question" (forces concrete honest synthesis) and "rank papers by importance" (for paper cluster assessments). Distinct from comparison-exploration-pattern (which is about comparing TWO approaches; this is about providing a BENCHMARK for evaluation).
- **anticipate-symmetry-degeneracies-in-ics.md** — **[NEW, navier-stokes E003]** When specifying ICs for numerical explorations involving directional/alignment-dependent quantities, anticipate symmetry degeneracies that make the quantity vanish identically. Z-invariant vortex tubes have zero VS (strain-vorticity orthogonality); ABC/Beltrami flows have zero VS (integration by parts). Either exclude degenerate ICs or flag them as known edge cases. Distinct from require-matrix-sanity-checks (runtime check) and specify-computation-parameters (parameter values).
- **distinguish-typical-vs-worst-case-bounds.md** — **[NEW, navier-stokes E006, UPDATED vasseur-pressure E002]** For bound-tightening explorations, explicitly require explorers to classify each tighter estimate as WORST-CASE BOUND (provable, usable in proofs) or TYPICAL-CASE VALUE (statistical property, not a bound). Without this, explorers present impressive typical-case improvements (C_{L,eff} ~ Re^{-3/8}) as bound tightenings when they cannot serve as proof tools. Include template: "If typical is dramatically better than worst-case, identify the structural property separating them." **NEW variant: "empirical analogue vs analytical quantity"** — when computing empirical analogues of analytical bounds, require explicit statement of what the empirical quantity IS and ISN'T (beta_eff != beta_p; the former tests smooth solutions, the latter bounds all weak solutions). Distinct from require-quantification-in-stress-tests (quantifying objections) and specify-failure-paths (handling failures).
- **adversarial-proof-review-structure.md** — **[NEW barandes-stochastic E005, UPDATED yang-mills-conjecture E007, UPDATED navier-stokes E008]** For adversarial reviews of a specific proof, use a 4+1-step structure: (1) trace proof step by step, (2) identify key assumption, (3) check if derived or imported, (4) check prior art, **(5) check whether the proved statement actually implies the claimed conclusion**. Variants: always require steelman when adversarial likely negative; ask "does the paper acknowledge its own limitations?"; **for matrix inequality proofs, verify reduction from operator bound to quadratic form bound**; **pre-specify 3-4 attack vectors per claim** [NS E008 — directed reviewer to productive criticisms, caught apples-to-oranges comparison and (5/9)^{1/4} misattribution]; **dual-mode adversarial: standard explorer (logic+literature) + math explorer (recomputation)** [NS E008 — standard-only missed opportunity for independent numerical verification]. Distinct from adversarial-synthesis-goal-structure (multi-claim review) and adversarial-check-between-phases (timing).
- **flag-cross-framework-connections.md** — **[NEW, vasseur-pressure E006]** When an exploration reveals a connection between two independently well-studied frameworks that have not been connected in the literature, flag this explicitly in future goals. Such connections are potential novel territory. E006: geometric regularity criteria + De Giorgi iteration = uncharted intersection. Distinct from prioritize-novelty-assessment (checking a specific result) and gap-finding (gaps within a single program).
- **adversarial-review-needs-complete-parallel-context.md** — **[NEW, vasseur-pressure S2-E007]** When launching an adversarial review in parallel with constructive explorations, the reviewer may lack findings from parallel work. Either launch adversarial reviews AFTER constructive explorations complete (preferred), or include summaries of parallel work as supplementary context. S2-E007 flagged "non-CZ pressure incomplete" when S2-E006 had already resolved it. Distinct from adversarial-check-between-phases (timing), preload-context (general loading), adversarial-synthesis-goal-structure (output format).
- **require-sensitivity-table-for-proof-analysis.md** — **[NEW, vasseur-pressure S2-E001]** When an exploration analyzes a proof chain, require a sensitivity table: step, mathematical tool, exponent contribution, sensitivity dβ/dδ, sharpness classification (SHARP/POTENTIALLY LOOSE/UNKNOWN), notes. Forces per-step assessment instead of global sharp/loose claims. S2-E001: table was the single most actionable output, immediately identifying Chebyshev as the only non-sharp link. Distinct from require-trend-tabulation-for-negative-results (empirical data) and use-classification-schemes (general classification). Complementary to decomposition-audit-before-attacking-barrier (when to audit vs. what format to require).
- **ask-what-replaces-the-bottleneck.md** — **[NEW, vasseur-pressure E008]** When testing an alternative formulation against a known barrier, always ask "what is the NEW bottleneck?" If the same numerical barrier reappears from a different mechanism, the obstruction is structural (not formulation-specific). E008: vorticity De Giorgi eliminates pressure but 4/3 reappears from trilinear form (1/2 + 5/6). The "what replaces it?" question was the most valuable part of the goal. Distinct from specify-failure-paths (general failure handling) and comparison-exploration-pattern (comparing approaches).
- **allow-analytic-extremizer-over-computation.md** — **[NEW, vasseur-pressure S2-E008]** When designing optimization-based explorations (SDP, LP, numerical optimization), allow explorers to find analytic extremizers. The computational approach should be the fallback, not the default. S2-E008: goal described ~100-line CVXPY SDP; explorer found the constant field u=(c,0,0) is a one-line extremizer that makes the SDP unnecessary. Protocol: state mathematical question clearly, specify computation as ONE possible tool, explicitly say "if you can find an analytic extremizer, do that instead." Distinct from allow-explorer-synthesis (conclusions open, not methods), check-bypass-not-just-improve-bottleneck (whether to use framework at all), specify-computation-parameters (parameters when computation IS needed).
