# Executive verdict

[VERIFIED] Decision target:

> Either there exists a preferred quantitative definition of a near-closed Tao circuit that is concrete enough to support an exact interaction ledger in Phase 1, or all plausible definitions still fail the precision gate and the strategy should stop with a definition-level negative result.

[CHECKED] Gate verdict: `pass`.

[CHECKED] There is a preferred testable definition, but only after one sharp correction: Tao's five roles cannot be modeled as five single Fourier modes. The exact object has to be a finite, sign-closed helical support split into five Tao role packets `A, B, C, D, A_next`.

[CHECKED] The main unexpected finding is structural: several Tao monomials are impossible for singleton roles in exact Fourier arithmetic. Examples:

- `A,A -> B` and `A,A -> C` cannot both occur with distinct singleton targets because one ordered pair has one sum.
- `B,C -> C` would require `k_B + k_C = k_C`, hence `k_B = 0`.
- `A,B -> A` would require `k_B = 0`.
- `D,A_next -> D` would require `k_A_next = 0`.

[CHECKED] This kills the naive five-conjugate-pair picture, but not the definition itself. The preferred object is packet-level, while Phase 1 should still compute an exact helical-triad ledger.

# Tao mechanism map to candidate circuit object

[VERIFIED] The Tao roles are

```text
A = current carrier,
B = slow clock,
C = tiny trigger,
D = conduit / rotor leg,
A_next = next carrier.
```

[VERIFIED] The load-bearing Tao monomials are

```text
A,A -> B
A,A -> C
B,C -> C
A,C -> D
C,D -> A
A,B -> A
A,C -> A
D,A_next -> D
D,D -> A_next
```

[CHECKED] The exact NS object therefore has to be a role-level directed hypergraph `G_target` with these nine quadratic edges. A coarse five-node graph is not enough for Phase 1 because the audit must classify monomials, not just node adjacencies.

[VERIFIED] Exact NS interaction law:

```text
∂t û(k) + ν|k|^2 û(k) = -i Σ_{p+q=k} (q · û(p)) P_k û(q),
P_k = I - (k ⊗ k)/|k|^2,
û(-k) = overline{û(k)}.
```

[CHECKED] In the helical basis this becomes an exact scalar triad sum

```text
(∂t + ν|k|^2) u_s(k)
  = Σ_{p+q=k} Σ_{sp,sq} C^{s,sp,sq}_{k,p,q} u_sp(p) u_sq(q),
```

with exact Waleffe/Leray coefficient `C`. This is the smallest honest Phase 1 language.

# Preferred definition

[CHECKED] A near-closed Tao circuit is a tuple

```text
NCTC = (S, {P_A,P_B,P_C,P_D,P_A_next}, G_target, Θ, u|I).
```

## Exact data

[CHECKED] `S` is a finite helical support:

- `S` contains finitely many exact helical modes `(k,s)` with `k != 0`.
- `S` is sign-closed: if `(k,s)` is in `S` then `(-k,-s)` is in `S`.

[CHECKED] The packets `P_A,P_B,P_C,P_D,P_A_next` partition `S`.

- Each packet may contain several conjugate pairs.
- Packets are exact subsets of Fourier-helical space, not approximate shell packets.

[CHECKED] `G_target` is the nine-edge role hypergraph listed above.

[CHECKED] Each role edge `R1,R2 -> R3` must be witnessed by at least one exact helical triad

```text
(p,sp),(q,sq) -> (k,s),   p + q = k,
```

with the sources in `P_R1,P_R2` and the target in `P_R3`.

[CHECKED] For an exact state `u`, define the intended drive of one role edge `e = R1,R2 -> R3` by

```text
Drive_e(u)
  := Σ | C^{s,sp,sq}_{k,p,q} u_sp(p) u_sq(q) |,
```

summed over all exact helical triads in `S` realizing that role edge.

[CHECKED] Define total intended drive by

```text
Drive_target(u) := Σ_{e in G_target} Drive_e(u).
```

[CHECKED] Define internal spectator leakage

```text
Leak_in(u)
  := Σ | C^{s,sp,sq}_{k,p,q} u_sp(p) u_sq(q) |,
```

over exact helical triads with both inputs in `S` and target in `S` that are not assigned to any edge of `G_target`.

[CHECKED] Define external spectator leakage

```text
Leak_out(u)
  := Σ | Σ C^{s,sp,sq}_{k,p,q} u_sp(p) u_sq(q) |,
```

over target modes `(k,s)` generated by source pairs in `S` but with `(k,s) not in S`.

[CHECKED] Define the scalar leakage witness

```text
Leak(u) := (Leak_in(u) + Leak_out(u)) / Drive_target(u),
```

whenever `Drive_target(u) > 0`.

[CHECKED] This is the right scalarization because it is:

- exact-ledger computable,
- role-sensitive,
- conservative against cancellation.

## Dominance and delayed-trigger conditions

[CHECKED] Let `u|I` be an exact NS solution segment on `I = [t0,t2]`. It is an `NCTC` with tolerance `Θ` if there exists `t1 in (t0,t2)` such that:

1. Pre-trigger regime on `[t0,t1)`:

```text
Leak(u(t)) <= η_leak,
||u|_{P_C}(t)||_2 <= θ_pre ||u|_{P_A}(t)||_2,
Drive_{D,D->A_next}(u(t)) <= η_seed Drive_{A,A->B}(u(t)).
```

2. Hierarchy conditions on the relevant active times:

```text
Drive_{A,A->C}(u(t)) <= η_seed Drive_{A,A->B}(u(t)),
Drive_{B,C->C}(u(t)) >= Γ_amp Drive_{A,A->C}(u(t)),
Drive_{A,C->D}(u(t)) >= Γ_rot (Leak_in(u(t)) + Leak_out(u(t))),
Drive_{D,D->A_next}(u(t)) >= Γ_hand (Leak_in(u(t)) + Leak_out(u(t))).
```

3. Activation window on `[t1,t2]`:

```text
t2 - t1 <= Δ_act,
||u|_{P_C}(t1)||_2 >= θ_act ||u|_{P_A}(t1)||_2,
||u|_{P_A_next}(t2)||_2 >= ρ_next ||u|_{P_A_next}(t1)||_2.
```

[CHECKED] This passes the precision gate because every clause reduces to exact triad arithmetic, exact coefficient lookup, finite ledger sums, or explicit thresholds.

## What would count as a positive obstruction

[CHECKED] A positive obstruction in Phase 1 or 2 would show that every admissible packet support intended to realize `G_target` fails in at least one of these ways:

- one required role edge has no exact triad witness,
- `Leak(u)` has a support-independent lower bound above admissible `η_leak`,
- the hierarchy `η_seed, Γ_amp, Γ_rot, Γ_hand` cannot hold simultaneously,
- the delayed-trigger inequalities force zero-mode or equivalent forbidden degeneracy,
- activation cannot be separated from spectator growth on any admissible interval.

## What would count as a counterexample / near-circuit construction

[CHECKED] A counterexample would be an explicit finite support `S`, explicit packet partition, explicit helicities, and an exact NS solution segment `u|I` satisfying every clause above for a stated tolerance vector `Θ`.
