# System Behavior — Meta Library

Lessons about how explorers actually behave: stalling patterns, reasoning limits, failure modes, and how to work with (not against) these behaviors. **4 entries** (UPDATED: **explorer-stalling-and-nudge-pattern** [yang-mills-conjecture meta-E008 — 75+ min stall at 119 lines, new data point for context pressure variant]. Prior: [YM s003-meta-s003-006 — multiple simultaneous background scripts caution]; [YM s003-meta-005 — standard explorers can run Python eigenvalue computations]; [RH s003-meta-009 — context pressure].).

## Entries

- **explorer-stalling-and-nudge-pattern.md** — Explorers consistently stall after initial output; nudges reliably break the stall. Monitor and nudge after 3-5 minutes. Variants: **research-buffering stall** (iterative search + writeup; fix: kill + extract + manual write; architecture fix: split-search-from-synthesis); **pre-computation coding stall** (nudge must include "code NOW" + output path + code path); post-computation thinking stall (paste results directly into nudge); computation-done-but-still-debugging stall (data saved but explorer loops on tool failure); **context pressure emergency** (Math Explorers: 50+ min [E008] or 33 min with large-matrix ops [E009]; pre-supply key numerical results in emergency nudge for instant summary writing).
- **computation-vs-reasoning-limits.md** — Explorers can evaluate formulas and implement simulations but cannot perform novel analytical calculations. Updated: math explorers can self-diagnose code bugs when given expected-output context; spectral gap scans on 4^4 lattice take ~8 min/β value; **formula optimization (pure algebra) takes ~2 min**; **4D lattice ~3× slower than 3D**.
- **explorer-crashes-and-path-confusion.md** — Explorers occasionally crash from context exhaustion and write to wrong paths; always verify output after completion. **NEW variant: CWD drift during computation** — explorer's working directory silently drifts to a different exploration, executing wrong task entirely. Fix: add Task 0 "confirm working directory" with pwd + ls + expected path.
- **synthesis-vs-research-mode.md** — Explicitly tell explorers "do NOT search, only synthesize" for construction tasks or they default to web searches.
