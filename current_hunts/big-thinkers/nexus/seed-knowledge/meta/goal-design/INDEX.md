# Goal Design — Meta Library

Lessons about how to write exploration goals: scoping, specificity, context loading, output format, failure paths, and structural requirements. **30 entries.** (UPDATED: **prioritize-novelty-assessment** [yang-mills-validation E002 — equation-level comparison variant] and **verify-goal-claims-before-delegating** [yang-mills-validation E005 — verify analytical predictions independently variant]. Prior: UPDATED verify-goal-claims-before-delegating [yang-mills-conjecture S002-meta-E007].)

## Entries

- **one-task-per-exploration.md** — Never combine multiple cognitive tasks in one exploration; multi-task goals time out or produce shallow results. Corollaries: split conceptual mapping + computation across explorer types; depth beats breadth for generating insight; **exception: two genuinely parallel computations with shared setup can combine** (neither depends on the other's result).
- **specify-failure-paths.md** — Include "if this fails, explain why and propose alternatives" to turn negative results into positive leads; includes variants for physical implication explorations, domain constraints, per-assumption honesty, no-go framing, **formalism applicability pre-screening** (ask "can this formalism yield the modification in principle?" before committing), **explicit skip permission for ambiguous sub-tasks** (say "skip if normalization is unclear; note what you'd need" — prevents silent omission), and **request structural explanation when construction may fail** (RH s002-E006 — "if the construction fails, determine if the failure is empirical or structural; if structural, provide the proof" → elicits algebraic impossibility proofs that close entire construction classes, far more valuable than numerical negatives alone).
- **preload-context-from-prior-work.md** — Paste relevant prior findings directly into goals; explorers have no memory and will re-research without it. *Updated: added variant for pre-loading exact formulas/numerical values (SED E004 cut exploration time from 36→14 min); added **"Verify Formulas Before Preloading"** variant (RH E004); added **"Verify Code Before Preloading"** variant (SED s002 E005 — GOAL.md code had ω_max declared but not applied in PSD body; explorer caught + fixed the bug, but 20 min lost; checklist: every declared param must appear in body; test against known result before pasting).*
- **use-line-count-targets.md** — Set explicit line count targets (150-800 depending on task type) to prevent both under-writing and scope creep.
- **name-specific-authors-and-papers.md** — Name target authors/papers for literature searches; combine with "and any others you find" for thoroughness.
- **use-classification-schemes.md** — Provide explicit severity/type classifications (FATAL/SERIOUS/MODERATE, DISCRIMINATING/INHERITED, etc.) to force honest assessment.
- **use-absolute-file-paths.md** — Always use full absolute paths in goals; relative paths cause explorers to write to wrong directories.
- **specify-temporal-recency.md** — Specify explicit time windows (e.g., "focus on 2024-2026") to get genuinely current results instead of established older work.
- **request-equations-for-construction.md** — For construction/synthesis tasks, explicitly ask for equations and derivations; without this, output is conceptually compelling but mathematically schematic.
- **require-claim-attribution.md** — Ask explorers to distinguish which claims are sourced from papers vs. their own reasoning; prevents blending literature and original analysis.
- **specify-rigor-level.md** — Explicitly request the rigor level (e.g., "theorem-level precision," "paper-by-paper inventory") to get detailed technical tracking rather than adequate-but-shallow summaries.
- **specify-computation-parameters.md** — For numerical simulation goals, specify exact parameters; includes variants for: Python code templates (strongest form; SED E005 confirms — fastest exploration when used correctly), **sequence computations fast→slow to enable early filtering** (SED E005), **specify integration region explicitly** (SED E005), **startup diagnostics for fragile imports** (RH s002-E006 — `python3 -c "import scipy.optimize"` before computation; pair with explicit fallback formula), **specify expected output file keys** (RH s002-E006 — provide exact key names for .npz/.pkl extraction so explorer reads saved data, not re-computes), threshold-finding scan questions, naming exact formula names, requesting comparative ratios, exact code for baseline comparison values, physics sanity checks, formula disambiguation, print-first, cross-check-first, parameter bins, normalization matching, sign+magnitude; **NEW: hierarchy-of-approximations for theory/synthesis explorations** (SED s003-meta-001 — for theory goals, provide the reference values at each approximation level so the explorer can construct the full picture, not just verify the formula).
- **prioritize-novelty-assessment.md** — When a specific theory/formula is identified, make novelty assessment Task 1 before validation or extension; includes variants for universal constants (literal string search), embedding "verdict on novelty" as mandatory output section, and **"find the closest prior work" framing** (sharper than "has this been published?" — forces equation-level comparison).
- **distinguish-identity-from-mechanism.md** — Require explorers to label whether a mathematical relationship is an algebraic identity (trivially true) vs. a physical derivation (requires mechanism); prevents conflating elegant algebra with physics.
- **instruct-incremental-writing.md** — Include "write section by section, not all at once" in GOAL.md to prevent multi-minute stalls. **For multi-test/staged explorations, use section-specific checkpoints: "after Test N, write Test N section before proceeding to Test N+1."** **[SECTION COMPLETE] marker variant**; **research-buffering stall is systematically unpreventable** — fix is kill + extract + manual write or split-search-from-synthesis. **Adversarial per-claim checkpoint**. NEW: **write-during-long-computation variant** [RH s003-meta-009] — for computations >15 min, submit as background job and write intermediate results first; decouples writing from computation to prevent context-pressure loss of all results. *12 confirmations across 7 missions.*
- **require-quantification-in-stress-tests.md** — In stress-test/adversarial explorations, ask explorers to compute quantitative bounds for each objection (not just argue it); a computed catch-22 is a finding, an argued one is a concern. **Extended to model comparisons:** also require χ²/dof for rotation-curve/observational fits — "the curve looks similar" is not a finding.
- **provide-specific-tools-for-reformulation.md** — For reformulation/extension explorations, specify the exact mathematical tools to use (e.g., "use RT formula + HQEC"); don't make the explorer survey approaches.
- **require-independent-verification-baseline.md** — For complex recursive/algorithmic computations (e.g., BCFW at 6-point), provide a non-recursive independent formula as the ground truth target; two implementations of the same algorithm cannot cross-validate each other.
- **require-gap-analysis-in-formal-mappings.md** — In formal mapping explorations, require "at least 3 gaps/limitations" analysis; without this, explorers underreport scope limitations and overstate the mapping's validity. Distinct from failure-paths: this is about honest scoping of a successful result.
- **allow-explorer-synthesis.md** — Don't over-specify conclusions; specify what to investigate and what format to produce, but leave synthesis conclusions open. Explorers can discover structural patterns (e.g., two-tier tree-vs-loop insight; N²/p ≈ 250-310 ratio; underexploited constraints insight; unexpected no-go theorems) the strategizer didn't anticipate — often the most valuable outputs. **NEW variant: "What would be a breakthrough?" primes explorer for high-value output** without specifying what the answer should be. **NEW variant: "Explain WHY this agrees with [prior result]" elicits novel bonus derivations** — the mechanism explanation is more valuable than the consistency check itself (SED s003-meta-001: symmetry argument from P&C consistency check).
- **separate-numerical-from-physics-parameters.md** — When designing parameter scans, explicitly fix numerical method parameters (dt, precision) separately from physics parameters (ω_max, coupling); "natural" co-variation choices (e.g., dt=π/ω_max) confound analysis and cause instabilities at scan extremes.
- **require-baseline-adjusted-significance.md** — For trend detection vs. a control parameter, require both raw significance and baseline-adjusted significance; raw comparison can overstate trend significance by ~2× when a β=0 offset exists.
- **require-matrix-sanity-checks.md** — For matrix construction + spectral analysis goals, add an explicit pre-analysis sanity check step: verify the matrix is Hermitian and non-degenerate (no all-zero eigenvalue sets) and the spacing distribution is roughly bell-shaped before running statistics. Without this, explorers proceed through full analysis on degenerate matrices (e.g., odd Dirichlet characters → antisymmetric → zero matrix), producing meaningless outputs that look like findings.
- **ask-forward-looking-literature-questions.md** — Always include "does this combination/result appear in the literature?" even when confident about the state of the art. Literature in active fields can advance faster than the library; a single forward-looking question is the cheapest way to discover this. Yang-mills s002 E001: found CNS 2025 (area law at β < 1/24, doubling SZZ threshold) as the most important finding, from exactly this type of question.
- **adversarial-synthesis-goal-structure.md** — NEW (SED s003-meta-004): 4-part output structure for finishing-strategy adversarial synthesis goals: (1) adversarial review per claim with verdict+novelty rating, (2) prior art search for the conclusion, (3) grand synthesis, (4) consolidated claims table. Pre-load all claims with evidence summaries, prior art candidates, and strongest objections directly into GOAL.md. Long goals (200-300 lines) are unavoidable and worth it. Tier 5 requires concrete falsifiable paths per claim, not just literature checks. *Updated (thermal-time s003-meta-002): prior art discovery can be the most important outcome — it provides an objective external anchor for novelty ratings.*
- **scope-computations-to-minimum-diagnostic.md** — For computationally intensive explorations, specify the MINIMUM computation needed. "Compute ratio at K=N=500,1000,2000,3000,5000 and extrapolate" is better than "validate with 5000 zeros and larger GUE" — the explorer will run all available approaches instead of the one diagnostic test. Distinct from one-task-per-exploration (which is about task types); this is about scoping within a single computational task.
- **characterize-maximizers-not-just-bounds.md** — **[NEW, YM s003-meta-002]** When a proof strategy involves bounding λ_max, frame goals as "characterize the set of maximizers" — the structural characterization (e.g., "F=4d ↔ pure gauge") reveals geometry and suggests proof routes. Distinct from allow-explorer-synthesis (question structure, not conclusion openness).
- **verify-goal-claims-before-delegating.md** — **[NEW, YM s003-meta-005/s003-007]** Before writing GOAL.md citing a prior result (e.g., "M(Q) ≼ M(I) confirmed by E004"), verify the cited claim actually matches what the exploration found. Two-level fix: (1) strategizer re-reads REPORT.md before citing; (2) GOAL.md includes "first compute X for 3-5 random examples — if it fails, pivot." Prevented two wasted explorations in yang-mills s003. Distinct from preload-context (providing findings) and include-trivial-control-checks (validating code).
- **include-predecessor-comparison-for-reformulations.md** — **[NEW, barandes-stochastic E004]** When evaluating a reformulation's value, provide a successful predecessor (e.g., amplituhedron) as structured comparison framework. Produces multi-dimensional evaluation table far clearer than standalone philosophical assessment. Variants: "one paragraph answer to the key question" (forces concrete honest synthesis) and "rank papers by importance" (for paper cluster assessments). Distinct from comparison-exploration-pattern (which is about comparing TWO approaches; this is about providing a BENCHMARK for evaluation).
- **adversarial-proof-review-structure.md** — **[NEW barandes-stochastic E005, UPDATED yang-mills-conjecture E007]** For adversarial reviews of a specific proof, use a 4+1-step structure: (1) trace proof step by step, (2) identify key assumption, (3) check if derived or imported, (4) check prior art, **(5) check whether the proved statement actually implies the claimed conclusion** [yang-mills-conjecture E007 — cube-face reduction proof was internally correct but only proved staggered mode bound, not full operator bound]. Variants: always require steelman when adversarial likely negative; ask "does the paper acknowledge its own limitations?"; **for matrix inequality proofs, verify reduction from operator bound to quadratic form bound** (the two are equivalent only when the subspace covers all top eigenvectors). Distinct from adversarial-synthesis-goal-structure (multi-claim review) and adversarial-check-between-phases (timing).
