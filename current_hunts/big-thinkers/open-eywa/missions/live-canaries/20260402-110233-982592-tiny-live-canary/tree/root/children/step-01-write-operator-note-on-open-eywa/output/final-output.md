# Operator Note on Open-Eywa

Open-Eywa is a distributed task orchestration system designed to manage complex workflows by breaking them down into discrete, manageable units called nodes. Each node represents a durable unit of work that encapsulates a specific task or step within a larger mission.

## Why the Node is the Durable Unit of Work
The node is the durable unit of work because it provides a clear boundary for task execution, state management, and fault tolerance. By isolating work into nodes, Open-Eywa ensures that progress is saved incrementally, enabling reliable recovery and continuation even if interruptions occur. This design simplifies tracking, debugging, and coordination across distributed systems.

## Key Mission Files for Human Operators
To understand the mission state and progress, human operators should first inspect the following files:

1. `input/context.md` - Provides the overall mission context, goals, and current state.
2. `output/final-output.md` - Contains the final results or outputs generated by the node.
3. `output/state.md` - Shows the current internal state or progress of the node.
4. `input/parent-instructions.md` - Details the instructions or task assigned to the node.

Reviewing these files gives a comprehensive view of what the mission is about, what the node is working on, and what has been accomplished so far.
