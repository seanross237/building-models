# Triad Dynamics Plan

## Core Judgment

No prize-facing Tao-adjacent mainline currently exists. Under this domain lens,
the only honest live frontier is to turn the mechanistic picture

- exact shell-crossing helical triads,
- constructive quadrature-type transfer sectors,
- recursive spill into companion/spectator channels,

into an intrinsic theorem on exact Navier-Stokes solutions.

I do **not** think another packetization, another explicit five-mode toy, or
another role-labeled near-circuit witness is a serious path to the Millennium
Prize. Those have already run into canonicity, exact closure, or
same-currency-realization failure. If this Tao-adjacent branch survives at all,
it survives as one of two things:

1. an intrinsic shell-interface recursive-spill theorem that blocks any Tao-like
delayed gate in exact NS, or
2. a sharp no-go theorem showing that no finite or finitely generated exact
helical carrier can realize the needed mechanism without arbitrary truncation.

If neither can be made intrinsic, then this whole branch should be treated as
mechanism mapping, not as a prize path.

## Main Theorem Target

**Recursive Spill Firewall on Exact Shell-Crossing Triads**

Prove an intrinsic theorem of the following shape:

For an exact NS solution, fix a dyadic shell interface and the exact
shell-crossing helical triads across that interface, weighted by their true
interaction coefficients and signed transfer. If a time window carries
sustained positive forward transfer in a coherent constructive sector, then
recursive closure at depth 2 already forces a quantitatively non-negligible
companion/spectator spill budget in the same exact currency. Therefore no
Tao-like delayed-window mechanism with isolated load channel and small spill can
persist on that interface.

The real content is not the slogan "spill happens." The target is a lower bound
in an intrinsic interface currency that does **not** require:

- packet boundaries,
- post hoc desired-edge labeling,
- role annotation imported from Tao,
- or a frozen ad hoc witness ledger.

## Fallback Theorem Target

**No Finite Tao-Carrier Theorem for Exact Helical Shell-Crossing Supports**

Prove that any finitely generated exact helical support family that tries to
realize a Tao-like one-bridge activation/return mechanism must fail in one of
the following exact ways:

- arithmetic incompatibility of the required triad identities,
- recursive support emission within at most two closure layers,
- or dependence on an arbitrary truncation convention rather than a genuine
exact closed subsystem.

This would upgrade the present negative evidence into a theorem-facing no-go:
exact NS does not admit a finite reduced carrier for the Tao circuit mechanism
in the shell-crossing helical setting.

## Route Structure

### Route 1. Intrinsic shell-interface spill theorem

This is the only route here that could plausibly become prize-facing.

Work on the exact oriented helical triad tensor restricted to one shell
interface. Build the theorem around interface-level constructive transfer and
recursively forced spill, not around packets. The objective is to prove that any
apparently coherent forward-transfer sector generates unavoidable spill fast
enough to kill delayed-threshold isolation.

### Route 2. Finite-carrier no-go theorem

If Route 1 cannot yet close globally, push the current negative exact-support
evidence into a clean theorem: no finite or finitely generated recursively
closed shell-crossing carrier can host the Tao mechanism in exact NS.

This is not a prize proof, but it is real theorem progress and would sharply
clarify what exact reduced-dynamics fantasies are off the table.

### Route 3. Frozen-ledger packet theorems as bounded support work only

The template-defect and leakage-budget results on the canonical one-bridge
ledger are worth either proving sharply or killing quickly. But they are not my
mainline, because even a successful result there remains a packet-family theorem
on one frozen ledger, not an intrinsic NS control law.

Use them only to calibrate what a true spill theorem would need to dominate.
Do not mistake them for the prize route.

### Route 4. Same-currency explicit realization search only as a kill test

I would only fund more explicit realization work if it is framed as a bounded
kill test for Route 2. If exact same-currency realization keeps requiring
support-dependent repair, then that is evidence for no finite-carrier
realizability, not a reason to keep engineering examples.

## First 2-3 Concrete Theorem Steps

### Step 1. Freeze an intrinsic shell-crossing interface object

Define a shell-interface ledger directly from the exact oriented helical triad
interaction tensor:

- quotient by conjugation and harmless relabeling,
- retain true coefficient/sign data,
- separate constructive forward-transfer weight from total active weight,
- define companion/spectator spill purely by recursive closure relative to the
same interface, not by Tao-role annotation.

This step succeeds only if the object is invariant under harmless refinement and
does not become tautological on one triad or arbitrarily weak on the first
nontrivial two-input cluster.

### Step 2. Prove a depth-2 recursive spill lower bound

Start from a coherent constructive sector on the shell interface and enumerate
the exact first and second recursive emission layers forced by closure. The
theorem target is a lower bound of the form

`Spill_depth<=2 >= c * ForwardConstructiveTransfer`

for some intrinsic spill functional on the interface, perhaps after excluding an
explicitly classified degenerate family.

This is the real load-bearing step. If the bound only appears after choosing
target edges or spectator classes by hand, the route has failed.

### Step 3. Convert the instantaneous bound into a delayed-window obstruction

Integrate Step 2 over a time window and prove that any sustained constructive
sector necessarily accumulates spill on the same time scale as the putative
load-channel growth. That should rule out the Tao-style delayed trigger logic in
exact NS at the shell-interface level.

If the time-integrated theorem is too hard immediately, prove the weaker but
still useful statement that any such window forces either

- order-one spill in the same window, or
- breakdown of constructive coherence before a delayed gate can form.

## Hard Stop Conditions

- Stop if the theorem object again requires packet boundaries, desired-channel
annotation, or imported Tao roles.
- Stop if the shell-interface observable is forced on one triad or can be made
arbitrarily small on the first honest multi-input exact cluster, as happened for
the weaker coherence ratios.
- Stop if recursive spill cannot be lower-bounded without a frozen witness sheet
or support-specific bookkeeping.
- Stop if further progress depends on ad hoc explicit-support searches rather
than a theorem on an intrinsic object.
- Stop calling this a prize-facing route if the best result remains "one packet
family on one frozen ledger is hostile/friendly."

## What Counts as Genuine Progress

- An intrinsic recursive-spill theorem on exact shell-crossing helical triads.
- A theorem that any coherent constructive transfer sector dies by forced spill
before a Tao-like delayed window can isolate.
- A sharp no-go theorem excluding finite or finitely generated exact helical
carriers for the Tao mechanism.
- A canonical extraction rule from an exact solution to the shell-interface
object that survives symmetries, refinement, and closure.

By contrast, better witness engineering, nicer packet arithmetic, or another
five-mode explicit candidate does **not** count as serious progress unless it is
used to prove one of the theorem statements above.

## What I Refuse To Spend Time On Because The Status File Closes It

- Clustered packet models with post hoc desired-channel bookkeeping.
- Hypergraph expansion or anti-circuit reformulations that still need non-
intrinsic role-community annotation.
- Tiny finite-support phase-locking searches already killed by recursive spill:
one-triad seeds, two-triad shared-mode seeds, and three-triad single-repeated
orbit seeds.
- Sign-defect functionals as a primary route while the desired-triad set remains
non-canonical.
- Generic "maybe Tao misses some NS structure" exploration without a theorem
object.
- Open-ended explicit singleton bridge hunting, except where it directly serves
the finite-carrier no-go theorem.

## Bottom-Line Recommendation

Treat this lens as a theorem-object engineering problem, not as a direct proof
program. The honest main attempt is:

1. define the intrinsic shell-crossing spill object,
2. prove a depth-2 recursive spill lower bound,
3. turn it into a delayed-window obstruction.

If Step 1 or Step 2 fails intrinsically, pivot immediately to the finite-carrier
no-go theorem and stop pretending a Tao-adjacent prize mainline currently
exists.
