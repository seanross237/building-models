# Critique: Triad Mechanism Dependencies

## Strongest objections

### 1. The draft promotes a mechanism-preserving extraction that the status file does not justify

The proposed main contradiction chain

`minimal blowup element => canonical extraction of one exact one-bridge object => canonical obstruction theorem => contradiction`

is not just unproved. From the current status file, it is upstream of the main
known obstruction.

The exact Tao-adjacent record does **not** currently say "we need a cleaner
definition." It says:

- the one-bridge ledger can be frozen on one witness-local authority sheet,
- but constructive continuation stops at
  `exact non-isolability / arbitrary-truncation requirement`,
- and then stops again at
  `source-basis / no-canonical-finite-reduction`.

That is a much harsher failure mode than the draft acknowledges. The missing
piece is not merely a canonical labeling convention. The missing piece is a
same-currency exact reduction protocol that still carries the actual nonlinear
mechanism.

As written, the draft smuggles in the conclusion that a minimal blowup element
admits a canonically extractable one-bridge mechanism carrier. That is exactly
what has not been earned.

### 2. The canonical object layer is not yet shown to preserve the load-bearing exact dynamics

The mechanism story the plan wants to control is Tao-like delayed-threshold
transfer in exact Navier-Stokes. But the status file now says the faithful
exact dynamics do not stay inside tiny clean subsystems:

- singleton and other tiny finite exact supports fail under triad closure;
- one-triad, two-triad, and three-triad phase carriers all spill under honest
  recursive closure;
- the first branching module admits a compatible quadrature-locked sector;
- the first recursive emission layer reinforces coherent quadrature structure
  rather than immediately frustrating it.

That means the dangerous and the stabilizing parts of the mechanism both live
in recursive spill, companion generation, conjugate completion, and spectator
burden. A canonical one-bridge object that forgets or quotients those away is
not preserving the mechanism. It is changing it.

The draft currently treats "harmless refinement" as if it were a bookkeeping
issue. In this domain it is not. Refinement changes which interactions are
counted as desired, forced, emitted, or spectator, and those distinctions are
exactly where leakage, template-defect, phase coherence, and sign-defect are
measured.

### 3. The plan hides decisive modeling choices under the word "canonical"

From the exact mechanism viewpoint, the following are not harmless presentation
choices:

- finite-support truncation or finite-closure convention,
- source basis / representative wavevector family,
- helicity tuple,
- desired-channel versus spectator classification,
- shell partition and interface definition,
- role labeling on the packet ledger,
- conjugate completion policy,
- comparison currency for coefficients and transfers.

The status file already says these choices are where previous attempts failed.
So Phase 0 cannot simply "freeze the object layer" and proceed. It has to prove
that the promoted object is invariant under admissible changes in exactly these
choices, or else admit that the object is still a model, not an exact-NS
carrier.

### 4. The draft is mechanism-upside-down relative to the current frontier

The status memo does not rank the critical-element route as the most mature
theorem path. It explicitly says the best near-term deliverables are:

- a sharp packet-family theorem for template-defect on the frozen one-bridge
  ledger,
- a sharp leakage firewall lemma on the same frozen ledger if the transfer step
  can be proved,
- and, as the cleaner intrinsic moonshot, a recursive closure-versus-locking
  theorem on exact shell-crossing triads.

By contrast, the critical-element bridge is described as conceptually
attractive but blocked earlier than before, at canonical-ledger extraction.

From a triad-mechanism perspective that ranking makes sense. You do not yet
know what exact object class actually preserves the nonlinear mechanism, so it
is premature to organize the whole plan around a minimal-element contradiction.

### 5. The route risks proving the wrong theorem on the wrong object

Suppose the draft successfully proves some obstruction theorem on an extracted
one-bridge class. Without a prior mechanism-preservation statement, that still
does not tell you the theorem hits exact Navier-Stokes blowup dynamics rather
than an artifact of:

- discarded recursive emissions,
- discarded shell-interface transfer,
- discarded heterochiral companion channels,
- or witness-specific desired-edge bookkeeping.

This is the main domain objection. In these branches, exact NS does not fail by
simple forward energy flux alone. It fails or survives through tightly coupled
helical triad geometry, sign structure, and recursive spill. Any theorem that
suppresses those may be exact on paper but irrelevant to the real mechanism.

## Missing assumptions or prerequisites

The draft needs to state, and then justify, assumptions it is currently
treating as implicit:

1. Why should a minimal blowup element exhibit a Tao-like one-bridge mechanism
   at all, rather than a more distributed shell-crossing organization?
2. Why should concentration-window extraction preserve the relative phase,
   helicity, and transfer geometry needed for the mechanism story?
3. Why is one role-labeled ledger the right invariant object, given the
   existing `source-basis / no-canonical-finite-reduction` obstruction?
4. What exact information is allowed to be discarded while still claiming
   mechanism fidelity: external emissions, spectator load, recursive closure,
   or only literal zero-contribution channels?
5. What is the theorem-level criterion for saying a refinement or relabeling is
   "harmless" when leakage and desired-channel counts can move under those
   operations?
6. If the intrinsic phase route is no longer expected to fail on the first
   branch, what mechanism feature is the one-bridge object supposed to retain
   that shell-crossing triads do not retain more honestly?
7. What exact PDE principle links an extracted ledger to repeated near-closure
   windows instead of a one-off witness pattern?

Without answers to those, the main route is still leaning on hidden model
selection.

## Concrete revisions

### 1. Demote the critical-element contradiction chain from main route to conditional hybrid

Do not make
`minimal element => canonical one-bridge extraction`
the default main route.

Rewrite it as a conditional branch that activates only after a
mechanism-preservation lemma is earned.

That lemma must show that the extracted object preserves, in one fixed
comparison currency:

- exact active triad coefficients,
- helicity/sign structure,
- recursive closure obligations,
- shell-crossing companion emissions,
- and the relevant phase or leakage observables.

If that cannot be proved, the critical-element bridge should remain parked.

### 2. Split Phase 0 into substrate selection and mechanism-preservation

Current Phase 0 is too soft. Replace it with:

- `Phase 0A: choose the exact substrate`
  Either the frozen one-bridge ledger with an explicit realization/closure
  convention, or an intrinsic shell-crossing triad interface object.
- `Phase 0B: prove mechanism-preservation`
  Show that template-defect, leakage, or phase-defect observables survive
  admissible refinement, conjugate completion, recursive closure, and source
  representation changes.

If 0B fails, stop. Do not proceed to PDE contradiction language.

### 3. Put exact shell-crossing dynamics back into the plan explicitly

The draft overfocuses on one-bridge extraction and underweights the status-file
lesson that exact phase organization survives the first branch and first
recursive emission layer.

A domain-faithful revision should explicitly ask:

- can recursive spill or spectator burden be turned into a theorem on exact
  shell-crossing triads?
- can any would-be delayed Tao-like window be ruled out only after including
  the forced companion families that exact NS generates?

That is closer to the exact dynamics than another role-labeled packet object.

### 4. Re-rank the theorem programs

The order should be:

1. exact packet-family theorems on the already frozen one-bridge ledger,
   because those are the only Tao-adjacent results closest to theorem form;
2. intrinsic shell-crossing recursive-spill / closure-versus-locking theorem,
   because it is more mechanism-faithful than packet annotation;
3. only then, if an actual preservation/extraction lemma appears, the
   critical-element contradiction bridge.

That ordering matches both the status memo and the mechanism picture.

### 5. Add a hard stop against hidden finite reduction

Insert an explicit stop rule:

"If the promoted object requires arbitrary finite truncation, witness-local
support freezing, or a non-intrinsic desired-channel partition to state the
theorem, then the route has not preserved exact Navier-Stokes dynamics and must
be demoted to mechanism mapping."

That hard stop is needed because the present draft is too willing to treat
canonicalization as a future paperwork task.

## Salvageability

The plan is salvageable without a full rewrite, but not as currently ordered.

It becomes defensible if:

- the critical-element contradiction is demoted to a conditional late-stage
  bridge,
- mechanism-preservation is promoted ahead of canonical extraction,
- the one-bridge object is no longer assumed to be the faithful carrier of the
  exact dynamics by default,
- and the intrinsic shell-crossing route is treated as the main exact-dynamics
  check rather than a side idea.

If those revisions are not made, the plan will keep talking as though the
problem is "find the right canonical object," when the status file already says
the deeper problem is "find an object that does not discard the dynamics doing
the real work."
