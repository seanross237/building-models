## Lesson

When a Phase 0 comparison narrows a mission to one preferred object pair only **narrowly**, the next exploration should not reopen object design.

Freeze the selected pair, keep its interface/filter conventions fixed, and move immediately to the smallest faithful exact support that still carries the load-bearing variable. The first adversarial question is no longer "is there another candidate object?" It is "does the frozen object stay nontrivial there, or does it collapse into a tautological or too-weak summary?"

In practice this smallest-support audit should be two-step:

1. On the smallest support where the relevant phase variable exists, ask whether the burden observable is forced or tautological.
2. On the first support large enough for genuine cancellation, ask whether the burden can already become arbitrarily weak while the target event stays alive.

## Evidence

- **exact-ns-phase-locking-firewall exploration-001** — Candidate family 1 (packet-sign proxies) inherited the already-closed canonicity failures, and candidate family 2 (raw phase concentration) survived only as backup because it was too weakly tied to constructive delayed transfer.
- The surviving pair `(C_ell, E_ell^transfer)` passed only narrowly. Once that happened, the honest next step was not another round of object shopping. It was the smallest-support audit asking whether `C_ell` remains a genuine phase-coupled obstruction variable or degenerates into a dressed-up cancellation ratio.
- **exact-ns-phase-locking-firewall exploration-002** — The smallest-support audit split cleanly into the two decisive screens. On one exact active transfer term, `C_ell` was identically `+/-1`; on the first exact two-input cluster, `C_ell` could be driven arbitrarily close to `0` while receiver gain stayed positive. Both failures were model-level negatives on the frozen pair, not invitations to reopen object design.
- The same meta note identified the main drift risk: reopening shell choices, interface conventions, or localized physical-space variants before the minimal-coupling question is settled would turn a model-level audit back into a definition campaign.

## Goal Pattern

1. State the frozen preferred pair verbatim.
2. Hold fixed the interface/filter/witness conventions that made the pair admissible.
3. Ask for the smallest exact support on which the load-bearing variable is still genuinely present.
4. Make the first attack in two stages: first the smallest-support tautology screen, then the first nontrivial-support weakness screen.
5. Treat failures at that stage as model-level negatives on the frozen pair, not as permission to reopen candidate-object design immediately.
6. Only return to alternative observables or localization layers if the minimal-support audit identifies a specific structural ingredient that the frozen pair lacks.

## Why This Matters

After a narrow pass, definition drift is the main waste mode. The mission can look active while it quietly replaces the selected object with nearby variants. Freezing the pair forces the next exploration to answer the theorem-facing question that the definition gate actually created, and the two-stage minimal-support screen gives a cheap early kill if the burden collapses immediately.
