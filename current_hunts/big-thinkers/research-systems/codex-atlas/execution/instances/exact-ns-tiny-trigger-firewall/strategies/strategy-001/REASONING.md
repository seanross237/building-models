## Iteration 1 decision

The strategy begins from a narrower mission than the prior exact-circuit line. The live question is not whether exact NS hosts Tao's whole near-closed circuit, but whether exact NS can host one energetically tiny delayed trigger that remains hidden until it becomes dynamically decisive.

I queried the receptionist before deciding the first exploration, as required. The returned material was decisive:

- the right Tao-side source is still the delayed-transfer five-mode circuit, with `c` / `X3,n` as the trigger variable;
- the best exact-NS Phase 0 object already lives in the prior helical definition file, but it is a five-role hypergraph rather than a trigger-only scalar;
- the sharp singleton exact object was already killed structurally by triad-closure non-embeddability;
- the packetized backup already has a known canonization problem;
- the right bookkeeping remains coefficient-weighted amplitude-level leakage, with pressure/Leray treated as the enforcement mechanism rather than as a separate route.

So the first exploration should not pretend there is a fresh large design space. Its job is to answer a single sharp question:

```text
Can a Tao-style tiny delayed trigger be defined as one preferred exact-NS object that is genuinely
narrower than the previously killed five-role near-closed circuit, or does every honest trigger
definition either collapse back to that same object or require a non-canonical packet backup?
```

That makes exploration 001 a definition-gate exploration with an immediate backup canonization test. If the preferred trigger object turns out to be just the prior five-role exact hypergraph with `a3` highlighted, then exploration 002 should become a fast static-support audit using the earlier singleton non-embeddability result as the fail-fast checkpoint. If exploration 001 instead shows that even the trigger notion itself cannot be made theorem-facing without packet freedom, the strategy can stop immediately with a definition-level failure.

## Iteration 1 result

Exploration 001 closed the strategy early.

The preferred exact trigger object can be stated sharply, but only in the same amplitude-level helical language already used for the prior near-closed Tao circuit:

- finite sign-closed support `S(K,σ)`,
- five distinguished amplitudes `a1,...,a5`,
- exact desired/internal/external bookkeeping,
- trigger-centered observables attached to role `a3`.

That means the trigger object is not genuinely narrower than the earlier five-role object. It is the same object with extra bookkeeping on:

- trigger smallness `|a3| / |a1|`,
- amplifier visibility,
- conduit/handoff visibility,
- leakage on roles `3,4,5`,
- back-reaction onto role `1`.

Because the desired monomial core still contains

```text
a1^2 -> a2,
a1^2 -> a3,
a2 a3 -> a3,
```

the preferred object inherits the previously established singleton non-embeddability result immediately. So there is no surviving fresh Phase 1 target.

The only backup is packetized. It fails the canonization gate for the same reasons already recorded in the predecessor line:

- no fixed support convention,
- no fixed packet projection/coordinatization,
- no fixed leakage/back-reaction ledger,
- no fixed restoration rule back to exact NS language.

So the correct early-stop label for this strategy is:

```text
model-level negative candidate
```

with a secondary definition-level failure of the packet backup.
