# Genuine Second-Budget Shared-Mode Seeds Must Add New Connected Support Beyond The First-Budget Pseudo-Support

Status: `INFERRED` second-budget exclusion rule using the frozen Step-2
pseudo-survivor verdict

A two-triad shared-mode drawing is not a new second-budget seed just because
it displays two triad equations.
If the second triad is literally the first triad again, or only adds extra
helical sign sectors on the same one-triad wavevector picture, the result is
only a duplicate or inert decoration.

The same exclusion applies when both triads stay inside the old mirror-complete
one-triad support

`{ plus/minus a, plus/minus b, plus/minus (a+b) }`.

Canonical redrawings such as
`(a, -a-b, -b)`
only repackage the six-wavevector pseudo-survivor already killed by the
Step-2 admissible-enlargement audit.
They do not create a genuinely new second-rung support.

At this rung the catalog also excludes disconnected add-ons, because the
budget being classified is the connected shared-mode two-triad budget.
So a genuine second-budget seed must stay connected through the shared orbit
and must add at least one new wavevector orbit beyond the old first-budget
pseudo-support.
