# Admissible Enlargement Of The Generic Five-Orbit Shared-Mode Fan Shows It Is Only An Over-Pruned Second-Budget Pseudo-Survivor

Status: `INFERRED` artifact-check verdict with `VERIFIED` admissible
enlargement policy

The generic second-budget shared-mode fan

`S_gen = { plus/minus a, plus/minus b, plus/minus c, plus/minus (a+b), plus/minus (a+c) }`

is not a genuine finite exact support on this branch.
It only looks tidy if the first closure-forced cross-triad target orbit is
suppressed after the Step-3 closure audit.

The admissible artifact check is to restore one such forced orbit, for example

`plus/minus (a+b+c)`,

while keeping the same support semantics, helical basis,
conjugate-completion rule, and recursive-closure convention, and then
recompute closure from scratch.

That enlargement does not stabilize the support.
On the enlarged ledger, active ordered pairs such as
`(a+b+c, b)` and `(a+b+c, c)` immediately force

`a+2b+c`

and

`a+b+2c`,

which lie outside the original two-triad shared-mode budget.

So the generic five-orbit fan is only an over-pruned pseudo-survivor.
Restoring its first forced orbit reproduces the spillover rather than
producing a second-budget exact fixed point.
