# Freeze The Role Ledger Before Candidate Formulas When Rebuilding An Exact Object From A Mechanism Memo

When a branch wants to turn an inherited mechanism memo into a new exact
object, freeze the role ledger first: stage order, trigger logic, amplitude
hierarchy, time-scale separation, and support semantics.

Do not start from candidate formulas. Formula-first exploration invites the
branch to confuse packaging choices with the mechanism it is actually trying to
preserve.

The same prompt should state explicitly which later controls are inherited and
which are branch policy. In particular, admissible leakage is often something
the branch must declare for itself rather than a theorem fact imported from the
original mechanism memo.

The same note now also says that once the role ledger is frozen, each promoted
candidate should still be forced to name one exact packet class, one finite
window, one threshold language, one canonical sign / amplitude / phase sheet,
and one downstream gate on that same ledger before it counts as an object
rather than a slogan.

That means the interaction template and the downstream gate should be written
in the same packet and threshold language. Otherwise the branch is still
moving between incompatible ledgers even if the role story sounds sharp.

For exact-support searches that are not intrinsically packet-level, the prompt
should also freeze the support object, recursive closure rule, spectator
policy, and admissible enlargement rule together. If those items are left
implicit or are declared piecemeal, later negative claims can be gamed by
pruning closure-forced modes or by silently switching from an intrinsic helical
support ledger back to packet-role encoding.

If a branch inherits only classwise screening totals, treat those totals as a
closure-burden ledger rather than as a finite exact closed packet.
A phrase such as
`one primary mode per role plus closure-forced companions`
is not a finite support description by itself.
The prompt must still freeze one explicit support-level closure convention and
companion list, or later steps can smuggle a reduced exact system by post hoc
truncation while pretending they are only unpacking inherited data.

Packet templates may still be useful as seeds or fixtures. But unless the
branch writes an exact invariant packet representation explicitly, packet
language should remain template data rather than the primary search object.

It also says to keep enforcement devices such as exact Leray / pressure
projection as mechanism notes unless they themselves define a distinct exact
object. Otherwise the step confuses enforcement with object definition.

Finally, if a behavior-based candidate survives that screen, write it in exact
projected channel data rather than importing Tao's averaged coefficients
directly. Once the branch switches to averaged-coefficient behavior language,
it has left the exact-NS object screen.
