# When Prior-Art Overlap Is Secondary, Let The Earliest Honest Obstruction Label Win

When a branch still has one genuine Navier-Stokes-specific discriminator, do
not overstate a prior-art screen by calling the result pure repackaging.

But do not let that surviving discriminator keep the branch alive either.

If setup or calibration has already failed to produce one frozen baseline,
one admissible gain currency, and one theorem-facing one-sided improvement,
those earlier obstructions should stay primary.

In that situation, prior-art overlap is still worth recording, but only as a
secondary kill label or supporting context:
the branch is prior-art-adjacent, not novelty-cleared, yet the honest stop
reason is still the earliest obstruction that already kills it.

The same ordering rule applies when a proxy bridge burden has already become
theorem-scale. Even if the closest comparator overlap is severe, the primary
verdict should stay `theorem-floor obstruction` when admission to the frozen
theorem floor has already failed earlier and more cleanly than the prior-art
screen.

This keeps verdicts from drifting in either direction:
not a false `pure repackaging` label when one narrow packet-level
discriminator survives, and not a false survivor label when that
discriminator never cashes out as estimate-level gain.
