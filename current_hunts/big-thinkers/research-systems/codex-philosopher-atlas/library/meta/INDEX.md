# Meta Library Index

Methodology, goal-design patterns, and system behavior lessons for Philosopher Atlas.

## Topics

- `exploration-goal-design/`: reusable prompt-design lessons for formula-first
  obstruction reports, killed-mode accounting, notation normalization, source
  scoping, prior-art packet design, earlier fixation of the operative control
  representation, anti-overfit localization design, architecture-choice
  prompts that distinguish live from already-closed rival branches, and
  diffusion-scale Eulerian tests for candidates whose only live content is
  dynamic persistence, plus bounded procedural novelty claims for steps that
  only freeze an auditable candidate table, and role-ledger-first exact-object
  reconstruction that separates inherited mechanism from branch-level policy
  choices such as leakage rules. The same topic now also records a sharper
  exact-object candidate rule: after the role ledger is frozen, each promoted
  candidate must still name one packet class, one finite window, one threshold
  language, one canonical packet sheet, and one downstream gate on that same
  ledger, while enforcement devices such as exact projection stay mechanism
  notes unless they define a distinct object. It now also records an
  exact-support-search refinement: freeze the support object, recursive
  closure, spectator policy, admissible enlargement rule, and the line between
  packet templates and intrinsic search identity together before trusting any
  negative class verdict. It also records that hard admissibility sheets only
  become invariant once class partitions and finite-window normalizations are
  frozen. It now also records that inherited classwise screening totals are
  only a closure-burden ledger until one explicit support-level closure
  convention and companion list are frozen. It now also records an intrinsic
  phase-object rule: promote the
  orbit-level exact interaction ledger before any scalar coherence score,
  treat aggregations as diagnostics unless the aggregation law is
  symmetry-forced, and reject raw formulas that name desired routes or
  spectator sets by hand. It now also records a
  downstream-freeze handoff rule: each promoted object should leave with one
  exact next question, one invariant gate, one smallest carried witness or
  stress set, and one named same-currency debt, while audited calibration
  dossiers stay theorem setup rather than theorem delivery until that debt is
  discharged. It now also records a witness-local bootstrap rule: if a step
  freezes one static witness before closure or dynamics are settled, the memo
  should also freeze one hard scorecard, one diagnostics-only bucket, one
  explicit no-rescue line, and one no-overclaim statement saying that the
  dossier is still static rather than dynamic success. It now also says that a
  clean static-screen pass still does not entitle the next step to assume a
  finite exact ODE or downstream dynamics before one closed support ledger is
  actually frozen.
  It now also records a delayed-transfer setup rule: if that success
  condition matters downstream, freeze the transfer partition, exact
  closed-support inflow observable, intrinsic clocks, spectator burden, and
  robustness rule before support search or trajectory scoring begins, so
  later evidence cannot be created by metric drift.
  It now also records a seed-screen rule for one-triad support searches:
  separate degenerate repeated-wavevector and collinear relations from honest
  seeds before closure work, so later negatives do not mix seed-definition
  failure with true closure failure.
  It now also records a higher-rung support-search rule: collapse candidate
  drawings by exact support identity after the frozen equivalences before
  closure, so re-centered copies and earlier pseudo-support decorations do
  not get miscounted as new ladder-level seeds, and then keep the surviving
  canonical families separate through closure so different first-spill
  mechanisms stay visible.
  It also records a sharper prior-art packet rule: naming one literature neighborhood is not enough;
  later steps should demand one nearest comparator with one exact shared
  structure, one exact mathematical delta, and one same-currency cash-out or
  explicit transfer lemma before identity work proceeds.
- `obstruction-screening/`: reusable rules for deciding whether a candidate
  actually acts on the live obstruction instead of merely restating it, with
  an added geometry-specific screen against static or Lamb-vector-only claims,
  plus a localization-exactness screen for rewrite audits, an explicit
  cutoff-and-ledger question for estimate-level gains, a hidden-assumption
  screen for comparator audits, an architecture-drift screen for candidates
  that only survive by becoming a different branch, a stop rule for branches
  whose full audited candidate set is only dynamically weak, a shortlist rule
  for branches whose baseline already looks negative, a continue-or-kill
  rule for branches already frozen to one bad term and one gain currency, and
  host-space audit rules distinguishing genuine framework-plus-endpoint
  packages from mere compactness prestige, scale-invariant appearance, or
  silent framework drift inside an inherited architecture. The same topic also
  records that an organized burden ledger is still not a live setup until one
  temporal architecture and one bridge are actually delivered, and that
  endpoint-first branches should separate actual theorem endpoints from
  mechanism language at the first gate, freeze admission rules on the exact
  endpoint object, freeze one-floor windows as one primary endpoint plus at
  most one stricter reserve together with solution class and localization, and
  treat gate clearance as narrower than tensor admission.
  It now also records that once one ledger is frozen, prior-art overlap should
  collapse family labels before packaging work begins. It also records a sharper
  threshold rule: once some modes are already killed by the test structure,
  only a non-cosmetic improvement on the same live inequality after later debt
  is restored counts as progress. It also records a support-stability rule:
  when object identity is central, separate true symmetries, fixed
  canonicalization, and genuinely different object changes before calling a
  representation move cosmetic. The same topic now also records a scope-lock
  rule for early setup gates: do not count the lock as real until both the
  baseline cash-out and the minimum gain threshold or admissible gain notion
  are frozen in the same burden currency that the later coercivity test must
  beat. It also records a transfer rule: if a claim is stated in a different
  burden currency, demand an explicit transfer lemma back to the same
  theorem-facing burden and reject debt repaid in some different ledger as
  narrative bookkeeping. It also records a Tao-screen rule for exact-but-
  noncoercive branches: separate exact structure from bookkeeping first, and
  use Tao sensitivity only as an admission filter unless it cashes out as a
  real gain on the frozen bad term. It now also records that for
  reformulation-first audits the endpoint-exchange question is often the
  earliest live gate, so cleaner native-variable language without an explicit
  same-currency return should be labeled exchange failure before later
  anti-repackaging or overlap screens. It now also records a label-order rule
  for prior-art screens: when one narrow
  discriminator survives but setup or calibration already fails, keep
  prior-art overlap secondary and let the earliest honest obstruction label
  stay primary. It now also records a stochastic-transport setup rule: one
  burden plus one comparator neighborhood is still only menu trimming until
  the branch freezes one theorem-facing observable and one sentence of
  leverage, with `local concentration exclusion first` as the sharper default
  filter before continuation language. The same topic now also records the
  solution-class admission rule for that branch family: once one theorem-facing
  weak-solution floor is frozen, kill a new branch unless an explicit
  admissibility bridge is on disk, and do not mistake endpoint-ordering
  success for branch admission. It now also records an anti-repackaging
  rule-writing lesson: state the failure buckets in the branch's own burden
  currency so expectation-only, filtration-only, localization-only, or
  parasitic-framework-only gains can be rejected mechanically. The same topic
  now also records that classification should be made on the restored full
  theorem-facing burden, and that any `ambiguous` middle bucket should be kept
  only for one exact missing transfer step. It also records a slate-discipline
  rule for frozen shortlists: do not reopen branch-adjacent families to make a
  menu look richer, keep at least one reserve comparator, and use active slots
  only for candidates that still name a nontrivial route back to the same
  frozen burden. It also records a sharper rewrite-audit distinction: derive
  the native exact equation first, charge debt at the first localization
  step, and separate candidates that act on the frozen burden natively from
  those that only touch it after reconstruction. It now also records a
  theorem-floor admission refinement for proxy-based branches: run the bridge
  check before package selection, count hidden Eulerian matching,
  relocalization, and imported stronger control as bridge debt rather than
  later cleanup, and keep `theorem-floor obstruction` primary when that debt
  is already theorem-scale. It now also records a criterion-first geometry
  refinement: freeze the endpoint to the most concrete named criterion on
  disk, force the gain currency to be `criterion input` when that endpoint is
  a criterion family, and label package-free insertion sentences as narrower
  than earned theorem statements. The same topic now also records a
  post-freeze robustness rule for packet branches: once one canonical packet
  sheet is fixed, later audits must distinguish honest dependence on that
  sheet from actual exact-symmetry invariance, and may not relabel sign-sheet,
  threshold, spectator-partition, or time-window drift as canonicalization
  after outcomes are seen.
  It now also records that candidate-by-candidate robustness audits may keep
  several honest survivors with different statuses on one frozen ledger, and
  that behavior-based finite-window survivors should remain admission filters
  unless they later cash out in the same theorem-facing burden currency. It
  now also records a post-repair shortlist rule: tighten the strong
  candidates on the same ledger and drop any narrow-window survivor that
  still cannot justify a concrete next object freeze in the same burden
  currency. It now also records an anti-family screening rule: use static cancellation
  cases to test admissibility filters rather than to carry the main
  obstruction verdict, and require near-degenerate adverse families to say
  what stays live and what breaks first so they remain informative rather than
  turning into zero-activity excuse cases. It now also records a
  symmetric-dossier rule for exact packet screens: friendly witnesses stay
  class-local until the hostile dossier runs on the same frozen ledger, and
  low-leakage friendly families are the right stress test when a
  behavior-based candidate may really be failing by event-trace rigidity
  rather than raw isolation. It now also records a late-stage repair-honesty
  rule for behavior-based candidates: split early exact filters from late
  transfer requirements on the same frozen event sheet, and if the late
  repair only works by shortening the event language after outcomes are seen,
  discard it as definition drift rather than a survivor. It now also records a
  frozen-ledger repair rule: use the final landed dossier as the controlling
  record for repair passes, tighten each admissibility sheet only to the
  largest recorded friendly value, leave saturated budgets unchanged, and
  keep already-earned hostile failure modes instead of fabricating sharper
  adverse tables. It now also records a multi-step authority rule: when
  repaired sheets descend through several steps, final step-level results
  control, subordinate explorations support, and conflicting draft thresholds
  should be logged as history only before later math starts. It now also
  records a post-repair collapse rule for frozen packet ledgers: shared
  friendly/hostile outcomes do not justify merging two repaired gates unless
  one exact implication or counterexample is actually on disk. It now also
  records a budget-limited enlargement rule: when the current rung has no
  honest survivor, use the smallest admissible enlargement only to kill the
  tidiest pseudo-survivor on the same ledger, and do not let that artifact
  check self-authorize the next budget. It now also
  records a closure-stage stop rule for exact packet branches: when the record
  proves forced interaction classes around a frozen witness but still does not
  freeze one finite closed support ledger, the honest verdict is exact
  non-isolability / arbitrary truncation before dynamics rather than an
  overclaimed class-exit or infinite-growth theorem. It now also records a
  sharper sign-sheet collapse rule for support-sheet audits: mixed classes
  only merge under exact frozen equivalences already on disk, not under
  imagined extra target/input permutations. It now also records a first-spill
  audit rule for exact-support branches: once conjugate completion is
  mandatory, inspect mirror-cross pairs on the full ledger before self-pair
  distractions, because those off-orbit interactions often provide the
  earliest honest closure failure. It now also records a first-spill
  evidence rule: once those spill targets are explicit on a frozen support
  ledger, a compact all-sign coefficient probe can justify the current-budget
  verdict without being mistaken for an infinite-closure theorem.
- `workflow-monitoring/`: reusable session-monitoring lessons for telling apart
