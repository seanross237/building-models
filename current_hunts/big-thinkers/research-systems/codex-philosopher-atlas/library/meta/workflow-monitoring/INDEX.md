# Workflow Monitoring

Reusable lessons for deciding whether an exploration session actually produced
substantive output.

## Entries

- `do-not-treat-initial-report-summary-scaffolds-as-completion-sentinels.md`:
  sentinel checks should distinguish empty scaffolds from real completed
  reports and prefer terminal-status or non-placeholder-summary checks over
  file existence alone.
- `distinguish-launcher-stalls-from-research-inconclusiveness.md`:
  monitoring should separate runtime execution failure from substantive
  research uncertainty so operational noise is not mistaken for weak findings.
