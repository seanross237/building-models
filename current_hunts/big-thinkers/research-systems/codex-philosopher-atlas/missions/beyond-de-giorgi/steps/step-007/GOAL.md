# Step 7 Goal: Chain Step 1 Audit One Host Space Against a Full Extraction Package

## Mission Context

**Mission:** Beyond De Giorgi — What Structural Property Could Break the NS
Regularity Barrier?

**Active chain:**
`Chain 02 - Critical Element Program With Extraction Audit, Early Overlap Control, and Secondary Tao Screen`

**Why this step exists now:** `step-006` killed the prior
exact-reformulation branch and `decision-009` ordered replanning. Planning run
`run-004` selected the current Chain 02 as the mission's next canonical path.
That chain is only live if one concrete critical or near-critical host space
supports an explicit extraction package. So the first execution step is not
rigidity shopping. It is a theorem-by-theorem host-space audit that fixes one
admissible solution framework or honestly returns a no-host obstruction.

## This Step Covers

This strategizer execution covers **Chain Step 1 only**:

- compare `L^3`, `\dot H^{1/2}`, and only if explicitly justified
  `BMO^{-1}`;
- audit each candidate against the same checklist:
  - local theory in the intended solution framework;
  - large-data perturbation or stability;
  - profile decomposition or equivalent extraction input;
  - symmetry normalization;
  - pressure or local-energy compatibility;
  - and a compactness notion usable by later rigidity work;
- record the exact known inputs, the exact solution-class commitments, the
  nearest prior-art overlap, and at most one sharply identified missing theorem
  on the critical path for each candidate;
- choose one host space only if its package is concrete enough to support Chain
  Step 2 without smuggling in new unproved infrastructure;
- otherwise issue an explicit no-host verdict and stop the branch at the
  package stage.

Do **not** define the minimal blowup object yet. Do **not** promote rigidity
quantities yet. Do **not** let Tao-sensitivity dominate the audit; in this
chain it is secondary evidence after definability and contradiction power, not
the main gate.

## Required Deliverables

Produce all of the following inside `RESULTS.md`:

1. **Package-audit table**
   - one section each for `L^3`, `\dot H^{1/2}`, and `BMO^{-1}` if and only if
     inclusion is justified;
   - for every audited host space, give a table with these columns:
     - `requirement`
     - `known input`
     - `solution-class commitment`
     - `overlap / prior-art note`
     - `missing piece`

2. **Host-space verdict**
   - classify each candidate as one of:
     - `viable pending one sharp missing theorem`
     - `not viable because extraction package is diffuse`
     - `not admitted for this chain`
   - if one host space wins, name it and state why it is better than the
     alternatives on this chain's exact checklist;
   - if none wins, say plainly that the chain stops at a package obstruction.

3. **Critical missing-theorem memo**
   - for each candidate, name at most one theorem-level missing ingredient on
     the critical path;
   - reject any host space whose viability depends on a stack of loosely named
     future results rather than one sharply bounded gap.

4. **Overlap-control note**
   - distinguish what is already standard or already partially occupied in the
     literature from what this chain would newly contribute;
   - include prior mission overlap, internal run-history overlap, and the
     nearest compactness-rigidity literature family.

5. **Step verdict**
   - state whether Chain Step 2 should exist;
   - if yes, specify the exact host space and exact package inputs that are now
     frozen for the critical-object formulation step;
   - if no, recommend branch invalidation immediately rather than rhetorical
     continuation.

## Kill Conditions

Trigger an early negative conclusion for this step if any of the following
happens:

- no single host space clears the checklist except for at most one sharply
  stated missing theorem;
- a candidate appears viable only by switching solution class, normalization,
  or compactness notion mid-argument;
- the supposed package depends on diffuse missing infrastructure rather than a
  narrow theorem-level gap;
- the only case for a host space is rhetorical familiarity or literature aura
  rather than an explicit extraction package.

If every candidate fails under those standards, count that as a successful
bounded negative result for this step and recommend branch invalidation rather
than continuation to Chain Step 2.
