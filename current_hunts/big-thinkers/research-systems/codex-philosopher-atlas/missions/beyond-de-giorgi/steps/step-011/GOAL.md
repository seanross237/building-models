# Step 11 Goal: Chain Step 1 Calibrate The Invariant Package And Tao Discriminator

## Mission Context

**Mission:** Beyond De Giorgi — What Structural Property Could Break the NS
Regularity Barrier?

**Active chain:** `Canonical Chain - Coupled Energy-Helicity Route With Early Coercivity Gate`

**Why this step exists now:** Planning run `run-007` selected the
energy-helicity branch as the next canonical chain, but only under strict
early gatekeeping. No coercive gain has been earned yet. Before the branch is
allowed to talk about endpoint leverage, it must first show that its invariant
package, Tao discriminator, baseline criterion, and quantitative success
standard can all be fixed concretely rather than left as attractive cascade
language.

This step is therefore the setup-and-calibration gate. If it cannot freeze the
chain's scope lock or make the Tao screen operational, the correct output is a
setup or calibration obstruction and the branch should stop before the
coercivity gate.

## This Step Covers

This strategizer execution covers **Chain Step 1 only**:

- write the exact energy and helicity transfer identities in one chosen
  invariant package;
- separate exact structure from bookkeeping choices;
- define the Tao-sensitive feature in advance and state exactly what it means
  for Tao-style averaging to destroy it;
- give one worked passing Tao-calibration example and one worked failing
  example;
- freeze one provisional endpoint family, one baseline criterion, one minimum
  quantitative gain threshold, one admission rule, and one novelty rule;
- and screen immediately for collapse into known helical-triad, decimation,
  shell-model, or same-helicity folklore.

Do **not** run the coercivity gate yet. Do **not** promote any candidate gain
yet. Do **not** spend the branch's credibility on same-helicity sectors,
polarized regimes, or informal cascade stories without a burden-level account.

## Required Deliverables

Produce all of the following inside `RESULTS.md`:

1. **Scope-lock sheet**
   - freeze one provisional endpoint family:
     continuation criterion,
     concentration exclusion,
     or stretching/enstrophy bottleneck;
   - freeze one invariant package to audit first:
     coarse-grained local balances,
     dyadic coupled fluxes,
     or one other sharply defined ledger;
   - freeze one exact Tao-sensitive feature to test;
   - freeze one frozen baseline criterion that later steps must beat or
     genuinely sharpen;
   - freeze one minimum quantitative gain threshold that still matters after
     localization, pressure, viscosity, and mixed-helicity debt are paid;
   - freeze one admission rule and one novelty rule in language that later
     steps can enforce mechanically.

2. **Invariant identity memo**
   - write the exact energy and helicity transfer identities in the chosen
     package;
   - distinguish exact identities from bookkeeping or decomposition choices;
   - state which terms or transfers would count as the relevant bad term or bad
     transfer pattern for the chosen endpoint family.

3. **Tao calibration memo**
   - define exactly what "destroyed by averaging" means at the chosen
     comparison level;
   - give one worked example that clearly passes the Tao screen;
   - give one worked example that clearly fails it;
   - explain why those two examples are enough to make the screen operational
     rather than rhetorical.

4. **Prior-art overlap memo**
   - screen the chosen package against Waleffe-type triad analysis, helical
     decimation, shell-model intuition, and same-helicity favorable-sector
     lore;
   - state whether the package is genuinely distinct, merely repackaged
     folklore, or still ambiguous;
   - label each substantive claim `[VERIFIED]`, `[INFERRED]`, or `[PROPOSED]`.

5. **Step verdict**
   - classify the outcome as one of:
     - `calibration passed, proceed to coercivity gate`
     - `setup obstruction, branch should stop before coercivity gate`
     - `calibration obstruction, branch should stop before coercivity gate`
     - `prior-art overlap obstruction, branch should stop before coercivity gate`
   - if the step passes, state exactly what is now frozen for Step 2;
   - if the step fails, say plainly that the branch should not continue into
     coercivity testing.

6. **Source map**
   - name the exact source files used for every identity, Tao-calibration,
     prior-art, baseline, and verdict claim;
   - distinguish planning-run evidence, prior mission evidence, and new
     synthesis created in this step.

### Exploration C: Make the Tao screen operational

- test whether the chosen invariant package genuinely contains an NS-sensitive
  feature under that screen.

  rather than just decorate the prose.

### Exploration D: Screen prior-art collapse and write the verdict

Only run this after the scope lock and Tao calibration are concrete.

Tasks:
- compare the chosen package to the nearest helical-triad, decimation,
  shell-model, and same-helicity prior-art families;
- decide whether the package is distinct enough to justify Step 2;
- write the final step verdict and exact recommendation for mission control.

Success standard:
- mission control receives either one calibration-cleared setup for the
  coercivity gate or a branch-killing obstruction at Step 1.

## Kill Conditions

Trigger an early negative conclusion for this step if any of the following
happens:

- the scope lock cannot be fixed concretely;
- the Tao screen cannot produce one clear pass case and one clear fail case;
- the chosen invariant package collapses into generic cascade bookkeeping,
  same-helicity folklore, or known prior-art without a distinct theorem-facing
  role;
- the step cannot name one bad term or bad transfer pattern in the burden
  language of the chosen endpoint family;
- the branch stays alive only by delaying the baseline criterion, gain
  threshold, or Tao discriminator to later steps.

If a kill condition fires, count that as a successful bounded negative result
for this step and recommend branch invalidation before the coercivity gate.
