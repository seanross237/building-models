# Step 9 Goal: Chain Step 1 Freeze the Theorem Endpoint Before Freezing Tensor Language

## Mission Context

**Mission:** Beyond De Giorgi — What Structural Property Could Break the NS
Regularity Barrier?

**Active chain:**
`Endpoint-First Pressure-Hessian Obstruction Test With Quantitative Reconstruction and Localization Gates`

**Why this step exists now:** `step-008` killed the prior compactness-rigidity
branch at the setup stage and `decision-013` ordered replanning. Planning run
`run-006` selected the current pressure-Hessian chain as the next canonical
path, but only in a stricter endpoint-first form. The planning critique was
explicit: the tensor branch cannot start by freezing representation language,
observables, or Tao branding and only later decide what theorem endpoint those
objects are supposed to serve. This step therefore exists to freeze one exact
cash-out line, one progress currency, one minimum quantitative threshold, and
the branch's admission rules before any tensor package or observable shortlist
is allowed to matter. If that freeze fails, the branch should die immediately
at Step 1.

## This Step Covers

This strategizer execution covers **Chain Step 1 only**:

- choose one exact theorem endpoint and one contradiction line;
- state what a positive deliverable would literally look like:
  an inequality,
  a continuation criterion,
  or a defect identity with an explicit role in that endpoint;
- choose one progress currency for later comparison;
- define one minimum quantitative success threshold that would still matter
  after full reconstruction and any required localization losses;
- state the admission rule that same-scale unconstrained exterior or harmonic
  freedom counts as immediate failure;
- state the rule that Tao sensitivity is mandatory but not sufficient;
- and decide whether this endpoint-level setup is coherent enough to justify
  later prior-art screening and tensor-package work.

Do **not** shortlist tensor observables yet. Do **not** choose an eigenframe,
pressure-Hessian, or stretching representation package yet. Do **not** run the
full prior-art overlap screen yet. Do **not** charge the full reconstruction or
localization debt yet. Do **not** let Tao sensitivity substitute for theorem
relevance.

## Required Deliverables

Produce all of the following inside `RESULTS.md`:

1. **Endpoint-and-cash-out memo**
   - list the serious theorem endpoints and contradiction lines actually
     considered;
   - choose exactly one primary endpoint and exactly one contradiction line, or
     say plainly that no coherent endpoint setup exists;
   - state what a positive deliverable would literally be at that endpoint;
   - label each substantive claim `[VERIFIED]`, `[INFERRED]`, or `[PROPOSED]`.

2. **Progress-currency and threshold ledger**
   - choose one progress currency:
     one-sided inequality,
     monotonicity,
     cancellation of a named harmonic-moment family,
     or quantified reduction of a specified nonlocal remainder;
   - define one minimum quantitative threshold that a later tensor mechanism
     would have to clear after reconstruction and any localization debt;
   - explain why that threshold would still count as meaningful progress rather
     than cosmetic rephrasing.

3. **Admission-rules note**
   - state the exact rule that same-scale unconstrained exterior or harmonic
     freedom counts as immediate failure for later steps;
   - state the Tao rule precisely:
     averaged-model sensitivity is mandatory but does not by itself establish
     theorem relevance;
   - keep both rules tied to the frozen endpoint rather than to abstract
     structural rhetoric.

4. **Step verdict**
   - classify the step outcome as one of:
     - `endpoint frozen, proceed to prior-art and packaging screening`
     - `endpoint obstruction, branch should stop before tensor packaging`
   - if the setup survives, state exactly what later steps are still allowed to
     vary and what is already frozen;
   - if the setup fails, say plainly that the branch should be invalidated at
     the endpoint gate rather than reopened rhetorically in Step 2.

5. **Source map**
   - name the exact source files used for every endpoint, contradiction-line,
     threshold, and admission-rule claim;
   - distinguish run-006 planning evidence, prior mission evidence, and new
     synthesis.

## Kill Conditions

Trigger an early negative conclusion for this step if any of the following
happens:

- no exact theorem endpoint and contradiction line can be frozen without later
  redefining success;
- the alleged positive deliverable cannot be stated as a literal inequality,
  criterion, or defect identity with a named endpoint role;
- the progress currency or minimum threshold becomes cosmetic once later
  reconstruction or localization debt is acknowledged;
- the branch stays alive only by postponing the rule on same-scale exterior or
  harmonic freedom;
- Tao sensitivity is doing conceptual work that the endpoint-level analysis has
  not actually earned.

If a kill condition fires, count that as a successful bounded negative result
for this step and recommend branch invalidation before any tensor packaging or
observable screening is opened.
