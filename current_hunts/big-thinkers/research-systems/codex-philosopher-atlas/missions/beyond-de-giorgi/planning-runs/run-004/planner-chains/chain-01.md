# Chain 01 - Tao Differential Diagnosis And Mechanism Ranking

## Central Premise

The mission is filtered by Tao's averaged-model obstruction, so the first live branch is to compare exact NS against the averaged model directly and promote only those exact structures that are both genuinely destroyed by averaging and concrete enough to drive a later proof architecture.

## Verifiable Result Target

Either:

- a ranked shortlist of 1-2 exact NS discriminators, each attached to one concrete attack architecture and one named bottleneck term; or
- a bounded negative result showing that every currently visible discriminator is identity-level, compatibility-blocked, or too weak to alter a proof mechanism.

## Why This Chain Is Meaningfully Different

This chain is not a proof attempt in one fixed architecture. It is a Tao-filtered mechanism audit whose product is a ranked map of the live non-De-Giorgi landscape.

## Ordered Steps

### Step 1 - Fix the comparison protocol

- Depends on: none.
- Action: state exactly what Tao's averaged equations preserve for this mission and what kind of evidence is required to say an NS feature is truly destroyed by averaging.
- Action: record the prior negatives that are now hard exclusions: far-field coefficient shrinkage, neutral-Eulerian geometry rhetoric, and fixed-protocol exact rewrites.
- Expected output: a comparison memo with a preserved/destroyed ledger and an explicit admissibility rule for candidate discriminators.
- Kill condition: if the comparison standard cannot be made concrete, stop and write an under-specified-filter memo rather than pretending the branch is live.

### Step 2 - Build a bounded discriminator list

- Depends on: Step 1.
- Action: shortlist at most four candidate NS-only structures, for example pressure-Hessian tensor algebra, strain-vorticity alignment identities, Lagrangian circulation or topology, or stochastic flow structure.
- Action: reject candidates that survive only as norm-level facts, static CZ geometry, or reformulation-level packaging.
- Expected output: a discriminator table with one-line Tao verdicts and one-line novelty notes relative to prior runs.
- Kill condition: if no candidate survives as more than identity-level structure, close the branch negatively.

### Step 3 - Attach each survivor to one concrete architecture

- Depends on: Step 2.
- Action: for each surviving discriminator, name one proof architecture where it would have to act: critical-element rigidity, tensor stretching control, Lagrangian persistence, or another explicitly named route.
- Action: name the exact bottleneck term or theorem gap that would have to change.
- Expected output: an insertion map from discriminator to architecture, bottleneck, and gain standard.
- Kill condition: if a candidate cannot be attached to a concrete bottleneck, demote it from the live set.

### Step 4 - Run a compatibility and anti-cosmetic screen

- Depends on: Step 3.
- Action: test the top candidates against the intended solution class, localization demands, and prior mission negatives.
- Action: reject any route whose first missing lemma is equivalent to the full regularity problem or whose supposed gain is only cosmetic.
- Expected output: a ranked shortlist with explicit first missing lemmas and failure modes.
- Kill condition: if every survivor is compatibility-blocked or circular, end with a ranked obstruction memo.

### Step 5 - Close with a portfolio handoff

- Depends on: Step 4.
- Action: promote at most two discriminators to follow-on chains and state exactly why they outrank the rest.
- Action: if nothing survives, state the strongest earned reason the exact NS-versus-averaged gap still fails to operationalize.
- Expected output: either a mission-ready ranking memo or a bounded negative diagnosis of the current landscape.
- Kill condition: if the final ranking claims more than the audited candidate list supports, narrow it until each promotion is fully earned.
