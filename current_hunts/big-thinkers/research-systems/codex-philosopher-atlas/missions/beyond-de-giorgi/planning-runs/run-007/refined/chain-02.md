# Refined Chain 02 - Material Observable Route With Front-Loaded Admissibility Gates

## Central Premise

Treat material exactness as suspect until it proves endpoint-facing coercive
value.

This branch should not assume that a Lagrangian or deformation-based observable
is stronger than its Eulerian counterpart. It remains live only if one material
observable:

- is definable in the intended solution class without assuming smooth flow-map
  control;
- is not merely an equivalent packaging of known BKM-type or cumulative-strain
  burdens;
- retains meaning after exact viscous and nonlocal strain terms are restored;
- yields one explicit, quantitatively benchmarked gain in a named theorem
  endpoint or contradiction line; and
- uses an NS-specific feature that Tao's averaged model destroys, where that
  feature appears inside the actual endpoint-facing estimate rather than only in
  narrative.

If those conditions fail, the correct output is a calibrated obstruction memo,
not a positive transport blueprint.

## Verifiable Result Target

Either:

- one proposition-scale conditional statement naming:
  one observable,
  one admissible solution framework,
  one exact theorem endpoint,
  one exact evolution or defect identity,
  one baseline known criterion that is being genuinely sharpened,
  one minimum quantitative gain threshold,
  and one genuinely narrower missing lemma; or
- one calibrated obstruction memo identifying the first honest failure:
  undefined observable at target regularity,
  prior-art-only repackaging,
  viscous/material formulation unusable,
  nonlocal strain freedom destroys coercivity,
  endpoint insertion gives no benchmarked gain,
  or the remaining lemma is equivalent to the full continuation problem.

## Why This Chain Is Still Worth Running

Even if it fails positively, it can still produce a useful obstruction map:
which exact material structures are real but analytically inert, which fail at
definability, and which collapse into known BKM-type burdens once viscosity and
nonlocality are written honestly.

## Scope Lock

Before any derivation, freeze all of the following:

- one exact theorem endpoint or contradiction line;
- one admissible solution class near the intended regularity threshold;
- one baseline known criterion for non-equivalence checking;
- one minimum quantitative success threshold that would still matter after all
  viscous, nonlocal, and localization debts are paid;
- one rule that Tao sensitivity is necessary but not sufficient; and
- one rule that an observable requiring near-classical flow-map control before
  the endpoint is reached is inadmissible.

If these cannot be fixed concretely, stop and write a setup obstruction.

## Ordered Steps

### Step 1 - Freeze the endpoint, solution class, and baseline criterion before choosing the observable

- Depends on: none.
- Action: choose one exact endpoint:
  a continuation criterion,
  a contradiction inequality,
  or another explicitly named cash-out line.
- Action: choose one admissible solution framework in which the observable is
  supposed to live.
- Action: choose one baseline known criterion to beat or genuinely sharpen:
  for example a BKM-type cumulative stretching burden, a cumulative strain
  criterion, or another named continuation package.
- Action: define one minimum quantitative gain threshold that would still matter
  after all later debts are restored.
- Expected output: an endpoint-and-baseline memo naming the theorem target, the
  admissible regularity class, the baseline criterion, and the success
  threshold.
- Kill condition: if no exact endpoint, admissible framework, and baseline test
  can be fixed together, stop and write a setup obstruction.

### Step 2 - Admit only one observable that survives definability and anti-repackaging screening

- Depends on: Step 1.
- Action: choose one observable only after checking that it is definable in the
  chosen framework without assuming smooth flow-map control or the desired
  continuation conclusion.
- Action: screen the candidate against prior families:
  Cauchy-formula stretching views,
  deformation-gradient identities,
  back-to-label criteria,
  and BKM-equivalent cumulative strain formulations.
- Action: write explicitly why the observable is not merely a reparameterization
  of the baseline criterion.
- Action: reject any candidate whose only apparent novelty comes from packaging
  or coordinate choice.
- Expected output: an admissibility memo naming one observable, its precise
  domain of definition, and one explicit non-equivalence claim relative to the
  baseline criterion.
- Kill condition: if the observable is undefined at target regularity or only
  repackages known burdens, stop and record the obstruction immediately.

### Step 3 - Write the exact evolution only with the full viscous and nonlocal debt visible

- Depends on: Step 2.
- Action: derive the exact evolution law or exact defect identity for the chosen
  observable in the admitted framework.
- Action: include every viscous commutator, second-derivative debt, and
  nonlocal-strain contribution needed to return to the full NS dynamics.
- Action: state one NS-specific feature that Tao's averaged model destroys, but
  keep the branch alive only if that feature has a plausible role in the
  endpoint-facing estimate rather than only in exposition.
- Action: reject any formulation whose exact law exists only in a regime
  stronger than the branch is allowed to assume.
- Expected output: a full-debt evolution memo with exact terms, defect terms,
  regularity requirements, and one Tao-sensitive feature tied to later use.
- Kill condition: if the exact material law becomes unusable once the full
  viscous or nonlocal debt is written, stop and write a formulation obstruction.

### Step 4 - Force the observable into one endpoint-facing inequality with a benchmarked gain

- Depends on: Step 3.
- Action: write one explicit inequality, criterion-sharpening statement, or
  contradiction line showing how control of the observable acts on the chosen
  endpoint.
- Action: state the benchmark clearly:
  what burden is reduced,
  in what norm or scale,
  and by how much relative to the baseline criterion.
- Action: require the Tao-sensitive feature from Step 3 to appear inside this
  endpoint-facing mechanism, not just in surrounding discussion.
- Action: reject routes that merely restate vorticity amplification or
  cumulative strain in material language.
- Expected output: an insertion memo containing one explicit endpoint-facing
  statement, one quantitative benchmark, and one sentence explaining where the
  Tao-sensitive structure enters.
- Kill condition: if the branch cannot show a benchmarked gain over the named
  baseline, stop and record an endpoint-insertion obstruction.

### Step 5 - Audit whether the remaining gap is genuinely narrower than full regularity

- Depends on: Step 4.
- Action: if a positive route remains, isolate the first missing lemma and test
  whether it is strictly narrower than the full continuation problem in scale,
  norm strength, or scope.
- Action: reject any missing lemma that is just a renamed request to control the
  original cumulative stretching or full nonlocal strain burden.
- Action: if the route fails, state the first failure precisely and allow
  overlapping failure labels rather than forcing a single tidy slogan.
- Expected output: either a proposition-scale target with one genuinely narrower
  missing lemma or a calibrated obstruction memo stating the first honest
  failure.
- Kill condition: if the remaining gap is equivalent to the original regularity
  problem, downgrade the branch to obstruction.

### Step 6 - Close only on an earned survivor

- Depends on: Step 5.
- Action: if the branch survives, write the final conditional result with:
  the observable,
  the admissible framework,
  the exact endpoint,
  the full-debt identity,
  the benchmarked gain,
  and the first genuinely narrower missing lemma.
- Action: if not, write the narrowest honest obstruction memo, making clear
  whether failure came first from definability, repackaging, viscous debt,
  nonlocal strain freedom, or lack of endpoint leverage.
- Expected output: a final comparison-ready chain result, either positive or
  negative.
- Kill condition: if the final positive statement hides the main regularity
  difficulty inside an unlabeled estimate, convert it to obstruction.
