# Chain 03 - Weak-Type Endpoint Architecture Audit Beyond Strong-Type De Giorgi

## Central Premise

The only admissible Lorentz or weak-type route is one that replaces the exhausted strong-type De Giorgi mechanism rather than tuning it. This branch is live only if weak-type structure supports a genuinely different endpoint criterion, concentration architecture, or contradiction line that never asks for a better `beta` or a prettier version of the old recursion.

## Verifiable Result Target

Either:

- one theorem-like weak-type endpoint program naming the frozen target criterion, the exact weak-type inputs, the localization rules, and the first missing lemma; or
- a bounded obstruction memo showing that the route collapses back into strong-type control, hidden De Giorgi recursion, or Tao-compatible generic harmonic analysis.

## Why This Chain Is Meaningfully Different

This chain is endpoint-functional and architecture-auditing. It is not a tensor route, not a helical-symbol route, and not a geometry/persistence route. It is only allowed to live if it avoids the dead De Giorgi ledger rather than disguising it.

## Ordered Steps

### Step 1 - Freeze anti-recursion guardrails and one endpoint target

- Depends on: none.
- Action: state explicitly what is forbidden:
  no attempt to improve `beta`,
  no new Chebyshev-level iteration,
  no pressure-term norm-shopping inside the old `U_k` recursion.
- Action: choose one legitimate endpoint target in advance, such as a weak-type local regularity criterion, a distribution-function control statement, or a concentration bound that feeds a different contradiction line.
- Action: choose one gain currency in advance:
  endpoint closure,
  distribution-tail decay,
  or localization-compatible weak-type control.
- Expected output: an architecture memo freezing the anti-recursion rules, one target endpoint, and one gain currency.
- Kill condition: if the only plausible targets already require the old De Giorgi recursion in disguise, stop and write a setup obstruction.

### Step 2 - Audit the exact weak-type input package

- Depends on: Step 1.
- Action: list only the weak-type or Lorentz inputs actually available or plausibly admissible for the branch, such as weak-type Calderon-Zygmund bounds, Lorentz interpolation, endpoint pressure estimates, and local-energy compatibility.
- Action: separate what is genuinely weak-type from what immediately upgrades to strong-type and loses its distinctness.
- Action: keep pressure/localization compatibility explicit so the branch does not silently drop the inherited weak-solution setting.
- Expected output: a weak-type input ledger with exact tools, compatibility notes, and known gaps.
- Kill condition: if the branch lacks a coherent weak-type tool package or instantly collapses back to strong-type estimates, end with an input obstruction.

### Step 3 - Build one non-De-Giorgi endpoint mechanism

- Depends on: Step 2.
- Action: choose at most one promoted mechanism:
  an endpoint regularity criterion,
  a weak-type concentration inequality,
  or a localized pressure-transport control statement.
- Action: state exactly how the chosen mechanism uses the Step-2 weak-type inputs and where it differs from the old strong-type iteration.
- Action: reject any mechanism whose progress depends on level-set recursion under another name.
- Expected output: an insertion memo from the weak-type package to one frozen endpoint target.
- Kill condition: if no mechanism survives without hidden iteration debt, close the chain negatively.

### Step 4 - Charge localization debt and apply the Tao screen

- Depends on: Step 3.
- Action: test whether the promoted weak-type mechanism survives cutoffs, pressure decomposition, and any truncation or distribution-function bookkeeping needed in the local setting.
- Action: ask whether the mechanism uses anything beyond generic harmonic analysis and weak-type singular-integral theory that Tao's averaged model could preserve.
- Action: classify failure precisely as localization collapse, strong-type collapse, hidden De Giorgi recursion, or Tao-insensitive endpoint packaging.
- Expected output: a localization-and-Tao memo with one promoted mechanism or a typed obstruction.
- Kill condition: if the route either reproduces the old `4/3` ceiling or survives unchanged in the averaged setting, stop and write the obstruction.

### Step 5 - Promote only an earned endpoint claim

- Depends on: Step 4.
- Action: if one mechanism survives, write a theorem-like conditional program naming the endpoint target, exact weak-type inputs, localization rules, gain, and first missing lemma.
- Action: if nothing survives, write the narrowest earned negative:
  no distinct weak-type package,
  no non-iterative mechanism,
  or no Tao-sensitive endpoint leverage.
- Expected output: either a theorem-like weak-type route or a precise endpoint obstruction memo.
- Kill condition: if the final claim cannot be stated without smuggling the old De Giorgi architecture back in, downgrade it to obstruction.
