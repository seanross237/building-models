# Canonical Winning Chain - Criterion-First Geometric Mechanism Stress Test With Theorem-Floor and Same-Currency Gates

## Central Premise

Geometry is not presumed to be the leading route. It is a hypothesis under audit. This branch is live only if one explicit geometry-aware package and one geometric observable can be defined directly on the suitable-weak / LEI theorem floor, or else on one proxy class whose descent back to that floor is quantitatively cheaper than the target theorem, and if that package-observable pair produces theorem-facing leverage beyond the closest live geometric prior art with all bridge costs, localization costs, and persistence costs paid in the same endpoint currency as the claimed gain.

If that cannot be shown, the correct output is a bounded obstruction memo or a hybrid geometry-transport handoff, not a rescued geometry narrative.

## Verifiable Result Target

Either:

- one live geometry packet containing a named endpoint, closest prior-art comparator, explicit package, admissibility standard, observable-to-endpoint map, quantitative debt ledger, and prototype insertion or estimate; or
- one bounded obstruction memo showing theorem-floor mismatch, package theater, endpoint-currency mismatch, persistence collapse, nonlocal remainder dominance, prior-art absorption, or a real-but-too-weak effect after full debt; or
- one hybrid handoff memo showing that the apparent gain becomes live only after importing transport structure.

## Why This Chain Is Better

This chain keeps the best feature of the geometric route, namely the highest novelty ceiling, but it imports the theorem-floor and same-currency discipline that the deterministic stretching route enforced more sharply. The branch is therefore not allowed to survive on package-internal elegance. It must touch the actual endpoint ledger early or stop.

## Scope Lock

Before the chain is active, it must freeze all of the following:

- one theorem-facing endpoint only:
  a Constantin-Fefferman-type criterion by default, or one named full-stretching contradiction target;
- one admissibility standard only:
  direct suitable-weak / LEI definability by default, or one named proxy class with explicit bridge burden;
- one closest live geometric prior-art comparator;
- one package only:
  tube-following localization,
  high-vorticity tracking,
  or one fully specified named alternative written out concretely;
- one primary observable family only, with exact object class named;
- one gain currency only:
  burden share of stretching,
  derivative count,
  threshold stability,
  criterion input,
  or localization stability;
- one favorable scenario and one hostile scenario for stress testing;
- one burden ledger carrying:
  proxy-bridge burden if any,
  `nabla xi` debt,
  boundary leakage,
  tube-identification ambiguity,
  relocalization cost,
  pressure or cutoff cost when relevant,
  temporal-window cost when relevant,
  and nonlocal remainder cost.

If these cannot be fixed concretely, stop with an under-specified geometry memo.

## Ordered Steps

### Step 1 - Freeze the endpoint, admissibility standard, comparator, and candidate proposition first

- Depends on: none.
- Action: choose one endpoint family only and write one candidate proposition, contradiction line, or insertion claim before selecting a flattering package.
- Action: state whether the observable will live directly on the theorem floor or on one proxy class, and if proxy-based, write the exact bridge burden back to suitable-weak / LEI data.
- Action: name the closest live geometric prior art as the primary comparator and keep the killed neutral-Eulerian package only as a secondary comparator.
- Action: state one minimum non-cosmetic gain in the chosen currency.
- Expected output: an endpoint-and-comparator memo containing one candidate theorem-facing statement and one admissibility claim.
- Kill condition: if no candidate statement can be written before package optimization, if the observable is not theorem-floor admissible and the bridge is already theorem-scale, or if the claimed gain is not clearly stronger than descriptive convenience, stop with an endpoint-free or theorem-floor obstruction.

### Step 2 - Choose one explicit package and one observable with a theorem-facing currency map

- Depends on: Step 1.
- Action: choose one package from the closed menu, or write one fully specified named alternative with no "another geometry-aware regime" escape hatch.
- Action: choose one primary observable family only, name its exact object class, and explain why it naturally speaks to the frozen endpoint and gain currency.
- Action: write the explicit observable-to-endpoint map, including any required transfer step from proxy class to theorem floor.
- Action: choose one favorable scenario and one hostile scenario, but use them only as later stress tests rather than as evidence that the route is live.
- Expected output: a package-and-observable memo with an explicit observable-to-endpoint map.
- Kill condition: if the map requires an unproven conversion or transfer lemma that is already as hard as the original problem, stop with a currency-mismatch or proxy-inflation obstruction.

### Step 3 - Derive the localized theorem-facing ledger before package optimism accumulates

- Depends on: Step 2.
- Action: derive the localized evolution or insertion sheet in the form actually relevant to the frozen endpoint.
- Action: mark exactly where pressure, cutoff, temporal-window, commutator, nonlocal, or regularity-sensitive terms enter.
- Action: separate exact terms from terms that only close off the theorem floor.
- Action: force all gains and defects into the same frozen endpoint ledger rather than package-native units.
- Expected output: a localized theorem-facing ledger with same-currency defects exposed.
- Kill condition: if the key gain only exists off the theorem floor, or if the defects cannot be expressed as credibly lower order in the frozen ledger, stop with an exactness or non-separation obstruction.

### Step 4 - Run a quantitative persistence and debt screen outside the package

- Depends on: Step 3.
- Action: quantify the full burden ledger immediately, including derivative debt, frame-tracking debt, leakage, relocalization cost, proxy-bridge cost, and nonlocal remainder cost.
- Action: test whether the observable persists on the actual theorem floor, not merely inside a continually refitted package.
- Action: allow one auxiliary observable only if its cost is separately charged and the primary observable still carries the route.
- Expected output: a persistence ledger comparing minimum gain against total package debt.
- Kill condition: if persistence depends on repeated retuning, hidden axis fitting, debt that already exceeds the minimum gain, or survival that is real but too weak after full debt, stop with a persistence-collapse or too-weak obstruction.

### Step 5 - Audit contact with full stretching in a representation-robust way

- Depends on: Step 4.
- Action: test the promoted observable against a canonical stretching representation and check that the conclusion survives translation to any alternate representation used for the claim.
- Action: separate near-field gain, nonlocal or exterior remainder, and localization-evolution debt.
- Action: require either a quantitatively meaningful burden share of the full stretching term or a precise explanation of how the controlled share propagates to the endpoint.
- Expected output: a full-stretching audit with unpaid remainder clearly identified.
- Kill condition: if the observable controls only a local depletion fragment, only works in one friendly representation, or never changes the inherited endpoint ledger, stop with a full-stretching obstruction.

### Step 6 - Produce a hard artifact before any promotion

- Depends on: Step 5.
- Action: produce one hard artifact:
  a prototype estimate,
  a representative insertion into the candidate proposition,
  or a localized stress test that runs both the favorable and hostile scenarios.
- Action: treat a failed lemma as obstruction evidence only, unless the failure still isolates a narrower live theorem-facing insertion point.
- Action: benchmark the artifact against the closest live geometric prior art and the neutral-Eulerian comparator.
- Expected output: an evidence packet showing either genuine gain or sharp failure.
- Kill condition: if the artifact merely repackages direction coherence, Beltrami depletion, or tube persistence without a stronger insertion point than prior art, stop with a prior-art absorption obstruction.

### Step 7 - Use Tao sensitivity and transport dependence as secondary discriminators, then state the narrowest earned claim

- Depends on: Step 6.
- Action: ask whether the artifact is plausibly erased by Tao-style averaging, but only after theorem-facing contact has already been shown.
- Action: ask whether the claimed gain actually depends on imported quasi-Lagrangian or transport machinery.
- Action: classify the route as:
  live geometry,
  hybrid geometry-transport,
  bounded obstruction,
  or real-but-too-weak after full debt.
- Action: state exactly which endpoint, admissibility standard, comparator, package, observable, burden currency, and stress tests were audited.
- Action: record the first missing lemma only if the route survived the hard-artifact gate.
- Expected output: one narrow final memo with no conclusion outrunning the artifact or the debt ledger.
- Kill condition: if the surviving leverage is transport-dependent, Tao-insensitive in a way that erases distinctiveness, or narrower than theorem-facing relevance, downgrade to hybrid or obstruction.

## Probability Assessment

**Probability that this canonical chain yields a presentable result: 0.79**

The likely presentable result is still a disciplined obstruction memo rather than a positive geometric mechanism. That is acceptable because this chain is explicitly designed to test whether geometry buys a theorem-facing lever after theorem-floor admissibility, same-currency budgeting, and full debt charging, not to rescue geometry by packaging alone.
