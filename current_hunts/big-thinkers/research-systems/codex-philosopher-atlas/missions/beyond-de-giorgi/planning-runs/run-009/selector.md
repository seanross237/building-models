   - Best direct mission relevance.
   - Strongest guard against fake progress because the anti-conjugation rule forces an estimate-level gain on the inherited LEI / CKN burden.

2. **Chain 03 - Deterministic Material Stretching Route With Viscous-Defect And LEI Admission Gates**
   - Best reserve after the stochastic transport branch failed.
   - Admissibility, viscous-defect, and localization gates make failure informative rather than diffuse.

3. **Chain 02 - Packet-Complete Helical Interaction Route With Bad-Transfer Freeze**
   - Highest upside algebraic route still on disk.
   - Worth selecting because it could expose a real exact-symbol discriminator that survives mixed-helicity restoration.
   - Ranked below Chain 03 because reconstruction and localization debt create a higher chance of elegant but non-cashing symbolic structure.

4. **Chain 04 - Adapted-Package Geometric Vorticity Route From Local Beltrami Deficit To Criterion Scale**
   - Honest geometry reboot with a mandatory package audit.
   - Useful as a reserve, but it starts from a weaker ledger because the old Eulerian package is already known to be weak and the new package risks hiding the gain inside the localization choice.
   - Lower portfolio value than the top three because its likely failure mode is package theater or prior-art collapse.

5. **Chain 05 - Critical Element Program With Pre-Admitted Exact-NS Firewall**
   - Correct only as a second-stage consumer.
   - Not suitable for current adversarial testing because it cannot move until another chain produces an admitted exact-NS firewall quantity.
   - Failure is mostly predetermined no-entry bookkeeping rather than fresh mechanism discrimination.

## Why These Three Make The Strongest Portfolio

The selected set maximizes meaningful diversity without paying for a weakly differentiated reboot. Chain 01 is the anchor because it attacks the clearest live thesis on the board: whether a genuine Navier-Stokes reformulation can change the mechanism on the inherited LEI / CKN ledger. Chain 03 gives the portfolio a deterministic transport-side branch that remains theorem-floor honest after the stochastic route failed, and its admission and coercivity gates mean that even a negative result produces a precise obstruction. Chain 02 preserves the highest-upside symbolic path, but only in a hardened packet-complete form that can be killed cleanly if mixed-helicity restoration or reconstruction erases the gain.

The excluded chains are weaker for adversarial testing at this stage. Chain 04 is a legitimate backup, but its main burden is proving that the package change buys something real rather than relocating the weakness, so it is less attractive than Chain 03 as the third mechanism family. Chain 05 should not consume one of the three slots because it is downstream architecture, not an upstream mechanism hunt; until a firewall quantity exists, it adds little beyond an immediate no-entry verdict.
