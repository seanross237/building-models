# Decision Memo - beyond-de-giorgi / decision-006

## Decision

`replan`

## Mission-Control Verdict

The active geometry chain should not proceed. `step-004` was explicitly
designed to screen dynamic plausibility before any kernel-level
`S omega . omega` representation work. It completed that screen on the fixed
Step-3 package and fired the branch-kill condition honestly.

The result is not a soft weakening. It is a branch-level negative. On the
inherited Eulerian parabolic package for filament-or-tube concentration, no
candidate cleared the `dynamically plausible` bar. The primary hybrid,
`direction coherence + tube persistence`, remained the best idea only as
`informative but dynamically weak`, and every comparator or fragility screen
was likewise dynamically weak or static-only. The step's own conclusion is
that moving to Chain Step 4 would be blind momentum, because the only visible
rescues require exactly the forbidden moves: tube-adapted relocalization,
threshold/window retuning, or stronger derivative control than the audited
package contains.

## Why `proceed` Is Wrong

Proceeding would ignore the gatekeeping standard of the current chain. Chain
Step 3 was supposed to decide whether any candidate had a credible persistence
story before the mission spent effort fixing a stretching representation. That
question has now been answered negatively on the package the branch itself
committed to.

This is enough to invalidate the current branch order. The next logical step
of the active chain is no longer the best next mission move.

## Why `terminate` Is Wrong

This is a branch kill, not a mission kill. The mission-level question remains
open: what NS-specific structure, if any, can bypass the De Giorgi barrier
outside the failed geometry package?

The planning record still preserves distinct reserve branches worth comparing
again, including the architecture-fixed exact-reformulation obstruction route
and the critical-element audit route from `planning-runs/run-002`. The latest
step does not show that the whole mission is complete either positively or
negatively. It shows only that the active geometry chain has failed on its
audited package.

## Required Next Controller Move

Mission control should replan now.

The replanning pass should treat the current geometry chain as invalidated and
compare the remaining live alternatives afresh. It should not auto-advance to
Chain Step 4, and it should not smuggle continuation through a quiet package
change. If geometry is revisited later, that must happen as a newly compared
branch with an openly changed package rather than as continuation of the
current chain.
