# Meta Note - Step 011 Exploration 003

Prior-art overlap should be treated as a secondary kill label when a branch
still has one real NS-specific discriminator but no theorem-facing gain.

Reusable lesson:

- do not call a branch `pure repackaging` if one narrow packet-level
  discriminator genuinely survives;
- but do not let that discriminator keep the branch alive when the setup lock
  and calibration still fail to produce a one-sided endpoint-facing gain;
- the earliest honest obstruction label should win, with prior-art adjacency
  recorded as supporting context rather than inflated into a false positive or
  the wrong primary cause.

For this step, prior-art overlap mattered, but the branch was already dead on
setup and calibration grounds.
