# Meta Note — Step 1 Exploration 002

## What Worked

- The run-001 planning stack already contained the right representation lesson:
  separate true symmetries, canonicalization choices, and substantive modeling
  changes before judging object stability.

## Reusable Lesson

- When `support` is mission-central, prefer packet identity at the smallest
  object level that keeps exact bookkeeping honest, and demote conjugate
  completion to mandatory canonical representation rather than primary object
  identity.

## Caution

- Treating conjugate completion, packet regrouping, or helical-basis changes as
  "obviously cosmetic" is a common source of false object stability.
  Freeze the canonicalization policy first, then ask what is actually cosmetic.
