# Chain 01 - Exact `F_SL(rho)` Template Frontier

## Central Premise

After Step `008`, the cleanest surviving positive theorem route is no longer
the invalidated `F_SS(1/12)` dynamic branch.
It is the carried scale-separated family `F_SL(rho)` under repaired
`Template-Defect Near-Closure`.

The branch should determine exactly which `rho` values satisfy repaired
`G_tmpl`, whether the current carried witness `rho = 1/16` is genuinely
boundary-sharp, and what this family-local result does and does not say about
exact closure.

## Verifiable Result Target

Either:

- an exact family-local theorem giving the admissible `rho` interval for
  repaired `G_tmpl` on `F_SL(rho)`;
- a corrected endpoint memo showing that only a narrower witness set, not a
  full interval, is currently earned; or
- a bounded negative showing that the repository does not yet support one exact
  `rho`-dependent defect ledger in the repaired Step-6 currency.

## Why This Chain Is Meaningfully Different

This chain is family-local and single-object.
It works only with repaired `G_tmpl` on the carried `F_SL(rho)` family.
It does not try to prove a support-level no-closure theorem, define a third
closure object, or search for a new explicit packet.

## Ordered Steps

### Step 1 - Freeze The Family, Authority Sheet, And Allowed Scope

Depends on: none.

Action: freeze the canonical one-bridge packet semantics, repaired
`G_tmpl(P_n; I) = (Delta_tmpl, Delta_spec)` with thresholds
`(1/4, 49/256)`, fixed window `I = [0, 1]`, and the carried family
`F_SL(rho)`.

Action: freeze the present family facts already on disk:
`F_SL(1/16)` is the carried friendly witness,
Step-4/5 arithmetic suggests a larger possible endpoint,
and Step-8 does **not** license any closure claim from this template pass.

Expected output: one frozen family-setup memo stating exactly what the branch
will prove, and what it refuses to infer about support-level closure.

Kill condition: if the current repository cannot recover one exact frozen
definition of `F_SL(rho)` in the same repaired template-defect currency, stop
and package the branch as a source-basis failure.

### Step 2 - Derive The Exact `rho`-Dependent Defect Ledger

Depends on: Step 1.

Action: derive or rigorously verify explicit formulas for
`Delta_tmpl(rho)` and `Delta_spec(rho)` on the carried family, using the same
normalized packet sheet and any existing checked arithmetic or scripts.

Action: keep the desired-channel and spectator contributions separate so the
later boundary computation is exact rather than a soft fit.

Expected output: one exact `rho`-ledger for repaired `G_tmpl`, ideally as
closed formulas plus one reproducible check path.

Kill condition: if the branch can recover only endpoint numerics and not one
honest family-level ledger, downgrade the branch to a witness-local memo and
stop.

### Step 3 - Solve The Admissible Interval And Identify The Sharp Boundary

Depends on: Step 2.

Action: solve the repaired inequalities
`Delta_tmpl(rho) <= 1/4` and `Delta_spec(rho) <= 49/256`,
identify the true admissible interval, and decide whether the carried witness
`rho = 1/16` is interior, boundary, or merely conservative.

Action: explicitly test the Step-4 lead for the candidate boundary
`rho_* = (-3 + sqrt(10)) / 2` if that arithmetic still survives the repaired
ledger.

Expected output: an admissible-interval theorem candidate or a corrected
boundary memo.

Kill condition: if the interval endpoint depends on retuning thresholds,
switching packet semantics, or untracked asymptotic error terms, stop.

### Step 4 - Audit What The Family Theorem Does Not Say About Closure

Depends on: Step 3.

Action: compare the family-local template result against the Step-8

Action: decide whether the family theorem is purely a repaired-template

### Step 5 - Package The Family-Level Verdict

Depends on: Step 4.

Action: package the strongest honest endpoint:
family-local theorem,
corrected witness-only memo,
or bounded negative.

Expected output: one comparison-ready theorem memo that states exactly what is
proved on `F_SL(rho)` and exactly what remains open beyond the carried family.

Kill condition: if the final claim generalizes from the audited `F_SL(rho)`
