# Chain 02 - Classwise Leakage Sharpness And Friendly-Minimality Theorem

## Central Premise

The repaired leakage sheet is now the cleanest obstruction-facing object on
disk:

`L_tot <= 1/4`,
`L_mirror <= 1/12`,
`L_companion <= 1/12`,
`L_feedback <= 1/16`,
`L_cross <= 1/24`.

The next honest theorem is to prove that this exact classwise budget is really
the smallest friendly-admissible sheet supported by the carried calibration set
`{F_SS(1/12), F_SL(1/16)}`,
then pin as much same-currency hostile overload as the present record
actually allows.

## Verifiable Result Target

Either:

- a theorem-scale sharpness/minimality statement for repaired
  `Windowed Spectator-Leakage Budget`
  on the frozen packet sheet, together with a same-currency hostile overload
  clause to the extent actually earned; or
- a corrected negative memo showing that the repaired vector is not genuinely
  sharp, is not stable under the frozen interaction currency, or cannot support
  an honest hostile comparison yet.

## Why This Chain Is Meaningfully Different

This chain is about off-template mass and exact spectator bookkeeping. It does
not try to prove template closeness, optimize the joint feasible set, or build
an explicit near-circuit dynamically.

## Ordered Steps

### Step 1 - Freeze The Exact Leakage Currency

Depends on: none.

Action: freeze the canonical one-bridge packet sheet,
the fixed window `I = [0, 1]`,
the exact
`D_on` / `D_off`
split,
the spectator partition
`mirror`,
`companion`,
`feedback`,
`cross`,
and the repaired budget vector
`(1/4, 1/12, 1/12, 1/16, 1/24)`.

Action: freeze the carried friendly calibration set exactly as handed off:
`F_SS(1/12)`
and
`F_SL(1/16)`.

Expected output: one leakage-theorem setup sheet fixing the admissibility
currency and the friendly calibration set.

Kill condition: if the same packet or witness set can only be compared after
changing the spectator partition or burden currency, the repaired leakage
object is not stable enough for theorem work.

### Step 2 - Derive Exact Classwise Ratio Ledgers For The Friendly Set

Depends on: Step 1.

Action: compute or rigorously verify the exact classwise leakage vector for
each carried friendly witness:
`F_SS(1/12)`
and
`F_SL(1/16)`.

Action: identify which witness saturates which repaired coordinate and which
coordinates remain slack on each witness individually.

Expected output: a two-witness friendly sharpness table in the frozen classwise
currency.

Kill condition: if one or more repaired coordinates cannot be recovered in the
same exact interaction currency on both witnesses, the carried calibration set
is not theorem-ready.

### Step 3 - Prove Or Correct The Repaired Budget Vector

Depends on: Step 2.

Action: test whether lowering any coordinate of the repaired vector excludes at
least one carried friendly witness. Distinguish genuine coordinate sharpness
from joint necessity across the two-witness set.

Action: if the repaired vector is not actually minimal, replace it with the
strongest simpler corrected vector still supported by the same frozen witness
set.

Expected output: a minimality theorem or corrected admissibility sheet with
proof obligations stated explicitly.

Kill condition: if the repaired vector can only be justified by informal
"almost saturation" language rather than exact classwise inequalities, stop
and downgrade the object to calibration memo status.

### Step 4 - Pin One Honest Hostile Same-Currency Ledger

Depends on: Step 3.

Action: write one explicit hostile
`F_DT(delta, eta)`
ledger in the same classwise interaction currency and compute which repaired
coordinates are uniformly overloaded on that ledger.

Action: distinguish three outcomes cleanly:
uniform hostile overload proved,
hostile overload only on some coordinates,
or hostile comparison still unresolved because the record does not yet pin the
needed same-currency data.

Expected output: a hostile overload sheet that names exactly which coordinates
carry the obstruction and which do not.

Kill condition: if the hostile ledger requires invented
`F_DT`
numerics, broader packet semantics, or a changed spectator partition, keep the
theorem friendly-minimality only and record the hostile side as unresolved.

### Step 5 - Package The Obstruction-Facing Result

Depends on: Step 4.

Action: package the strongest honest endpoint:
either a classwise minimality theorem for the repaired leakage sheet with any
earned hostile overload clause,
or a sharp negative memo saying that the leakage sheet is still only a dossier
artifact rather than a theorem object.

Action: state explicitly whether the result supports only friendly
admissibility, friendly-plus-hostile separation, or neither.

Expected output: a theorem candidate or a bounded negative memo with one named
failure type.

Kill condition: if the final claim collapses back into
"exact NS has too much coupling"
without a proved classwise statement, the chain failed.
