# Exploration 001 Goal - Freeze The Step-3 Robustness Taxonomy

## Objective

Freeze one source-grounded transformation taxonomy for the Step-3 robustness
audit of the three promoted Step-2 candidates:
`Template-Defect Near-Closure`,
`Windowed Spectator-Leakage Budget`,
and
`Delayed-Threshold Itinerary`.

The taxonomy must separate:

- true exact symmetries
- fixed canonicalization choices
- substantively different modeling choices

and it must state which parts of the frozen packet / candidate / gate sheets
must be invariant under the first class, which dependencies are acceptable only
because they were frozen up front in Step 1 and Step 2, and which changes
should count as a different model rather than a harmless rewrite.

## Success Criteria

- Name the exact transformations that count as true exact symmetries for the
  canonical one-bridge role-labeled helical packet family with mandatory
  conjugate completion.
- Name the exact choices that belong only to canonicalization under the frozen
  policy:
  conjugate representative,
  helical basis,
  role-label ordering,
  amplitude normalization anchor,
  phase anchor or gauge,
  and sign-sheet bookkeeping.
- Name the exact moves that count as substantive model changes:
  packet regrouping,
  support-semantics changes,
  changing spectator partitions,
  threshold retuning,
  time-window changes,
  sign-sheet changes,
  or importing Tao's averaged freedoms by hand.
- State, candidate by candidate, which parts of the criterion, packet object,
  threshold language, and downstream gate must be symmetry-invariant versus
  merely fixed by canonicalization.
- Tag all major conclusions as `[VERIFIED]`, `[INFERRED]`, or `[PROPOSED]`.
- Name exact local files supporting every major taxonomy claim.

## Failure Criteria

- Mixes exact symmetries, canonicalization, and substantive model changes into
  one undifferentiated stability screen.
- Treats post hoc threshold retuning or packet-regrouping as cosmetic.
- Treats the fixed Step-1 / Step-2 sign / phase / normalization sheet as if it
  were optional later bookkeeping.
- Fails to say what the candidate gates must remain invariant under.

## Constraints

- Use only local repository materials.
- Do not run the full candidate verdict for Step 3. This exploration only
  freezes the taxonomy and the invariance requirements.
- Keep the packet object fixed:
  the canonical one-bridge role-labeled helical packet family with mandatory
  conjugate completion.
- Keep the shared five-channel interaction core fixed:
  `A -> B`,
  `A -> C`,
  `B,C -> C`,
  `C,A <-> D`,
  `D,D -> E`.
- Do not rescue a candidate by changing packet semantics, spectator partitions,
  thresholds, time windows, or Tao-like feature commitments.
