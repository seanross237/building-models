# Exploration 002 Goal - Build The Anti-Circuit Dossier

## Objective

Using the frozen family ledger and pass/fail sheet from `exploration-001`,
build the obstruction-facing Step-4 dossier for the anti-circuit families.

The dossier must test the three surviving candidates on fixed explicit families
covering:

- `Beltrami cancellation`
- `mirror-mode completion`
- `nearly degenerate triads`
- `tiny trigger modes`

## Required Output

Produce a report that does all of the following:

1. Records the exact interaction data used on each anti-circuit family:
   helical coefficient/sign ledger,
   desired-channel forcing,
   classwise spectator forcing,
   and event-time traces when relevant.
2. Applies the same predeclared admissibility filters and thresholds from
   `exploration-001` without retuning.
3. Reports, for each candidate on each anti-circuit family, whether the
   candidate passes, fails, or remains ambiguous.
4. Gives one named reason for each outcome tied to the packet dossier rather
   than to a slogan.
5. Separates obstruction-facing evidence from any merely degenerate or vacuous
   cases.

## Success Criteria

- The dossier yields a clean anti-circuit read rather than another general
  rhetoric memo about exact entanglement.
- At least one anti-circuit family distinguishes:
  genuine desired-channel collapse,
  forced spectator overload,
  and
  failure of delayed activation.
- Any ambiguity is named precisely, for example:
  missing exact coefficient ledger,
  vacuous zero-activity packet,
  or
  unresolved event-time trace.

## Failure Criteria

- The exploration fails if it changes thresholds or packet normalization after
  seeing one family's outcome.
- It fails if it treats a zero-activity Beltrami packet as a positive near-
  circuit witness rather than screening it through the predeclared
  admissibility filters.
- It fails if the report collapses the non-aggregated diagnostic vectors into a
  single score.
