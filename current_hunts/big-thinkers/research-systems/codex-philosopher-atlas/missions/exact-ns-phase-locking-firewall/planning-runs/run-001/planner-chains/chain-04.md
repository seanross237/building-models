# Chain 04 - Intrinsic Coherence Functional Audit

## Central premise

The mission may fail at the object-definition level: every plausible
phase/coherence observable might secretly depend on a hand-chosen desired
channel or packet trace. This chain tests that risk directly. It aims either to
promote one genuinely intrinsic exact phase/coherence functional or to kill the
mission branch with a sharp memo saying no such object survives canonicalization.

## Verifiable result target

Either:

- one intrinsic exact phase/coherence functional with a clear downstream theorem
  question; or
- a decision-grade memo showing that every candidate imports packet labels,
  desired channels, or frozen transfer annotations by hand.

## Prior context to carry forward

- Earlier work already showed that packet-level objects can be honest and
  useful without yet being intrinsic enough for theorem-facing use.
- The mission explicitly forbids a phase object that only makes sense after
  selecting a Tao-like desired subgraph by hand.
- Exact NS object audits must distinguish true symmetries, fixed
  canonicalization choices, and substantive model changes.

## Ordered steps

### Step 1 - Generate a minimal candidate family of intrinsic observables

- Depends on: none.
- Actions:
  - write at most three candidate object classes, for example:
    transfer-direction coherence,
    phase-window occupancy,
    transfer-weighted phase entropy,
    or a coherence-versus-spectator budget;
  - require each candidate to be definable on an exact support class without
    post hoc desired-channel labels.
- Expected output:
  - one candidate table with formulas, domains, and intended theorem use.
- Kill condition:
  - if no candidate can be written without immediate desired-channel annotation,
    stop with a definition-level failure.

### Step 2 - Run the intrinsicity and canonicalization screen

- Depends on: Step 1.
- Actions:
  - test each candidate against exact symmetries, gauge choices, conjugate
    completion, helical sign conventions, and frozen normalization rules;
  - separate harmless dependence on a declared canonical packet sheet from fatal
    dependence on arbitrary bookkeeping.
- Expected output:
  - an audit matrix labeling each candidate as:
    intrinsic,
    canonical-but-honest,
    or fatally non-intrinsic.
- Kill condition:
  - if every candidate falls into the last bucket, end the mission branch
    sharply rather than reopening packet audits.

### Step 3 - Derive the exact evolution or balance law for survivors

- Depends on: Step 2.
- Actions:
  - for each surviving candidate, derive the exact balance/evolution law from
    the Fourier/helical interaction ledger;
  - identify which exact terms could support either an obstruction theorem or a
    construction test.
- Expected output:
  - one evolution ledger per survivor,
  - one note stating the live downstream question each survivor supports.
- Kill condition:
  - if a survivor has no exact evolution law or only reproduces static packet
    bookkeeping, discard it.

### Step 4 - Audit survivors on inherited friendly and hostile fixtures

- Depends on: Step 3.
- Actions:
  - evaluate the surviving objects on inherited fixtures such as `F_SL`,
    `F_SS`, and `F_DT`, but only as normalized audit cases;
  - check whether the object sees anything phase-specific rather than simply
    rediscovering template defect or leakage under a new name.
- Expected output:
  - a fixture audit table,
  - one note identifying whether the object adds genuine new information.
- Kill condition:
  - if the object cannot distinguish itself from prior packet gates, it is not
    the mission's desired phase/coherence object.

### Step 5 - Promote a survivor or close the branch

- Depends on: Step 4.
- Actions:
  - if one survivor remains, package one exact downstream theorem or
    counterexample test for it;
  - otherwise write the negative result precisely:
    no intrinsic phase object survives without hidden transfer annotation.
- Expected output:
  - either a promoted object memo,
  - or a bounded negative mission memo.
- Kill condition:
  - if the final writeup drifts into general "phase mixing" rhetoric with no
    exact formula, the chain fails.

## Why this chain is meaningfully different

This is the only chain whose honest success may be a sharp definition-level
negative. It is not trying to prove the firewall directly or build a
counterexample. It exists to stop the run from spending serious budget on phase
language that never becomes an intrinsic exact NS object.
