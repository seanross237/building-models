# Selector Verdict - exact-ns-phase-locking-firewall / run-001

## Selected Portfolio

### 1. Chain 02 - Small Exact Phase-Locked Counterexample

Selected.

This is the strongest first inclusion because it is the only direct kill route
in the set. A concrete exact phase-locked subsystem that survives honest
conjugate and spectator accounting would falsify the firewall immediately.

It also fails usefully. Even if the construction collapses, it can still return
a precise obstruction memo naming whether the blocker was coefficient rigidity,
spectator takeover, lack of an honest delayed window, or instability of the
locking manifold.

### 2. Chain 03 - Finite Phase-Frustration Compatibility Theorem

Selected.

This is the best theorem-facing complement to Chain 02 under a three-slot
budget. It attacks the problem at the finite compatibility level rather than
through long-time dynamics, so it has a cleaner adversarial readout: either the
required phase constraints are already inconsistent, or there is a surviving
compatible assignment worth handing directly to the construction branch.

Its failure value is strong because even a non-obstructive outcome still
produces a sharp finite phase sheet instead of a vague negative.

### 3. Chain 04 - Intrinsic Coherence Functional Audit

Selected.

This is the most differentiated guardrail in the portfolio. It directly tests
whether any intrinsic phase/coherence object exists without smuggling in packet
labels or desired-channel annotations by hand.

That makes it especially valuable in a three-chain budget: even failure is a
decision-grade result, because it can close the entire phase-functional branch
before more expensive theorem work accumulates on a non-intrinsic object.

## Full Ranking

1. Chain 02 - Small Exact Phase-Locked Counterexample
2. Chain 03 - Finite Phase-Frustration Compatibility Theorem
3. Chain 04 - Intrinsic Coherence Functional Audit
4. Chain 01 - Dynamic Phase-Decoherence Tax On Delayed Transfer

## Why This Is The Strongest Three-Chain Portfolio

- It maximizes diversity across constructive falsification, finite algebraic
  obstruction, and definition-level audit.
- It preserves a high-upside path by keeping Chain 02 live while still
  retaining a theorem-facing obstruction route in Chain 03.
- Every selected chain yields useful failure output:
  Chain 02 can return a named obstruction atlas,
  Chain 03 can return either a finite incompatibility theorem or a compatible
  assignment for construction,
  and Chain 04 can return a sharp intrinsicity kill memo.
- It rejects the main redundancy cleanly. Chain 01 is the best excluded branch,
  but under a three-slot budget it overlaps with Chain 03 on phase-obstruction
  logic and with Chain 04 on the need for a genuinely intrinsic phase object.

## Why Chain 01 Was Not Selected

Chain 01 is not a weak branch. It is the strongest excluded chain because it
aims directly at the desired dynamical obstruction theorem. But it loses on
marginal portfolio value in this selector pass.

- It is the most operationally diffuse route in the set, since it needs both a
  new intrinsic defect object and a time-integrated evolution inequality.
- Its first step is entangled with the same intrinsicity risk that Chain 04
  tests more cleanly and cheaply.
- Relative to Chain 03, it offers a narrower failure cone: a failed dynamical
  tax is less reusable than a solved finite compatibility problem or an
  explicit compatible assignment.

If Chain 03 later finds a compatible finite phase sheet but Chain 02 still
cannot realize a stable delayed window, Chain 01 is the natural reserve branch
to reactivate.
