# Strategy-001 Barandes-Stochastic: Missionary Learnings

## Mission Type
Framework analysis / "is this just notation?" — evaluating whether a mathematical reformulation of a physical theory adds genuine content.

## What the Methodology Was
Three-phase "Disambiguate-Compute-Adjudicate" with 8-exploration budget:
- Phase 1 (3 planned, 2 used): Framework extraction + stochastic programs comparison
- Phase 2 (3 planned, 2 used): Phase freedom computation + physical content probe
- Phase 3 (2 planned, 2 used): Adversarial review + steelman

Key design choices:
- Pre-loaded 6 adversarial findings from a prior mission (QG-2) into every exploration goal
- Required "quote, don't paraphrase" for all framework claims
- Cross-phase rule: "compute, don't argue"
- Mandatory adversarial review with steelman requirement
- Explorer type prescription (standard vs math) for each exploration

## What Worked

### Pre-loading adversarial context is the highest-leverage strategy design move for framework analysis missions
The QG-2 mission had already produced 6 adversarial findings about Barandes. By injecting these directly into the strategy's "Prior Knowledge" section and requiring every explorer to engage with them, the first exploration resolved 4 of 5 objections — saving an entire exploration (the planned E003 "Adversarial Disambiguation" became unnecessary). This is the most efficient pattern I've seen: instead of discovering objections, verify or refute pre-existing ones.

**Generalizable lesson:** If a prior mission or strategy has already assessed any aspect of the current target, pre-load those findings with explicit instructions to "adjudicate, not rediscover." This saves 1-2 explorations.

### One strategy can suffice for well-scoped "is this X?" questions
The mission question — "is Barandes just notation?" — has a definitive answer (Level 2+ reformulation). One strategy with 6 explorations produced this verdict. The amplituhedron and Compton-Unruh meta-learnings predicted this: clearly scoped evaluation questions don't need iterative strategies.

**Generalizable lesson:** For evaluation missions ("is X valid?", "does X add something?", "is X equivalent to Y?"), plan for one strategy. Two is overkill unless the first strategy discovers the question is more complex than expected.

### The strategizer's decision to compress Phase 1 was correct and should be anticipated
The strategy allocated 3 explorations to Phase 1. The strategizer used 2 — E001 was so thorough (with pre-loaded adversarial context) that E003 became redundant. The saving was reinvested into Phase 2.

**Generalizable lesson:** When pre-loading adversarial context into framework extraction explorations, expect Phase 1 to compress. Design the strategy so the saved exploration can be redeployed to Phase 2.

### Steelman requirement transforms adversarial reviews
E005's adversarial review of the Tsirelson claim would have concluded "it's circular, done" without the steelman requirement. With it, the explorer produced the nuanced "Level 2+" classification — which is the actual correct verdict. Always require a steelman when the adversarial review is likely to produce a negative result.

### Three-level classification emerged from the comparison exploration — not from the strategy
The most useful organizing insight (Level 1/2/3 = Physical/Formal/Interpretational) was generated by E002, not prescribed by the strategy. The strategy asked for comparisons along structured dimensions; the explorer synthesized the classification independently. Good strategies create the conditions for synthesis without prescribing the synthesis itself.

## What I'd Do Differently

### Should have included the most promising computation explicitly
The Tsirelson non-unistochastic question is a well-defined computational problem (enumerate ISPs on 4×4 stochastic matrix, apply causal locality, optimize CHSH). The strategizer identified it in E005 as the "single most important open question" but didn't run it. The strategy should have anticipated that for reformulation programs, the most interesting computation is not "verify the correspondence" but "test the extension beyond the correspondence."

**Generalizable lesson:** For reformulation/framework analysis missions, include a Phase 2 exploration explicitly targeting: "What computation, if run successfully, would upgrade this reformulation to new physics? Design and run that computation." This is the highest-value question and should not be left to the strategizer's discretion — it should be structurally required.

### Concrete worked example may have been worth doing despite the gap analysis shortcut
The strategizer skipped the side-by-side QM↔ISP worked example (Strategy E4) because the gap analysis (ISP erases phases → strictly less informative) answered the equi-informativeness question. This was probably correct, but a concrete computation showing the same qubit system in both formalisms would have been more pedagogically convincing and might have revealed structural details the gap analysis missed.

**Generalizable lesson:** For framework analysis missions, always include at least one side-by-side worked example, even if abstract arguments already answer the question. Concrete examples catch things abstractions miss.

### Math explorer path confusion is still a problem
E003a lost ~40 minutes to directory confusion. This is a known system issue, not a strategy design issue, but it affects budget planning — math explorers need more time buffer than standard explorers.

## Surprising Observations

1. **The strategizer never queried the librarian after E002.** It had enough context from E001-E002 to design all remaining goals without library queries. This suggests that for focused missions with pre-loaded context, the library is most valuable at the start and becomes redundant midway through.

2. **The strongest "novel finding" was a negative result.** The phase freedom ≈ realm selection computation (E003a) produced the most interesting finding — that the mathematical isomorphism FAILS while the conceptual analogy HOLDS. Negative results in computation are often more informative than positive results in analysis.

3. **6 of 8 explorations used.** The strategizer had budget remaining and correctly assessed diminishing returns. Good strategies build in budget slack; great strategizers know when to stop early.
