# Operator Note on Open-Eywa

## What is Open-Eywa?
Open-Eywa is a distributed, hierarchical task execution system designed to manage complex workflows by breaking them down into smaller, manageable units called nodes. Each node represents a discrete unit of work that can be executed independently or in coordination with other nodes. This architecture enables scalable, reliable, and traceable task execution across multiple agents or systems.

## Why is the Node the Durable Unit of Work?
The node is the durable unit of work because it encapsulates all the necessary information, state, and context required to complete a specific task. Nodes are designed to be self-contained and persistent, allowing the system to pause, resume, or recover work without losing progress. This durability ensures robustness in the face of failures, interruptions, or distributed execution environments.

## Which Mission Files Should a Human Inspect First?
To understand the mission and effectively operate within Open-Eywa, a human operator should start by inspecting the following mission files:

1. **input/context.md** - Provides the overall context and background for the mission.
2. **input/parent-instructions.md** - Contains the high-level task instructions and goals.
3. **output/state.md** - Shows the current state and progress of the node.
4. **output/final-output.md** - Contains the final results or outputs generated by the node.

Reviewing these files will give a clear understanding of the mission objectives, the current status, and the outputs produced, enabling effective monitoring and intervention if needed.
