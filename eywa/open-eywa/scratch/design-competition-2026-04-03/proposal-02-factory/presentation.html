2. Eywa

What is Eywa?

Task hopper

One task enters. The root node decides the first move.
→

Same chassis

Every node shares the same DNA. Only instructions and variables change.
→

Assembly choices

Solve, plan, delegate, or split into child nodes.
→

Run bin

Every node writes a full record back to the factory floor.
Stable part
Node DNA
Prompt chassis, tool policy, logging, and child creation rules.
Variable part
Instructions
Goal framing and extra context supplied to the node.
Variable part
Parameters
The knobs Bonsai learns to tune over time.
3. Bonsai

How does Bonsai choose variables?

Run history

Scores, time, token use, cost, fanout, and task tags.
→

Match similar jobs

Use the task fingerprint to find past runs that look alike.
→

Pick variables

Choose the next variable set from averages, heuristics, or learned selection.
→

Retest

Run again and compare the score against the benchmark yard.
domain task_shape verification_mode ambiguity_level decomposition_need
The optimizer only works if every run is labeled well enough to find the right lookalike later.
4. Potter

How does Potter refactor safely?

Branch worktree

Copy the code into a separate sandbox before changing anything.
→

Rewrite the gears

Make the code simpler, smaller, and more modular.
→

Dry run

No model call should change the observable tree or artifact outputs.
→

Equivalence gate

If the dry-run result changes, reject the refactor.
Allowed change
Structure
Improve the code path without changing system behavior.
Forbidden change
Outputs
Dry-run and benchmark-visible behavior must stay identical.
Side score
Quality
Clarity, flexibility, and robustness are judged after equivalence.
5. Contracts

What data must every run record?

TaskSpec
Task in one place
Raw task, normalized task, tags, constraints, and provenance.
NodeSpec
Node in one shape
Parent link, depth, instructions, variables, and handoff mode.
RunRecord
Run in one log
Prompt ref, model, tools, output, score, time, tokens, and cost.
BenchmarkCase
Case in one yard
Gold output, scoring rule, validation mode, and stability flag.
DryRunCase
Equivalence proof
Expected control flow, artifact hash, and score. This is what keeps Potter honest.
6. Open questions

What still needs a decision?

Tag set
Which five tags are canonical?
The system needs a stable task fingerprint before learning can work well.
Equivalence
What counts as the same?
Dry-run proof may need structural checks, artifact hashes, or both.
Promotion
When does a run become a benchmark?
The benchmark yard should stay clean, stable, and easy to score.
If these three decisions are tight, the rest of the factory can stay simple.
