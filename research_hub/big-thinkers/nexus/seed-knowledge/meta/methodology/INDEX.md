- **divergent-survey-pattern.md** — The four-part pattern (survey + deep dive + synthesis + novelty assessment) for Phase 1 landscape mapping.
- **devils-advocate-after-construction.md** — Attack immediately after building; give the attacker the full document, not a summary.
- **predict-attack-synthesize-sequence.md** — The proven strategy cycle: extract predictions, then attack, then synthesize into final document.
- **adversarial-explorations-need-null-hypothesis.md** — Every devil's advocate must test whether the null hypothesis (no framework needed) explains the same observations.
- **repair-pattern.md** — One repair per exploration, structured as: what changed, what survived, what's fixed, what's broken, before/after scorecard.
- **benchmark-comparison-early.md** — Compare new frameworks to established benchmarks early to reveal the gap before over-investing.
- **verification-catches-library-errors.md** — Verification explorations don't just confirm; they catch wrong premises and misleading claims in the knowledge base.
- **comparison-exploration-pattern.md** — When two approaches seem related, a direct comparison exploration with a multi-dimensional framework and forced verdict ("same, complementary, or distinct") is highly productive. *Updated: added Verlinde vs. T_U/T_dS comparison (E005) — detailed comparison revealed RAR failure and deep-MOND-only limitation not visible in the initial survey.*
- **math-explorer-dimensional-analysis.md** — Math Explorer + dimensional analysis is an excellent combination for "is this dimensionally possible?" questions; produces decisive quantitative verdicts with plots.
- **decisive-negative-pivot.md** — When dimensional analysis produces discrepancies of >10 orders of magnitude, it's a hard kill; pivot immediately to what CAN work at the right scale.
- **scale-spread-in-numerical-surveys.md** — In numerical survey explorations, the spread of scales (Planck to cosmological) matters more than many systems at similar scales; reveals where bounds are tight vs. trivial.
- **gap-finding-in-existing-programs.md** — Identifying open problems and gaps in existing research programs is as valuable as inventing new theories; the "what's already been computed" comparison table format makes gaps immediately visible.
- **multi-ansatz-sweep-pattern.md** — When a problem has 2–4 mathematically similar candidate ansatze, test all of them in one exploration; rules out N-1 candidates in a single pass and produces a diagnostic comparison table. *Updated: Dense Fine Sweep variant (RH s002-E005): for non-monotone parameter behavior, add a fine grid around the suspected extremum — coarse sweep missed the true maximum (p=809 vs p=997) and underestimated neighbor fluctuations (±0.4).*
- **dual-mechanism-robustness.md** — When two independent physical mechanisms produce the same phenomenological result (via different assumptions and mathematics), that is the strongest form of robustness evidence; worth a dedicated confirmation exploration.
- **sequence-computation-approaches.md** — When the goal is to compute something by two methods, sequence them: verify Method 1 against an independent baseline first, then compare Method 2 against the verified result. Simultaneous attempts can't cross-validate each other.
- **standard-explorer-for-literature-surveys.md** — Use standard explorers (not math explorers) for literature surveys: ~20 min vs. 50+ min, equivalent depth. Reserve math explorers for computation, dimensional analysis, code execution.
- **investigate-why-on-discrepancies.md** — When results disagree with theoretical predictions, investigate WHY rather than just reporting the discrepancy. The explanation is often the most interesting finding (e.g., SED O(β) vs O(β²) gap → Langevin vs ALD distinction became the central result).
- **adversarial-check-between-phases.md** — Run a lightweight adversarial check after Phase 2 (before Phase 3 computation begins); early reframing is more valuable than corrections after major compute is committed. *Updated (E008): upgraded to full standard-test adversarial exploration early — the Compton-Unruh Bullet Cluster failure (decisive lensing falsification) was not caught until E008 but should have been run at E003–E004; for any model in a well-studied domain, run the canonical checklist of standard tests before Phase 3.*
- **check-algebra-before-multi-approach.md** — Before designing multiple exploration approaches to derive a "missing factor" from a ratio formula, pre-screen algebraically: if both numerator and denominator share the same structural factor (e.g., 2π), that factor cancels identically and no internal mechanism can restore it. Don't send 4 approaches when algebra predicts failure — spend the time on the one external lead that might work.
- **sign-flip-diagnostic-for-critical-thresholds.md** — NEW (SED E007): When two variants bracket a qualitative threshold (one undershoots, one overshoots), the sign reversal locates the critical parameter via linear interpolation. n=2 (negative Δe) + n=3 (positive Δe) → n*≈2.61. Reusable: design variants to straddle the expected threshold; treat the crossover value as the primary finding.
- **staged-computation-goals.md** — NEW (amplituhedron E006): Structure math explorer computation goals as 4-stage sequential pipelines where each stage verifies before the next builds on it. Enables independent debuggability, physics sanity checks per stage, and emergent insights not in the original goal.
- **parallel-math-explorer-explorations.md** — NEW (amplituhedron E005+E006): When two math explorer computations are logically independent, run them simultaneously. Each takes ~20 min; parallel execution gives ~2× wall-clock saving with no quality cost.
- **verify-unexpected-findings-immediately.md** — NEW (yang-mills s002 E005-006): When a computation produces an unexpected major finding, design an immediate stress-test exploration targeting the most plausible failure mode (different dimension, adversarial configs). Include a direct comparison table; treat outcome as decisive. E006 verified E005's 45× Hessian slack in 4D (138× confirmed) and adversarially (≥176× across all search strategies).
- **work-backward-from-constraint.md** — NEW (thermal-time strategy-001 meta-exploration-002): For constructive explorations, frame the task as "what theory/value satisfies this constraint?" rather than forward-computing predictions. Produces definitive answers anchored to constraints and unexpected byproducts (e.g., no-go theorems, uniqueness conditions). Contrast: forward framing generates correct calculations that may not resolve the key question; backward framing forces a definitive derivation. Structure: (1) identify the constraint, (2) derive what satisfies it, (3) check consistency, (4) allow unexpected results.
- **include-trivial-control-checks.md** — NEW (thermal-time meta-003): Every computation exploration should include one "should equal X by construction" check (different code path, limit case, or symmetry). The thermal-time E003 control check C_global = C_full (at machine zero for all λ) confirmed that the 82.7% discrepancy was physical, not a bug. Without the check, a large unexpected result is always suspect. Cost: near zero. Payoff: immediate trust or bug detection.
- **structural-vs-quantitative-discrepancy.md** — NEW (thermal-time meta-003): When two predictions disagree, diagnose whether the discrepancy is structural (different signal topology — wrong number of peaks, different symmetry class) or quantitative (scale factor, coefficient). Parameter tuning cannot fix structural discrepancies — they require a different model. In E003, C_full had two FFT peaks and C_local had one — no value of ω_eff can make a single sinusoid match a beating signal. Ask in goals: "Is the discrepancy structural or quantitative?"
- **require-trend-tabulation-for-negative-results.md** — NEW (RH s003-meta-004): When testing a scaling law or universality claim, require a table of the metric at each parameter value + direct test for monotone trend. Makes negative results decisive: "β_max = 1.32, 1.15, 1.02 (decreasing)" is a clean refutation; "results vary" is not.
- **split-search-from-synthesis.md** — NEW (RH s002-meta-008): For explorations requiring both iterative literature searches AND structured writeup, split into two: (1) search exploration (produce raw dump, write as you go), (2) synthesis exploration (read dump, write structured report, no new searches). Proposed architectural fix for the research-buffering stall pattern (3 consecutive failures: RH E006-E008). Not yet confirmed by successful split execution.
- **useful-failure-extracts-secondary-results.md** — NEW (YM s003-meta-s003-007): When an exploration's main goal fails (e.g., prove X when X is false), the exploration often produces valuable secondary results: algebraic invariants (B_□ B_□^T = 4I₃), analytical proofs for special cases (Δ = 14(cosε−1) ≤ 0), structural constants (Haar average E[M(Q)] = 6I). Design goals to capture secondary output. Complements decisive-negative-pivot and allow-explorer-synthesis.
- **gradient-ascent-on-projected-quantity.md** — NEW (YM s003-meta-006): When testing whether a bound holds on a specific eigenspace, run gradient ascent on the projected quantity (e.g., λ_max(P^T R P)) rather than the full matrix. More informative (shows margin from violation), computationally cheaper, and tests the correct target. YM s003-E006: gradient ascent on 9×9 P^T R P stayed at −8 to −11 (margin 8–11 from 0), cleaner than full 192×192 λ_max which gave margin 1.6.
- **lattice-convergence-fixed-frequency.md** — NEW (thermal-time s002-meta-001): For lattice QFT convergence tests, use fixed physical frequency (not fixed mode index) across N values; mode-0 convergence gives IR artifacts (omega_0 -> 0 as N -> infinity). Multi-angle strategy: test both fixed-mode and fixed-frequency to distinguish genuine physics from artifacts. Thermal-time E003: mode-0 showed fake convergence N^{-0.46}, fixed-freq showed true divergence N^{+0.33}.
- **verify-identity-generalization-before-extending.md** — NEW (yang-mills-conjecture meta-E008): When extending a proof to a larger space, FIRST check whether the key identities (trace identity, budget identity, etc.) still hold. The E006 staggered proof relied on a trace identity (c + Tr(P) = 64) that fails for general spatial patterns (varies by factor ~3). A quick test at the outset would have ruled out the approach immediately.
- **consider-sdp-sos-for-constrained-proofs.md** — NEW (yang-mills-conjecture meta-E008): For matrix-inequality proofs over constrained spaces (λ_max(M|_V) ≤ c), consider SDP/SOS formulations from the start. The constraint structure (essential for the bound to hold) is naturally handled by SDP relaxations, while algebraic approaches struggle to exploit constraints. E008: unconstrained eigenvalue reaches ~21 but bound is 16 on constrained space; enormous numerical margin (best adversarial: 15.997) suggests certificate exists. *Updated (S002-E003): tight-bound variant — when infimum = 0, loose inequalities (Cauchy-Schwarz, AM-GM) are ruled out; prefer SDP/SOS or exact algebraic decomposition.*
- **budget-adversarial-for-high-dim.md** — NEW (yang-mills-conjecture S002-meta-E003): Budget adversarial optimization from the start for verification in high-dimensional parameter spaces (≥20D). Random uniform sampling has poor coverage: 200K random tests on SO(3)^10 (30D) missed a counterexample with min eigenvalue = -2.13 that 20 adversarial runs found immediately. Always pair random testing with adversarial optimization (≥10 independent starts). *Updated (S003-E004): domain-specific structured starts variant — include physics-motivated extremal configuration classes (e.g., anti-instantons for Hessian eigenvalue bounds) as GD seeds; anti-instanton starts found 63% more negative eigenvalue than random GD.*
- **special-case-proof-guides-generalization.md** — NEW (yang-mills-conjecture S002-meta-E004): When a proof works for a special case, analyze WHY (the structural property that made it easy), then frame the general case as "restore that property + bound the deviation." E004: sum_S proof on rotation axes worked because u=v (bilinear → quadratic); deviation u−v = −(I−R^T)T has same structure as baseline f(R,T) terms, suggesting the natural generalization route.
- **check-affine-structure-before-bounding.md** — NEW (yang-mills-conjecture S002-meta-E005): Before attempting bounds, check whether the quantity is linear/affine in some parameters. Affine dependence enables per-parameter independent minimization with closed-form bounds. E005: M9 affine in D (verified 3.5e-15) decomposed 18D minimization into 6 independent per-pair Cauchy-Schwarz problems, unlocking the sum_S ≥ 0 proof after 7 prior approaches failed.
- **verify-counterexample-before-investigating.md** — NEW (yang-mills-conjecture S002-meta-E007): When finding a potential counterexample, the FIRST priority is verifying it against the exact problem statement — not investigating consequences. Check that the mathematical object being computed matches the conjecture's definition (representation, convention, normalization). A false counterexample can derail strategy. Protocol: stop → re-read exact statement → verify computation matches → resolve ambiguity → only then investigate.
- **decompose-correction-into-psd-and-indefinite.md** — NEW (yang-mills-validation S002-meta-E002): When analyzing a matrix inequality C = B − A ≥ 0, decompose C into PSD + indefinite components (C = C₊ + C₋). Reveals the mechanism by which the inequality holds (or fails), quantifies margins, and guides proof strategy. Protocol: identify sign-definite components, verify decomposition, check at relevant eigenspace, use ratio C₊/|C₋| to assess margin.
