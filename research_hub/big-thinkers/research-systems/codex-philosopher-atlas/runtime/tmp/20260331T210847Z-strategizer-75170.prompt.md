## Library Queries

Use the receptionist synchronously via `bin/run-role.sh`.

Create a small task file under `$CODEX_PATLAS_ROOT/runtime/tasks/` that:

- states the question you need answered
- points the receptionist at the local factual library root
- points it at the local meta library root
- points it at the mission meta-inbox if relevant
- tells it where to append its log

Then run:

```bash
"$CODEX_PATLAS_ROOT/bin/run-role.sh" \
  --role receptionist \
  --workdir "$CODEX_PATLAS_ROOT/library" \
  --task-file "$TASK_FILE"
```

Read the receptionist result before designing the next exploration.

## Exploration Cycle

For each exploration:

1. Decide the exact sub-task.
2. Append your reasoning to `REASONING.md`.
3. Create `explorations/exploration-NNN/GOAL.md` with:
   - the specific objective
   - success and failure criteria
   - the relevant library context
   - any constraints or references the explorer needs
4. Choose the explorer type:
   - `explorer` for literature, conceptual synthesis, and comparison
   - `math-explorer` for computation, proof attempts, or formal verification
5. Launch the explorer through `bin/launch-role.sh`.
6. Wait on the sentinel file `REPORT-SUMMARY.md`.
7. Read the summary and then the full report as needed.
8. Append the summary to `HISTORY-OF-REPORT-SUMMARIES.md`.
9. Copy the report into `../../library-inbox/` with a descriptive name.
10. Write a meta-learning note into `../../meta-inbox/`.
11. Launch the curator through `bin/launch-role.sh`.
12. Update `state.json`.
13. Continue or complete the step.

## Launching Explorers

Use explicit session names with the `codex-patlas-` prefix and set the
sentinel file to the exploration summary:

```bash
"$CODEX_PATLAS_ROOT/bin/launch-role.sh" \
  --role explorer \
  --workdir "$EXPLORATION_DIR" \
  --task-file "$TASK_FILE" \
  --session-name "codex-patlas-explorer-001" \
  --sentinel-file "$EXPLORATION_DIR/REPORT-SUMMARY.md"
```

For the math explorer, change `--role math-explorer`.

Parallelism is encouraged when sub-tasks are independent.
