# Windowed Spectator-Leakage Budget Compares Off-Template Mass To The Desired Five Channels Class By Class

Status: `INFERRED`

The second honest Step-2 candidate notion is `Windowed Spectator-Leakage
Budget`.

It compares the integrated magnitude of all forced off-template channels
against the integrated magnitude of the desired five channels, class by class,
on one declared finite activation window, using one hard admissibility sheet
`(L_tot, L_mirror, L_companion, L_feedback, L_cross)`.

This turns `near-isolation` into a branch-auditable object on the exact packet
ledger instead of leaving leakage as a generic concern with no declared
comparison rule.

At Step 3, the honest invariant is whether the coherently transported exact
packet passes the same leakage sheet in the same interaction currency. That
sheet is meaningful only relative to the frozen sign/amplitude/phase and
conjugate-completion conventions, the fixed spectator partition, and the fixed
`D_on`/`D_off` split.

Changing the spectator partition, changing the definition of `D_on` or
`D_off`, changing `Lambda_tot` or the class budgets, changing `T_act`,
changing support semantics, regrouping the packet, or recoding the same packet
under a different sign sheet defines a different model rather than a rewrite.

The completed Step-3 audit therefore classifies
`Windowed Spectator-Leakage Budget` as `stable after canonicalization`.
With the frozen spectator partition, normalization, interaction currency, and
window intact, `(L_tot, L_mirror, L_companion, L_feedback, L_cross)` remains
the cleanest obstruction-facing gate of the three promoted candidates.

The completed Step-5 repair shortlist sharpens this same object to the
admissibility sheet
`L_tot <= 1/4`,
`L_mirror <= 1/12`,
`L_companion <= 1/12`,
`L_feedback <= 1/16`,
and
`L_cross <= 1/24`.
On that repaired sheet the friendly witnesses
`F_SS(1/12)`
and
`F_SL(1/16)`
still pass, while the hostile `F_DT(delta, eta)` family remains a named
failure by classwise leakage overload on the same frozen ledger.

For the downstream Step-6 object freeze, the carried-forward friendly stress
set is
`F_SS(1/12)`
together with
`F_SL(1/16)`,
because those two families jointly saturate the repaired classwise budgets
more sharply than any single recorded friendly family.

At the completed Step-6 freeze this same repaired object is promoted
unchanged rather than narrowed.
The Step-5 repair only tightened the already frozen admissibility sheet; it
did not change the packet semantics, the fixed window `I = [0, 1]`, the
spectator partition, the `D_on` / `D_off` split, or the interaction
currency.

The strongest currently earned claim therefore stays local to the audited
packet ledger:
on that frozen sheet, this repaired classwise leakage gate separates the
friendly stress set `F_SS(1/12)` plus `F_SL(1/16)` from the hostile
comparator `F_DT(delta, eta)`.
The local record does not yet justify a theorem that passing
`G_leak`
forces template closeness, forces the delayed-threshold itinerary, or proves
a global no-near-circuit theorem beyond the audited packet class and carried
stress set.
