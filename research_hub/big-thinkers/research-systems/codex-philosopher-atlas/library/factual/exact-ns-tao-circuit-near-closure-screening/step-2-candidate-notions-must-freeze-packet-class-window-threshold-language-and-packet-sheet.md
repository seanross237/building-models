# Step-2 Candidate Notions Must Freeze Packet Class Window Threshold Language And Packet Sheet

Status: `INFERRED` admission rule with `VERIFIED` packet-sheet dependency

A Step-2 near-closure candidate becomes real only when it names:

- one exact packet class;
- one finite activation window;
- one threshold language;
- and one canonical sign / amplitude / phase sheet.

The same candidate definition should also place its interaction template and
its downstream gate in that same packet and threshold language. Otherwise the
branch is still moving between incompatible ledgers.

Without that freeze, the branch still has a slogan rather than a candidate
object that can be compared on exact projected channel data.
