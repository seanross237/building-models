# Template-Defect Near-Closure Compares One Canonical Packet To One Fixed Five-Channel Tao Template On One Window

Status: `INFERRED`

The first honest Step-2 candidate notion on the current exact-NS Tao-circuit
branch is `Template-Defect Near-Closure`.

It compares the exact role-projected Navier-Stokes vector field generated by
one canonical role-labeled helical packet against one fixed five-channel
Tao-like template on one declared finite activation window, using one
invariant template-defect observable `(Delta_tmpl, Delta_spec)`.

That makes the object auditable in the branch's own language: one packet
class, one role projection, one fixed template, and one finite window, rather
than a slogan about generic Tao-likeness.

At Step 3, the honest invariant is whether the coherently transported exact
packet still passes the same template-defect and spectator-defect tolerances.
Dependence on the frozen helical sign sheet, amplitude anchor, phase anchor,
conjugate representative, helical basis, and role order is acceptable only
because those choices were fixed up front.

Changing the role hierarchy sheet `(h_seed, h_clk, h_amp, h_rot, h_next)`,
changing `lambda_tmpl` or `lambda_spec`, changing `T_act`, regrouping the
packet, changing support semantics, swapping sign sheets, or importing
Tao-style averaged gate freedoms defines a different model rather than a
rewrite.

The completed Step-3 audit therefore classifies `Template-Defect Near-Closure`
as `stable after canonicalization`.
With the frozen packet sheet, template hierarchy, activation window, and
defect tolerances held fixed, `(Delta_tmpl, Delta_spec)` remains a meaningful
Step-4 screen on explicit pro-circuit and anti-circuit families.

The completed Step-5 repair shortlist sharpens this same object to the
admissibility sheet
`Delta_tmpl <= 1/4`
and
`Delta_spec <= 49/256`.
On that repaired sheet the recorded friendly witnesses
`F_SS(1/12)`
and
`F_SL(1/16)`
still pass, while the hostile `F_DT(delta, eta)` family remains a named
failure by late-rotor / next-shell collapse on the same frozen ledger.

For the downstream Step-6 object freeze,
`F_SL(1/16)`
is the carried-forward friendly stress witness because it is the worst
recorded friendly packet on both repaired defect coordinates.

At the completed Step-6 freeze this same repaired object is promoted
unchanged rather than narrowed.
The Step-5 repair only removed unused slack; it did not change the packet
class, the fixed window `I = [0, 1]`, the canonical packet sheet, or the
observable `(Delta_tmpl, Delta_spec)`.

The strongest currently earned claim therefore stays local to the audited
packet ledger:
on that frozen canonical sheet, this repaired role-template gate separates
the carried friendly witness `F_SL(1/16)` from the hostile comparator
`F_DT(delta, eta)`.
The local record does not yet justify a theorem that small
`(Delta_tmpl, Delta_spec)` forces the leakage budgets, forces the
delayed-threshold itinerary, or defines a global near-circuit notion beyond
the audited packet class.
