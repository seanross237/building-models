# Packet Audits Must Separate True Symmetries Canonicalization And Substantive Model Changes

Status: `VERIFIED` need for a separate canonicalization tier with `INFERRED`
branch classifications

Exact packet audits, and adjacent support-sheet audits derived from packet
templates, should use three separate layers instead of treating every
representation move as cosmetic:

- true exact symmetries already justified for the chosen packet presentation
- fixed canonicalization choices such as conjugate representatives, helical
  basis, role-label order, amplitude normalization, and phase anchor
- substantive model changes such as packet regrouping, support-semantics
  changes, optional conjugate completion, relaxing recursive closure, pruning
  closure-forced spectators, threshold retuning, time-window changes, or a
  different helical sign sheet

This separation keeps later robustness checks honest. A later candidate may
vary genuine invariances and canonical representatives, but changes in packet
grouping, support object, sign sheet, spectator policy, closure rule, or
convenience thresholds should be counted as different packet models rather than
as harmless rewrites.
