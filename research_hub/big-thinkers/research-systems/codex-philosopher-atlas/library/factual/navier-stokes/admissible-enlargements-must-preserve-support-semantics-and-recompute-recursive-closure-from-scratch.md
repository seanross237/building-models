# Admissible Enlargements Must Preserve Support Semantics And Recompute Recursive Closure From Scratch

Status: `VERIFIED` need for admissible enlargements with `INFERRED`
branch-preservation rule

An admissible enlargement keeps the same exact object type, helical basis,
conjugate-completion rule, recursive closure convention, and support-level
identity. The first enlargement move is to add one new independent helical
representative to the seed ledger and then recompute the full recursive
closure from scratch; only after that class is exhausted should the shell-span
budget widen.

Changing the helical sign sheet, changing basis, switching back to
packet-grouped identity, relaxing recursive closure, or redefining support by
desired channels or Tao roles is not an enlargement inside the same search
class. Those moves change the model being searched.
