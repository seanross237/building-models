# Exact Support Search Object Is A Finite Canonical Helical Support Sheet With Mandatory Conjugate Completion

Status: `INFERRED` Step-2 object freeze with `VERIFIED` branch reset away from
packet identity

For this exact-support branch, the primary search object is not a role-labeled
packet and not bare Fourier support. It is a finite set of canonical helical
representatives on one frozen helical basis and one frozen
conjugate-representative convention, together with the mandatory conjugate
completion needed to realize the exact real-valued support sheet.

That means two candidates with the same wavevector set but different helical
sign sheets are different exact objects. Packet families may still be used as
fixtures or seeds after translation into explicit helical support ledgers, but
they should not override support-level identity unless the branch writes an
exact invariant packet representation explicitly.
