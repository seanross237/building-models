# Navier-Stokes

Reusable findings on exact support objects, closure policy, and representation
discipline for Fourier/helical Navier-Stokes branches, including both
packet-centered and support-sheet-centered searches.

## Entries

- `support-means-a-role-labeled-helical-packet-with-mandatory-conjugate-completion.md`:
  in this mission family, `support` is a finite role-labeled helical packet
  together with the forced conjugate completion needed for a real-valued
  velocity field.
- `conjugate-completion-is-mandatory-canonicalization-not-primary-packet-identity.md`:
  conjugate completion must be enforced in the canonical realization, but it is
  bookkeeping for a real-valued packet rather than the packet's primary
  identity.
- `helical-signs-amplitude-normalization-and-phase-anchor-belong-to-the-frozen-packet-sheet.md`:
  later candidate packets must declare one sign sheet, one amplitude
  normalization, and one phase anchor instead of leaving those choices implicit.
- `packet-audits-must-separate-true-symmetries-canonicalization-and-substantive-model-changes.md`:
  packet audits should distinguish genuine invariances from fixed
  canonicalization choices and from changes that create a different packet or
  support-sheet model, including closure-rule or spectator-policy drift.
- `exact-support-search-object-is-a-finite-canonical-helical-support-sheet-with-mandatory-conjugate-completion.md`:
  for exact-support searches that are not intrinsically packet-level, the
  object should be a canonical helical support sheet with mandatory conjugate
  completion rather than a packet template or bare Fourier shadow.
- `recursive-exact-closure-must-use-the-full-active-ledger-and-exact-projected-helical-coefficients.md`:
  exact-support closure must recurse on the whole active ledger using exact
  projected helical coefficients, not a first-generation or schematic support
  graph.
- `closure-forced-spectators-belong-on-the-exact-support-ledger-from-the-first-pass.md`:
  closure-forced spectators belong on the exact support ledger from the first
  pass and cannot be pruned later without changing the model.
- `admissible-enlargements-must-preserve-support-semantics-and-recompute-recursive-closure-from-scratch.md`:
  enlargements stay inside one search class only if support semantics, basis,
  conjugate-completion policy, and recursive closure rule are preserved while
  closure is recomputed from scratch.
- `smallest-first-exact-support-searches-should-order-by-closed-size-shell-span-depth-and-family-dimension.md`:
  `smallest` should be defined on the fully closed support budget tuple
  `(N_closed, ShellSpan, Depth, ParamDim)`, with later negative claims limited
  to a declared escalation ladder through those rungs.
