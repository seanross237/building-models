Status: `INFERRED` artifact-check verdict with `VERIFIED` admissible
enlargement policy

The tempting six-mode mirror-complete triad ledger `S_tidy` is not a genuine
first-budget survivor on this branch. It only looks closed if closure is
illegally truncated to the original triad orbit and the required mirror-cross
pairs are suppressed.

The sharp artifact check is to add one admissible new helical representative
on the already forced orbit `r = p - q`, keep the same support semantics,
same helical basis, same conjugate-completion rule, and same recursive
closure convention, and then recompute closure from scratch on
`S_enl,0 = S_tidy union { +/- (r, sigma_r) }`.
That enlargement does not rescue the ledger: the original active pairs
`(k, p)` and `(k, q)` still force the same first-pass targets
`2p + q` and `p + 2q`, and those targets do not collapse back onto the old
orbit without violating the seed catalog's nondegeneracy conditions.

So the enlargement pass agrees with the closure negative rather than weakening
it. The six-mode picture is only an over-pruned pseudo-survivor, not a real
finite exact fixed point at the current budget.
