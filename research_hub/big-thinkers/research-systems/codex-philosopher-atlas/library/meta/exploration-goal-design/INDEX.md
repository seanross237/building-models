# Exploration Goal Design

Reusable prompt-design lessons for tightly scoped exploration runs that need to
reconstruct inherited arguments rather than re-derive background.

## Entries

- `require-an-explicit-formula-sheet-for-obstruction-branches.md`: force the
  report to pin down the live formula instead of paraphrasing the obstruction.
- `compute-exactly-which-modes-the-test-structure-kills.md`: prevent fake
  progress from quotienting out modes that were already annihilated.
- `anchor-reconstruction-goals-to-a-small-source-list.md`: keep reconstruction
  runs tied to copied mission evidence and frame thin benchmark packets
  honestly.
- `build-a-dedicated-prior-art-packet-before-benchmarking-a-literature-heavy-branch.md`:
  sharpen later novelty screens by reducing dependence on planning prose and by
  treating endpoint adjacency as an overlap burden rather than as soft support.
  The same note now also says that one named literature neighborhood is not a
  comparator packet: the step should write one nearest comparator with one
  exact shared structure, one exact mathematical delta, one already-standard
  gain, and one same-currency cash-out or explicit transfer lemma, or else
  stop before identity work.
- `normalize-notation-and-name-the-operative-bound-up-front.md`: remove sign
  drift and force a choice between schematic and concrete inherited bounds.
- `state-nonlocal-moment-dependence-in-the-goal.md`: say up front when a branch
  depends on a remote moment problem, so local claims can be screened quickly.
- `write-forbid-novelty-claim-rules-for-prior-art-benchmarks.md`: make prior-art
  overlap operational by requiring explicit relabeling bans for each candidate
  family.
- `fix-a-provisional-stretching-representation-early-in-geometry-screens.md`:
  make geometry screens choose a provisional full-stretching representation
  before candidate triage gets too schematic.
- `choose-the-scenario-class-that-makes-a-narrow-hybrid-concrete.md`: when a
  survivor is already narrow, pick the scenario that makes that survivor
  concrete and keep alternative geometries as explicit comparators only.
- `when-hidden-normalization-is-a-risk-start-with-the-least-adapted-localization-that-keeps-the-scenario-visible.md`:
  when localization itself can overfit the branch, fix the least adapted
  package that keeps the target scenario visible before defining observables.
- `when-dynamic-persistence-is-the-only-live-content-force-a-one-diffusion-scale-eulerian-test.md`:
  when a candidate only survives as alleged persistence, force it to answer on
  one fixed diffusion-scale Eulerian window and, if a hybrid is live, compare
  its halves on that exact package too.
- `name-the-live-architecture-and-the-closed-rival-branches-up-front.md`:
  when a step must choose among inherited architectures, make the prompt name
  both the live target and the already-closed rivals so prior negatives act as
  hard constraints rather than vague context.
- `claim-procedural-novelty-when-a-step-only-freezes-an-auditable-table.md`:
  when no new mechanism has been found yet, state the novelty as fixing one
  reusable audited table rather than overstating the branch's content.
- `freeze-the-role-ledger-before-candidate-formulas-when-rebuilding-an-exact-object-from-a-mechanism-memo.md`:
  when a branch is trying to turn an inherited mechanism memo into a new exact
  object, freeze the role ledger first and state explicitly which later
  controls, such as leakage rules, are branch policy rather than inherited
  theorem facts. The same note now also requires each later candidate to name
  one exact packet class, one finite window, one threshold language, one
  canonical packet sheet, and one downstream gate on the same ledger, keeps
  projection rigidity as a mechanism note unless it defines a distinct
  object, forbids behavior candidates from drifting into Tao's averaged
  coefficients when the branch claims to stay exact, and now also says that
  exact-support searches should freeze support object, recursive closure,
  spectator policy, admissible enlargements, and template-versus-object
  boundaries together before any class-level negative claim is trusted.
  It now also says that inherited classwise screening totals are only a
  closure-burden ledger until one explicit support-level closure convention
  and companion list are frozen.
- `when-an-intrinsic-phase-object-is-the-goal-promote-the-orbit-ledger-before-scalar-scores.md`:
  if a branch wants an intrinsic phase/coherence object, start from the
  orbit-level exact interaction ledger, treat scalar scores as diagnostics
  unless the aggregation law is symmetry-forced, and explicitly ban raw
  formulas that name desired routes by hand.
- `fix-the-class-partition-and-finite-window-normalization-before-writing-hard-admissibility-sheets.md`:
  hard admissibility sheets only become invariant gates after the class
  partition and finite-window normalization are frozen.
- `downstream-object-freezes-should-attach-one-next-question-one-invariant-gate-one-carried-witness-and-one-named-debt.md`:
  when a step hands repaired or shortlisted objects downstream, make each
  object leave with one exact next question, one invariant gate or
  admissibility sheet, one smallest carried witness family or stress set, and
  one named same-currency debt; treat audited calibration dossiers as theorem
  setup rather than theorem delivery until that debt is discharged.
- `witness-local-bootstrap-steps-should-freeze-one-hard-scorecard-one-diagnostics-bucket-and-one-no-rescue-line.md`:
  when a branch freezes one static witness locally before closure or dynamics
  are settled, the handoff should lock one hard scorecard, one
  diagnostics-only bucket, and one explicit bookkeeping-versus-rescue line,
  and it should say outright that the frozen dossier is not yet dynamic or
  near-circuit success. A static-screen pass still does not entitle the next
  step to assume one finite exact ODE before a closed support ledger is
  actually frozen.
- `freeze-intrinsic-delayed-transfer-metrics-before-support-searches.md`:
  if a branch needs delayed transfer later, freeze the transfer partition,
  exact inflow observable, intrinsic clocks, spectator burden, and robustness
  rule before support search or trajectory scoring so ordinary fast exchange
  or one-orbit luck cannot be relabeled as evidence.
- `separate-seed-definition-failure-from-closure-failure-in-one-triad-support-searches.md`:
  when the budget is one exact triad orbit, screen out degenerate
  repeated-wavevector and collinear relations at the seed stage so later
  negatives do not mix non-seeds with genuine closure failure.
- `classify-higher-rung-support-seeds-by-exact-support-identity-before-closure.md`:
  when a support search moves above the first rung, collapse candidates by
  exact support identity after the frozen equivalences before closure or
  enlargement, so re-centered copies and old pseudo-support decorations are
  not miscounted as new ladder-level seeds, and then keep the surviving
  canonical families separate through the closure audit before writing one
  global verdict.
