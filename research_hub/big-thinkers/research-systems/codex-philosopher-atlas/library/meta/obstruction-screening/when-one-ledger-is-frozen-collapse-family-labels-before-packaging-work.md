# When One Ledger Is Frozen, Collapse Family Labels Before Packaging Work

Once an endpoint-first branch has already frozen one theorem ledger, one live
bad term, and one admissible gain currency, prior-art overlap screening should
collapse family labels before any packaging work begins.

This prevents a familiar failure mode: nearby names such as
`full stretching control`, `alignment`, `eigenframe`, `restricted-Euler`, or
`harmonic-moment cancellation` can look like a diverse menu even when they are
only slogans, comparators, aliases, or mechanism vocabulary around one live
family.

The screen should therefore ask the collapse question first:
which labels name genuinely distinct families that act on the same frozen bad
term, and which labels merely rename one pressure-side route or one known
prior-art neighborhood?

If the answer collapses back to one family, record that before any packaging
or observable-promotion work. That keeps later steps honest about novelty: the
branch may still test one live family, but it should not pretend to have
several surviving directions.
