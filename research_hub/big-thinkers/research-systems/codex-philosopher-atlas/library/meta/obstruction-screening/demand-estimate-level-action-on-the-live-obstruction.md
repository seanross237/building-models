# Demand Estimate-Level Action On The Live Obstruction

For Tao-filter or obstruction-screening steps, require each candidate to answer
two mechanical questions:

- What estimate changes?
- What exact term or coefficient becomes smaller?

If a candidate cannot name the quantity it shrinks, treat it as dead for the
branch even if it is genuinely Navier-Stokes-specific in some broader sense.

When a branch has several tempting bookkeeping languages, freeze one gain
currency before comparing candidates. Derivative-count or exponent-improvement
claims do not count unless they return to a smaller term or coefficient in
that same fixed estimate.

When a claim is stated in a different burden currency, demand an explicit
transfer lemma back to the same full theorem-facing burden. Derivative-count,
integrability, surrogate-cancellation, or compactness-side claims remain
unresolved until they name how, after later debt is restored, the same live
term comes back with a genuinely smaller coefficient under the same protocol.
If the debt is charged in some different ledger, treat the branch as back in
narrative bookkeeping rather than mechanical comparison.

For reformulation-first audits, that endpoint-exchange question is often the
first real kill gate. A cleaner native variable, sharper exact identity, or
more favorable anchor case is not yet an exchange path. If the packet never
states the same-currency cash-out or the transfer lemma back to the frozen
theorem burden, record `exchange failure` as the primary obstruction and keep
later anti-repackaging or prior-art labels secondary.

For rewrite audits with a frozen cutoff, the operational version of that rule
should be asked directly:

- After the cutoff is inserted, what exact coefficient becomes smaller?
- What ledger entry proves it?

If several candidate rewrites are mainly tempting algebraic cleanups of the
same term, it is often better to audit them side by side. That makes shared
estimate-level washout after localization debt is restored visible early,
instead of letting each candidate sound novel in isolation.

Variable choice also does not count on its own. If a vorticity-side,
projected, or other reformulation still leaves the same frozen bad term and
coefficient untouched, treat the new language as non-discriminating for the
audit.

When the test structure already kills some modes, exclude cosmetic thresholds
explicitly. "Better modulo constants" is not a serious threshold if constants
were never live in the first place.

For endpoint-first branches, mechanism language is still subordinate to the
same estimate-level rule after later debt is restored. Named cancellation,
tensor packaging, or surrogate subestimates count only if, after
reconstruction and localization debt are repaid, the same theorem-level
inequality comes back with a genuinely smaller live coefficient by a nonzero
margin.

That sharper screen can be asked directly:

- After later debt is restored, does the same live term still come with a
  smaller coefficient?
- Is the improvement uniform and non-cosmetic, rather than just a cleaner
  representation or a gain on a surrogate piece?

If the frozen endpoint is itself a criterion family, phrase the gain currency
in that language. On criterion-first geometry branches, the only honest
Step-1 currency is `criterion input`: a new idea counts only if it weakens the
comparator's criterion-scale burden on the same theorem-facing ledger, not if
it merely improves stretching rhetoric or proxy geometry language.
