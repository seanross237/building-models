# Obstruction Screening

Reusable rules for branches that must prove they act on the exact live
obstruction rather than on a nearby reformulation.

## Entries

- `demand-estimate-level-action-on-the-live-obstruction.md`: make each
  candidate identify the exact estimate, gain currency, quantity it shrinks,
  and the ledger entry that proves the shrinkage after cutoff insertion. The
  same rule now also screens out cosmetic thresholds on already-killed modes
  and demands a net improvement on the same theorem-level inequality after
  later debt is restored. It now also requires an explicit transfer lemma for
  claims stated in a different burden currency, and treats debt repaid in a
  different ledger as narrative bookkeeping rather than mechanical
  comparison. It now also records that in reformulation-first audits this
  endpoint-exchange test is often the first real kill gate, so cleaner native
  variables without a same-currency cash-out stay exchange failures rather
  than live candidates awaiting later labels. It now also says that when the
  frozen endpoint is a criterion family, the gain currency should be stated as
  `criterion input`.
- `separate-theorem-endpoints-from-mechanism-language-at-the-first-gate.md`:
  in endpoint-first branches, sort theorem endpoints from mechanism language
  immediately; bottleneck themes or branded defect mechanisms are not serious
  survivors until they act on the named live term inside one inherited ledger.
  The same note now also records that the admission rules themselves should be
  phrased against the exact endpoint object, that a one-floor record should be
  frozen as one primary endpoint plus at most one stricter reserve, that
  solution class and localization must be frozen with that window, that
  criterion-first geometry branches should freeze to the most concrete named
  criterion rather than to `full stretching` rhetoric, and that clearing this
  gate does not yet admit a tensor package.
- `ask-which-identities-remain-exact-after-localization.md`: before comparing
  rewrites, derive the native exact equation first, then ask which identities
  survive the fixed cutoff and where the resulting operator and commutator
  debt re-enters the estimate at the first localization step.
- `distinguish-native-action-on-the-frozen-burden-from-post-reconstruction-action.md`:
  when one theorem-facing burden is frozen, separate candidates that act on it
  natively from candidates that only reach it after reconstruction, since that
  exchange gap is often the real debt-bearing obstruction.
- `for-geometry-branches-only-dynamic-coherence-or-persistence-clears-the-tao-screen.md`: static geometry usually collapses; only dynamic coherence, persistence, or a true stretching-facing algebraic lever should survive.
- `name-the-first-hidden-stronger-assumption-in-comparator-audits.md`: make the
  first smuggled upgrade explicit, since that is often where a comparator
  really fails; this entry now also treats hidden Eulerian matching and
  relocalization as first-order bridge burdens for proxy-based geometry routes.
- `use-a-bounded-shortlist-to-make-likely-collapse-an-earned-later-claim.md`:
  even when the branch baseline looks negative, freeze a narrow candidate
  table so later failure is audited rather than assumed; once that shortlist
  is frozen, keep branch-adjacent families out, preserve a reserve
  comparator, and spend active slots only on candidates with a nontrivial
  route back to the same frozen burden.
- `when-a-candidate-only-survives-by-architecture-drift-count-that-as-failure.md`:
  if a candidate only stays alive by drifting into a different architecture,
  record that as a negative screening result rather than as a near miss.
- `when-all-audited-candidates-are-only-dynamically-weak-stop-before-representation-work.md`:
  when every audited route is only dynamically weak or static-only, stop the
  chain before downstream representation work and demand a new ingredient.
- `record-when-a-candidate-claim-is-only-a-proxy-level-insertion-line.md`:
  when a branch can only write a package-free insertion sentence into an
  existing criterion, file that as an insertion line rather than as an earned
  theorem statement.
- `when-one-bad-term-and-one-gain-currency-are-frozen-a-tao-screen-may-end-the-branch.md`:
  once one live term and one gain currency are fixed, a Tao screen may end the
  branch with a bounded obstruction memo rather than a forced survivor list.
- `when-no-honest-survivor-remains-use-enlargement-to-kill-the-tidiest-pseudo-survivor-and-keep-the-verdict-budget-limited.md`:
  if a fixed budget has no honest survivor, use the smallest admissible
  enlargement only to kill the tidiest pseudo-survivor on the same ledger,
  and keep the resulting negative budget-limited rather than treating it as
  automatic permission to escalate.
- `a-scope-lock-is-not-real-until-baseline-cash-out-and-threshold-are-frozen-in-the-same-burden-currency.md`:
  a setup gate is still obstructed until both the baseline cash-out and the
  minimum threshold or admissible gain notion are frozen in the same burden
  currency that the later coercivity test must use, with later debt charged
  back in that same ledger.
- `collapse-family-labels-before-equivalent-packaging-screening-on-endpoint-first-branches.md`:
  once the endpoint ledger is frozen, collapse slogans, subquestions, and
  observable aliases into the smallest honest family menu before package
  selection; if no packaging improves the same live coefficient, stop before
  tensor-package freeze.
- `host-space-admission-requires-architecture-compatibility-not-just-compactness-prestige.md`:
  before any Step-1 host-space ranking, distinguish genuine frameworks from
  host-space language; compactness rhetoric or Hilbert-space elegance is not
  enough unless the candidate also fits the inherited architecture, or names
  one explicit replacement package, endpoint, and temporal setup.
- `when-one-weak-solution-floor-is-frozen-kill-new-branches-without-an-explicit-admissibility-bridge.md`:
  once one theorem-facing weak-solution floor is frozen, a new branch should
  be killed early unless it also supplies an explicit admissibility bridge
  into that floor; planning-level warnings about smooth flows, filtrations,
  stopping, localization, or reconstruction already support a setup
  obstruction when no bridge packet exists. The same note now also says that
  proxy-based branches must pass that admission check before package
  selection, and that theorem-scale bridge debt should be labeled
  `theorem-floor obstruction`.
- `scale-invariant-appearance-inside-lei-does-not-by-itself-clear-a-host-space-audit.md`:
  a norm's appearance inside scale-invariant LEI quantities does not by itself
  provide local theory, stability, extraction, or normalization for that norm
  as the branch's host space.
- `freezing-burden-columns-does-not-by-itself-make-the-setup-live.md`:
  even a clean burden ledger is only bookkeeping until one temporal
  architecture and one bridge into the later contradiction program are
  actually delivered on the same framework.
- `when-one-ledger-is-frozen-collapse-family-labels-before-packaging-work.md`:
  once one theorem ledger, live bad term, and gain currency are frozen, do the
  prior-art collapse first so slogans, aliases, and mechanism labels do not
  masquerade as several surviving families before packaging work starts.
- `separate-true-symmetries-canonicalization-and-substantive-object-changes-before-claiming-support-stability.md`:
  when support identity is central, freeze the smallest honest packet object
  first and sort true symmetries, canonicalization, and genuinely different
  object changes before declaring anything cosmetic or stable. The same note
  now also records that once one canonical packet sheet is frozen, later
  robustness must distinguish dependence on that honest ledger from
  mathematical nonexistence, and may not relabel sign-sheet, threshold,
  spectator-partition, or time-window changes as after-the-fact
  canonicalization. It now also says that mixed sign-sheet classes should only
  collapse under exact frozen equivalences already on disk, not under hoped-
  for extra target/input permutations.
- `in-exact-support-audits-check-mirror-cross-pairs-before-self-pair-distractions.md`:
  when conjugate completion is mandatory, the first honest closure spill often
  comes from distinct mirror-cross pairs on the full ledger, so vanished
  self-pair doubling or a tidy seed diagram should not be mistaken for exact
  support stability.
- `for-first-spill-support-audits-a-compact-all-sign-coefficient-probe-can-carry-a-budget-limited-verdict.md`:
  once the first spill targets are explicit on a frozen support ledger, a
  small all-sign coefficient probe can certify that they stay live and can
  support a budget-limited spill verdict without a full ODE reconstruction.
- `a-tao-screen-can-be-operational-on-an-exact-but-noncoercive-ledger-if-it-is-only-used-as-an-admission-filter.md`:
  Tao sensitivity may become operational before coercivity, but only after
  exact structure is separated from bookkeeping and only as an admission
  filter rather than as a one-sided gain.
  It now also says that behavior-based finite-window survivors should stay in
  that admission-filter role unless they later produce same-currency
  theorem-facing cash-out. It now also says that after a repair pass, strong
  candidates should be tightened on the same ledger while any narrow-window
  survivor without a concrete next object freeze should leave the shortlist.
- `do-not-upgrade-a-friendly-exact-witness-before-the-hostile-dossier-runs-on-the-same-ledger.md`:
  a friendly exact packet witness is still only class-local until the hostile
  dossier is run on the same frozen ledger; do not promote a one-sided
  construction example into theorem evidence before that symmetric comparison.
- `when-forced-interaction-classes-are-known-but-no-finite-support-ledger-is-frozen-label-the-gap-exact-non-isolability.md`:
  when a frozen packet witness already has known forced interaction classes
  but still lacks one finite closed support ledger, the first honest
  obstruction label is exact non-isolability / arbitrary truncation before
  dynamics, not an overclaimed class-exit or infinite-growth theorem.
- `static-cancellation-cases-should-test-admissibility-filters-not-carry-the-main-obstruction-verdict.md`:
  static zero-activity anti-families are useful only as filter tests; they
  verify admissibility discipline but should not carry the main obstruction
  verdict for a live packet branch.
- `for-near-degenerate-anti-families-name-what-stays-live-and-what-breaks-first.md`:
  degeneracy language only becomes informative anti-family evidence when the
  report names, on one frozen ledger, which channels remain live and which
  mechanism fails first.
- `robustness-audits-may-keep-several-honest-survivors-with-different-statuses.md`:
  once the candidate table and object ledger are frozen, a robustness audit
  may honestly keep several survivors with different labels such as
  `stable after canonicalization` and `use-case-limited but honest` rather
  than forcing a global veto.
- `when-two-repaired-survivors-share-one-ledger-keep-both-unless-an-exact-implication-or-counterexample-collapses-them.md`:
  after a repair pass, keep two exact survivors on the same frozen ledger if
  they still act on different burdens; shared dossier outcomes are not enough
  to collapse them without an exact implication or counterexample.
- `use-the-low-leakage-friendly-family-to-distinguish-event-trace-rigidity-from-isolation-failure.md`:
  when a behavior-based candidate remains ambiguous even on the friendly
  family where leakage is already small, treat the main bottleneck as
  event-trace rigidity rather than raw isolation.
- `when-a-late-behavioral-repair-only-works-by-shortening-the-event-language-count-it-as-definition-drift.md`:
  when a late-stage behavioral candidate is ambiguous, split the early exact
  filter from the late transfer requirement on the same frozen sheet; if the
  late repair only works by shortening the event language after outcomes are
  known, discard it as definition drift rather than an honest survivor.
- `for-frozen-ledger-repair-passes-use-the-final-dossier-and-tighten-only-to-recorded-friendly-maxima.md`:
  when a branch reaches a frozen-ledger repair pass, use the final landed
  dossier as the controlling record, tighten each admissibility sheet only to
  the largest recorded friendly load, leave saturated budgets unchanged, and
  keep already-earned hostile failure modes instead of inventing sharper
  adverse tables.
- `when-repaired-sheets-descend-through-several-steps-make-step-level-results-control-and-treat-conflicting-drafts-as-history-only.md`:
  when one repaired sheet is inherited across several steps, name one
  controlling step-level source before new math, use subordinate exploration
  reports only as support, and log conflicting draft values as historical
  variance rather than as live threshold options.
- `when-prior-art-overlap-is-secondary-let-the-earliest-honest-obstruction-label-win.md`:
  if a narrow NS-specific discriminator survives but setup or calibration has
  already failed, keep prior-art overlap as supporting context and let the
  earliest honest obstruction label stay primary; this now also includes
  theorem-floor obstruction when a proxy bridge already looks theorem-scale.
- `for-stochastic-transport-branches-do-not-call-the-scope-lock-real-until-one-observable-carries-one-sentence-of-leverage.md`:
  a stochastic transport branch may narrow honestly to one burden and one
  comparator neighborhood, but it remains menu-level until one theorem-facing
  observable and one explicit leverage sentence are frozen; in that setting
  `local concentration exclusion first` is a sharper filter than generic
  continuation language, and even that endpoint-ordering success does not by
  itself clear solution-class admission on the inherited weak-solution floor.
- `write-anti-repackaging-rules-in-the-branchs-own-burden-currency.md`:
  anti-repackaging screens should be written as exact failure buckets in the
  branch's own burden language, so expectation-only, filtration-only,
  localization-only, or parasitic-framework-only consequences do not masquerade
  as progress. The same note now also says to classify by the restored full
  theorem-facing burden and to keep any `ambiguous` middle bucket only for one
  exact missing transfer step.
