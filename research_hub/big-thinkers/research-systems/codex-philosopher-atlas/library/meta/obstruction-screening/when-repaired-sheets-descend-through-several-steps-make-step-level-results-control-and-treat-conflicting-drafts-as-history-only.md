# When Repaired Sheets Descend Through Several Steps, Make Step-Level Results Control And Treat Conflicting Drafts As History Only

When a branch inherits one repaired threshold sheet through several earlier
steps, write the authority rule explicitly before any new math.
Use final step-level result files as the controlling source, use supporting
exploration reports only as subordinate evidence, and keep earlier
conflicting draft values only as historical variance.

This matters whenever the inherited record contains drift, such as an earlier
exploration draft with one threshold and a later step-level repair with
another.
If the branch has already frozen the repaired sheet, do not turn that
variance into a fake live calibration choice.

Name one controlling source path on disk, log the older conflicting values as
history only, and then carry the repaired sheet forward unchanged.
That keeps later steps from silently choosing among drafts after the branch
record has already been closed.
