# Step 3 Goal: Chain Step 3 Run A Tiered Robustness Audit Instead Of A Single Invariance Veto

## Mission Context

**Mission:** Exact NS No-Near-Closed Tao Circuit

**Active chain:**
`Canonical Winning Chain - Co-Evolving Definition Freeze and Exact Interaction Audit With Invariant Downstream Gates`

**Why this step exists now:** `step-002` succeeded at the branch's second gate.
It promoted exactly three candidate notions of `near-closed Tao circuit`,
attached all three to the same exact five-channel helical interaction core,
named the forced spectator classes, and assigned one downstream gate to each
candidate. Mission control therefore keeps the current chain active. The next
unresolved bottleneck is no longer object vagueness or missing interaction
templates. It is whether those candidates survive a disciplined robustness
audit rather than depending on representation cherry-picking, tuned thresholds,
or hidden modeling drift.

## This Step Covers

This strategizer execution covers **Chain Step 3 only**.

  `Template-Defect Near-Closure`,
  `Windowed Spectator-Leakage Budget`,
  `Delayed-Threshold Itinerary`;

  `not well-defined`,

  or `not useful for the target theorem or counterexample question`.

No audit may rescue a candidate by changing packet semantics, redefining the
spectator partition by convenience, retuning thresholds after seeing outcomes,
or replacing the Step-2 object with Tao's averaged-model coordinates.

## Required Deliverables

Produce all of the following inside `RESULTS.md`:

1. **Transformation taxonomy**
   - list the exact transformations treated as true exact symmetries;
   - list the choices treated as canonicalization only;
   - list the moves treated as substantive modeling changes;
   - explain briefly why each item belongs in that class.

2. **Robustness sheet for each candidate**
   - state which parts of the candidate are symmetry-invariant;
   - state which parts depend on canonicalization and whether that dependence is
     stable under the frozen policy;
   - state which substantive modeling sensitivities are honest use-case limits
     and which would be fatal;
   - state explicitly which Step-1 and Step-2 commitments are doing real work.

3. **Robustness matrix**
   - classify each candidate as exactly one of:
     `symmetry-stable`,
     `stable after canonicalization`,
     `use-case-limited but honest`,
     or `fatally arbitrary`;
   - give one short reason for the classification.

4. **Downstream-gate stability note**
   - for each candidate, say whether its Step-2 downstream gate remains
     meaningful after the robustness audit;
   - name the exact data Step 4 would need next for each surviving candidate.

5. **Step verdict**
   - state whether Chain Step 4 is now well-posed;
   - if yes, list which candidates survive into Step 4 and which do not;
   - if no, state plainly that the branch should stop and name the earned
     negative failure bucket.
