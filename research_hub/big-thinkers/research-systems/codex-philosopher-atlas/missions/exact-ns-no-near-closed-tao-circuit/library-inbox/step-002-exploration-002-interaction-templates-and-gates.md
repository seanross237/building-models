# Step 2 Exploration 002 - Interaction Templates And Gates

## Shared Exact Interaction Core

- `[VERIFIED]` All promoted Step-2 candidates track the same ordered exact
  role core:
  `A -> B`,
  `A -> C`,
  `B,C -> C`,
  `C,A <-> D`,
  `D,D -> E`.
- `[VERIFIED]` All promoted candidates must also account for the same forced
  extras:
  mirror / conjugate companions,
  same-scale companion triads,
  long-leg feedback,
  and cross-scale next-shell feedback.
- `[VERIFIED]` Exact Leray / pressure projection remains the mechanism that
  enforces coefficient rigidity and spectator leakage; it should not be treated
  as a distinct candidate notion.

## Downstream Gates

- `[INFERRED]` `Template-Defect Near-Closure`
  uses one invariant template-defect observable
  `(Delta_tmpl, Delta_spec)` on one canonical packet family and finite window.
- `[INFERRED]` `Windowed Spectator-Leakage Budget`
  uses one hard admissibility sheet
  `(L_tot, L_mirror, L_companion, L_feedback, L_cross)`.
- `[INFERRED]` `Delayed-Threshold Itinerary`
  uses one ordered-threshold admissibility sheet
  `(t_clk, t_trig, t_rot, t_next; sign sheet; spectator pass/fail)`.

## Reusable Lesson

- `[INFERRED]` A Step-2 candidate is ready for robustness audit only when its
  exact interaction template and its downstream gate are stated in the same
  packet and threshold language.
