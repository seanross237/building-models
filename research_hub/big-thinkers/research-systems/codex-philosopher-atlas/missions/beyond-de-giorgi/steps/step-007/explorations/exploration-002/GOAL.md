# Exploration 002 Goal

## Objective

Audit `\dot H^{1/2}` against the same Step-1 extraction-package checklist used
for `L^3`, and decide whether `BMO^{-1}` deserves admission at all under this
chain's standard.

If the local record justifies admitting `BMO^{-1}`, audit it under the same
requirement-by-requirement standard. If not, exclude it explicitly and explain
why the exclusion is sharper than a vague dislike of the space.

## Required Questions

Answer all of the following from local repository materials:

1. What exact `\dot H^{1/2}` package inputs are actually present locally for:
   local theory, perturbation/stability, extraction input, symmetry
   normalization, pressure/local-energy compatibility, and usable compactness?
2. What solution class does `\dot H^{1/2}` naturally commit the branch to, and
   where does that fit or clash with the mission's fixed suitable-weak / LEI
   floor?
3. Does the local record isolate one sharply bounded missing theorem for
   `\dot H^{1/2}`, or is the package gap diffuse?
4. Should `BMO^{-1}` be admitted for audit under this chain's standard?
5. If `BMO^{-1}` is not admitted, what exact locally supported reason justifies
   exclusion:
   missing large-data stability, missing nonlinear profile decomposition,
   mismatch with LEI/local-energy architecture, or some other sharper local
   obstruction?
6. If `BMO^{-1}` is admitted, produce the same audit table and final
   classification for it.

## Working Hypotheses To Test, Not Assume

- `\dot H^{1/2}` has cleaner Hilbert-space compactness language than
  `BMO^{-1}`, but the local record may not bridge it cleanly to suitable weak
  solutions and local-energy compatibility.
- `BMO^{-1}` is likely admissible only for small-data mild theory and is
  therefore a poor host for the large-data minimal-blowup package this chain
  needs.
- The sharpest honest outcome may be:
  `\dot H^{1/2}` fails because its extraction package is diffuse, and
  `BMO^{-1}` is excluded before full audit.

Confirm, refine, or reject those hypotheses from source-backed local evidence.

## Success Criteria

- Produce a requirement-by-requirement `\dot H^{1/2}` audit table with columns:
  `requirement`, `known input`, `solution-class commitment`,
  `overlap / prior-art note`, `missing piece`.
- Produce either:
  - a `BMO^{-1}` audit table under the same format, or
  - an explicit `not admitted for this chain` ruling with one sharply stated
    reason grounded in local sources.
- Label major claims `[VERIFIED]`, `[INFERRED]`, or `[PROPOSED]`.
- End with clear classifications for both `\dot H^{1/2}` and `BMO^{-1}`.

## Failure Criteria

- The report keeps `BMO^{-1}` alive only by citing prestige or small-data lore
  rather than a large-data extraction package.
- The report treats Hilbert-space compactness language as enough, without
  reconciling the branch with the LEI/local-energy floor.
- The report uses more than one diffuse future theorem stack while still
  claiming viability.

If failure occurs, say so directly and recommend rejection or exclusion rather
than continuation by rhetoric.
