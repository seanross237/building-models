# Exploration 003 Goal

## Objective

Screen the frozen package against the nearest prior-art families and recommend
the Step-11 verdict.

This exploration should assume the scope lock and Tao memo are already as
concrete as the repository permits. Its job is to decide whether the package
is genuinely distinct, merely repackaged folklore, or ambiguous in a way that
still blocks Step 2.

## Required Questions

Answer all of the following from local repository materials:

1. How does the frozen package compare with:
   Waleffe-type triad analysis,
   helical decimation,
   shell-model intuition,
   and same-helicity favorable-sector lore?
2. Which substantive claims are `[VERIFIED]`, `[INFERRED]`, or `[PROPOSED]`?
3. Does the package stay distinct at theorem-facing burden level, or collapse
   into sign-sensitive but non-coercive cascade bookkeeping?
4. If the branch fails, what is the narrowest honest failure label:
   `setup obstruction`,
   `calibration obstruction`,
   `prior-art overlap obstruction`,
   or a stacked combination?
5. If the branch somehow survives, what exact frozen package would Step 2 be
   allowed to use without changing terms?

## Working Hypotheses To Test, Not Assume

- The most honest verdict is negative because the local record already warns
  that exact energy-helicity coupling is likely exact-but-noncoercive and
  prior-art-adjacent.
- Same-helicity favorable sectors and shell-style transfer stories are
  novelty failures unless they carry a quantitatively relevant fraction of the
  full burden.
- The branch may die for more than one reason at once, but the report should
  still name the first decisive obstruction as sharply as possible.

Confirm, refine, or reject these hypotheses from source-backed evidence.

## Success Criteria

- Produce a compact overlap-and-verdict memo with sections:
  `prior-art screen`,
  `distinctness assessment`,
  `typed obstruction or clearance`,
  `Step-2 freeze if any`.
- Label major claims `[VERIFIED]`, `[INFERRED]`, or `[PROPOSED]`.
- End with one explicit Step-11 verdict recommendation only.

## Failure Criteria

- The report calls descriptive overlap a live theorem distinction.
- The report keeps the branch alive by postponing the obstruction to Step 2.
- The report understates a stacked failure by forcing a falsely clean positive.

If failure occurs, say so directly and recommend
`prior-art overlap obstruction` or the sharper stacked label supported by the
record.
