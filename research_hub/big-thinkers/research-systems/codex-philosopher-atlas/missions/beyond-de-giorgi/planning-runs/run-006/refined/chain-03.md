# Refined Chain 03 - Weak-Type Endpoint Audit With Early Mechanism Gate

## Central Premise

This branch is live only if one weak-type or Lorentz package does more than repackage borderline estimates. It must support, in one coherent local program:

- one exact Navier-Stokes-specific feature that the averaged model should destroy;
- one endpoint or concentration mechanism that uses that feature essentially rather than rhetorically;
- and one localization-compatible contradiction line that does not recreate De Giorgi by another name.

If no such package survives, the correct output is an obstruction memo showing whether the failure is generic-analysis collapse, localization collapse, hidden replacement iteration, prior-art collapse, or absence of any mechanism beyond endpoint packaging.

## Verifiable Result Target

Either:

- one theorem-like weak-type endpoint program naming the exact NS-specific feature, the exact weak-type inputs, the localization rules, the contradiction line, and the first missing lemma; or
- one calibrated obstruction memo naming the first honest break, with stacked failure labels allowed when the debts genuinely remain simultaneous.

## Why This Chain Is Meaningfully Different

This version keeps the anti-recursion discipline of the original chain, but it no longer treats weak-type structure itself as evidence of a live route. A candidate earns promotion only if it passes an early mechanism gate, an early Tao screen, and a localization-first audit before the chain spends effort on endpoint formatting.

## Scope Lock

Before substantive execution begins, fix all of the following:

- the banned replacement-iteration list:
  literal `U_k` recursion,
  epsilon-improvement loops,
  stopping-time or covering iterations,
  endpoint bootstraps,
  and profile-extraction or compactness steps that merely replay the same iterative burden in another language;
- the burden columns that must stay separate:
  generic weak-type tools,
  claimed NS-specific feature,
  localization and pressure debt,
  prior-art overlap,
  Tao-sensitivity,
  and contradiction power;
- the admission rule that no endpoint family is frozen finally until a concrete mechanism survives the early screen;
- the policy that a theorem-like positive output is allowed only if its first missing lemma is genuinely narrower than full regularity.

If these items cannot be fixed concretely, stop and write a setup obstruction.

## Ordered Steps

### Step 1 - Freeze guardrails and run an early package audit

- Depends on: none.
- Action: state the banned hidden substitutes for De Giorgi, not just the literal old recursion.
- Action: inventory the available weak-type or Lorentz inputs, but split them immediately into:
  generic harmonic-analysis assets,
  framework burdens such as local-energy compatibility,
  and any candidate NS-specific ingredient.
- Action: front-load prior-art overlap against known endpoint criteria, Lorentz formulations, and known nonclosure patterns.
- Expected output: a package-audit ledger with separate columns for generic tools, NS-specific candidates, localization burden, and overlap risk.
- Kill condition: if the package is only generic endpoint harmonic analysis, or if the branch is already prior-art-equivalent in substance, stop and write a generic-package or overlap obstruction.

### Step 2 - Apply the Tao gate and localization gate before endpoint lock

- Depends on: Step 1.
- Action: for each surviving package, state what exact feature of the true Navier-Stokes nonlinearity it claims to use and why Tao-style averaging should destroy that feature.
- Action: in the same pass, test whether the package survives cutoffs, truncations, pressure decomposition, and distribution-function bookkeeping in the local weak-solution setting.
- Action: reject any package whose operative content remains weak-type Calderon-Zygmund, interpolation, or endpoint singular-integral theory after this screening.
- Expected output: at most two admissible packages, each with one NS-specific feature note and one localization note.
- Kill condition: if no package is both Tao-sensitive and localization-compatible at the architectural level, stop and write a Tao/localization obstruction.

### Step 3 - Demand one actual mechanism before choosing the endpoint target

- Depends on: Step 2.
- Action: for each surviving package, name one exact mechanism:
  a propagation law,
  coercive bound,
  monotonicity surrogate,
  compactness-rigidity bridge,
  or another concrete contradiction engine.
- Action: explain how the mechanism uses the Step-2 NS-specific feature and weak-type inputs, and why it is more than endpoint repackaging.
- Action: reject any candidate that avoids literal level-set recursion but still depends on hidden replacement iteration through covering, epsilon-regularity improvement, repeated truncation, or equivalent bootstrap loops.
- Expected output: one promoted mechanism dossier and at most one backup.
- Kill condition: if every candidate is only an endpoint family without an engine, stop and write a no-mechanism obstruction.

### Step 4 - Freeze one endpoint target and write the full local control attempt

- Depends on: Step 3.
- Action: choose one endpoint target only after a mechanism survives:
  one regularity criterion,
  one concentration statement,
  or one contradiction line.
- Action: write the full local control attempt, including pressure splitting, cutoff terms, weak-type multiplication debt, and any distribution-tail bookkeeping.
- Action: classify the result explicitly as:
  actual control,
  only softer reformulation,
  strong-type collapse,
  localization collapse,
  hidden-iteration relapse,
  or Tao-compatible generic analysis.
- Expected output: a control-attempt memo with one exact endpoint target and a full debt ledger.
- Kill condition: if the route cannot be written without reimporting stronger norms, recreating the old burden stack, or downgrading to pure endpoint packaging, stop and write the typed obstruction.

### Step 5 - Name the first missing lemma and demand a real complexity drop

- Depends on: Step 4.
- Action: identify the first missing lemma in the promoted route and measure whether it is genuinely narrower than full regularity.
- Action: state whether the claimed progress is a real complexity drop or only a relabeling of the full difficulty into one compressed statement.
- Action: allow the failure diagnosis to remain stacked if more than one debt survives honestly.
- Expected output: either a narrowed conditional program with one real missing lemma or a stacked architectural obstruction.
- Kill condition: if the first missing lemma is effectively the whole regularity problem, or if multiple unresolved debts are being hidden under one label, downgrade to obstruction.

### Step 6 - State the narrowest earned claim only

- Depends on: Step 5.
- Action: if positive, write one theorem-like endpoint program naming:
  exact weak-type inputs,
  exact NS-specific feature,
  exact localization rules,
  exact contradiction line,
  and the first missing lemma.
- Action: if negative, write one obstruction memo naming the first honest break, with stacked labels allowed where earned:
  generic-analysis collapse,
  localization collapse,
  hidden replacement iteration,
  prior-art collapse,
  or no distinct mechanism.
- Action: remove all unevaluated alternatives and keep only the single earned positive route or the single earned obstruction map.
- Expected output: a final-comparison-ready weak-type chain with no rhetorical slack.
- Kill condition: if the final claim cannot be stated without vague mechanism language or hidden debt compression, narrow it further until it is fully earned.

## Success Standard

Do not count this branch as successful because it produces sophisticated Lorentz-space language. It succeeds only if it either:

- isolates one weak-type route with a concrete NS-specific mechanism and a genuinely narrower missing lemma; or
- proves, in a disciplined way, that the remaining weak-type package is generic, nonlocalization-compatible, or architecturally mechanism-free once De Giorgi substitutes are banned.
