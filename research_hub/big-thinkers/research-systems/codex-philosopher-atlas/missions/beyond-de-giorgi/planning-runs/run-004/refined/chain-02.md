# Chain 02 - Critical Element Program With Extraction Audit Before Rigidity

## Central Premise

A compactness-rigidity route is only live if one specific critical or near-critical host space supports an explicit extraction package. The program should first audit that package theorem by theorem, then define the hypothetical critical element, and only then test rigidity quantities that are actually defined on that class.

## Verifiable Result Target

Either:

- a theorem-like conditional program naming one host space, the exact extraction and stability inputs, the critical-element class, one admissible rigidity quantity, and the first missing lemma whose scope is strictly narrower than full regularity; or
- an obstruction map identifying the first fatal break as one of: extraction, critical stability, normalization or solution class, definability of the rigidity quantity, or failure of contradiction even under almost periodicity.

## Why This Chain Is Meaningfully Different

This chain still explores concentration-compactness and rigidity, but it refuses soft ranking memos, refuses novelty theater around Tao's averaged model, and forces all claims to be tied to one audited host space and one explicit contradiction mechanism.

## Ordered Steps

### Step 1 - Audit one host space against a full extraction package

- Depends on: none.
- Action: compare `L^3`, `\dot H^{1/2}`, and only if justified `BMO^{-1}` against an explicit checklist: local theory in the intended solution framework, large-data perturbation or stability, profile decomposition or equivalent extraction input, symmetry normalization, pressure or local-energy compatibility, and a compactness notion usable by the later rigidity step.
- Action: select a primary host space only if each checklist item can be tied either to a named known input or to one sharply identified missing theorem.
- Expected output: a package-audit table with columns for requirement, known input, solution class, and missing piece, plus one chosen host space or an explicit no-host verdict.
- Kill condition: if no single host space clears the checklist except for at most one sharply stated missing theorem on the critical path, stop and record a package obstruction.

### Step 2 - Define the hypothetical minimal blowup object in the chosen framework

- Depends on: Step 1.
- Action: state the minimizing quantity, solution class, symmetry group, normalization conventions, and the exact compactness or almost-periodicity statement the extraction package would deliver.
- Action: specify how scaling, spatial translation, pressure normalization, and any Galilean issue are handled in the chosen framework.
- Expected output: a theorem-like dossier of the form: under inputs A, B, and C, a minimal counterexample with properties P, Q, and R exists.
- Kill condition: if the critical object cannot be stated without importing additional unproved ingredients beyond the Step 1 package audit, stop and record a formulation obstruction.

### Step 3 - Screen rigidity quantities only on the actual critical-element class

- Depends on: Step 2.
- Action: list candidate rigidity quantities or identities that are well-defined on the Step 2 class and note whether they provide sign, monotonicity, coercivity, unique-continuation leverage, or another explicit contradiction mechanism.
- Action: treat failure in Tao's averaged model only as a secondary discriminator, not as a gate. Reject any candidate that is undefined on the class, not stable under the compactness limit, or too weak to support a contradiction path.
- Expected output: one promoted rigidity quantity and at most one backup, each with a short mechanism statement explaining exactly how it would interact with the critical element.
- Kill condition: if no candidate survives definability plus mechanism screening, stop and record a rigidity-definability obstruction.

### Step 4 - Force a genuine complexity drop in the contradiction step

- Depends on: Step 3.
- Action: run the promoted rigidity quantity through the critical element and state the contradiction pathway line by line, including where compactness, almost periodicity, pressure structure, and solution-class regularity enter.
- Action: identify the first missing lemma and require that it be strictly narrower than arbitrary finite-energy regularity, using the compactness-normalized class in an essential way.
- Expected output: either a conditional contradiction theorem schema or a failure memo classified as one of: non-closure under limits, lack of coercivity, insufficient regularity to state or pass the quantity, or failure of contradiction even under almost periodicity.
- Kill condition: if the first missing lemma is effectively the full regularity problem, or if the failure mode cannot be localized, downgrade to an obstruction result.

### Step 5 - State the narrowest earned claim in theorem form

- Depends on: Step 4.
- Action: write the final result as either a conditional theorem blueprint with exact assumptions, exact known inputs, exact missing lemma, and conditional consequence, or as an obstruction statement naming the first fatal break and why later steps cannot start.
- Action: keep the final claim tied to the single audited host space and the single promoted rigidity quantity. Remove all unevaluated alternatives.
- Expected output: a final-comparison-ready chain that is either an executable conditional compactness-rigidity program or a precise obstruction map.
- Kill condition: if the final claim cannot be written in theorem-like form, the chain has not earned a presentable result.
