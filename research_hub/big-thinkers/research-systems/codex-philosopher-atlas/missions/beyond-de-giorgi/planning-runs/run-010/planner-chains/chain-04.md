# Chain 04 - Critical Element Program With Pre-Admitted Exact-NS Firewall

## Central Premise

Compactness-rigidity is not a primary mechanism hunt anymore. It becomes live only if an upstream branch first produces one theorem-facing exact-NS firewall quantity that is definable on the compactness class, survives localization and limits, and genuinely narrows the problem. Without that firewall, this chain should terminate immediately with a no-entry obstruction.

## Why This Chain Is Meaningfully Different

This is the only contradiction-architecture branch in the portfolio. It does not try to discover a new mechanism inside the PDE; it asks whether a mechanism already isolated elsewhere can survive extraction, almost-periodic normalization, and contradiction. That makes it structurally different from the transport, geometry, and packet-symbol chains.

## Ordered Steps

### Step 1 - Enforce the firewall admission gate before any host-space work

- Depends on: one upstream chain having produced a concrete firewall candidate.
- Action: write the exact firewall quantity, the theorem endpoint it supports, and why it is genuinely exact-NS rather than generic harmonic-analysis structure.
- Action: require that the quantity be definable on the intended compactness class and stable enough to survive compactness limits.
- Expected output: either a firewall-admission memo or an immediate no-entry verdict.
- Kill condition: if no concrete firewall packet exists, stop immediately with a firewall-missing obstruction.

### Step 2 - Freeze one host space, one framework, and one normalization package

- Depends on: Step 1.
- Action: choose one host space only, such as `L^3` or `\dot H^{1/2}`, and one solution framework.
- Action: audit local theory, stability, extraction, pressure compatibility, and compatibility of the firewall quantity with compactness.
- Action: freeze one normalization package for the hypothetical minimal object.
- Expected output: one host-space and normalization audit.
- Kill condition: if the host space still lacks a coherent extraction package or the firewall quantity is not definable there, stop with a host-space obstruction.

### Step 3 - Define the critical object in theorem-like form

- Depends on: Step 2.
- Action: state the minimizing quantity, symmetry normalization, compactness statement, and exact class of the hypothetical minimal blowup object.
- Action: ensure the firewall survives in that class with no hidden extra assumptions.
- Expected output: a critical-object dossier.
- Kill condition: if the critical element cannot be formulated without importing diffuse new assumptions, stop with a formulation obstruction.

### Step 4 - Run the contradiction pathway line by line

- Depends on: Step 3.
- Action: insert the firewall into the almost-periodic or compactness-normalized setting.
- Action: identify exactly where contradiction should arise and what remaining lemma is needed.
- Action: reject any route whose first missing lemma is equivalent to full regularity.
- Expected output: one contradiction-path memo plus a first-missing-lemma statement.
- Kill condition: if the first missing lemma does not reduce complexity relative to the full problem, stop with a no-complexity-drop obstruction.

### Step 5 - Use Tao sensitivity only as secondary evidence and state the narrowest earned claim

- Depends on: Step 4.
- Action: ask whether the admitted firewall would survive Tao-style averaging, but only after definability and contradiction power are established.
- Action: if positive, write one theorem-like conditional blueprint naming the host space, framework, critical-element class, firewall quantity, contradiction line, and first narrower missing lemma.
- Action: if negative, classify failure as missing firewall, host-space incoherence, critical-object formulation failure, firewall non-definability, or no genuine complexity drop.
- Expected output: a final-comparison-ready compactness route or obstruction memo.
- Kill condition: if Tao sensitivity is the main reason the route looks interesting, downgrade it to obstruction.
