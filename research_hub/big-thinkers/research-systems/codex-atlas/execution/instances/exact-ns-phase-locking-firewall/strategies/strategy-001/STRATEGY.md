# Strategy 001: Freeze One Intrinsic Triad-Phase Object, Then Force or Kill the Firewall

## Objective

Turn the mission question

```text
does Tao-like delayed forward transfer in exact Navier-Stokes force a
quantitatively strong phase-locking burden across exact interacting triads?
```

into one theorem-facing exact-NS program with:

- one preferred intrinsic phase-coherence observable and at most one backup,
- one preferred intrinsic delayed-transfer event,
- and one sharp outcome in the mission's allowed terminal categories:
  1. positive phase-locking firewall,
  2. counterexample: delayed transfer without strong phase locking,
  3. definition-level failure,
  4. model-level failure.

This strategy is not allowed to drift back into packet roles, Tao-edge annotation,
generic resonance folklore, or amplitude-only bookkeeping with no phase variable left.

## Methodology

### Phase 0: Intrinsic Phase-Object Gate

Start from the exact helical Fourier form of Navier-Stokes and define one
preferred phase/coherence observable that is intrinsic enough to be theorem-facing.

Acceptable preferred-object shapes include:

- a normalized triad-phase alignment observable built directly from exact helical interaction monomials,
- a scale-local phase concentration quantity on an intrinsic exact triad population,
- or a coherence-defect quantity comparing signed forward transfer with absolute transfer on exact triads.

The preferred object must freeze:

- the exact triad population it is measured on,
- the phase variable or phase concentration it uses,
- the normalization,
- the symmetry / gauge / conjugation convention,
- and the exact sense in which it measures "coherence" rather than amplitude only.

Required deliverables:

- one preferred observable with an explicit formula,
- at most one backup observable,
- one invariance ledger stating what is intrinsic and what is still choice-dependent,
- one sentence explaining why the object does not secretly reintroduce packet roles or desired-channel labels.

Phase 0 kill conditions:

- If the observable depends on post hoc Tao-role labeling, packet annotation, or a hand-picked desired triad class, stop with `definition-level failure`.
- If harmless conjugation, representative, or shell-window choices move the object materially, stop with `definition-level failure`.
- If the object collapses into an amplitude-only quantity with no real phase content, stop with `model-level failure`.

### Phase 1: Delayed-Transfer Event Definition

Fix one preferred delayed forward-transfer event that is intrinsic to exact NS and
faithful to the Tao-facing mechanism without role labels.

Good candidate shapes include:

- a short-window net forward flux burst across one intrinsic dyadic scale interface,
- a sustained positive transfer imbalance on an intrinsic scale-local exact triad population,
- or an exact helical transfer event with a concrete receiver-shell gain witness.

Required deliverables:

- one preferred event definition and at most one backup,
- one exact-NS fidelity ledger describing preserved versus discarded structure,
- one theorem target and one counterexample target stated in terms of that event and the preferred phase observable.

Phase 1 kill conditions:

- If the event only becomes Tao-faithful after reintroducing desired-channel roles, stop with `definition-level failure`.
- If the event is so coarse that it no longer preserves the phase structure the mission is supposed to study, stop with `model-level failure`.

### Phase 2: Smallest Honest Exact-Support Audit

Before any long-window forcing campaign, test the preferred observable and event
on the smallest honest exact setting that still preserves real phase dynamics.

This phase should prefer:

- one exact triad or smallest exact triad cluster where the transfer term and phase variable can be written explicitly,
- one reduced model only if it states exactly which phase-sensitive exact-NS features it preserves,
- and one cheap structural question before any large computation:
  can the delayed-transfer target remain live while the preferred coherence observable stays small in the minimal faithful setting?

Required deliverables:

- one explicit transfer identity showing where the phase variable enters,
- one verdict on whether the preferred observable is nontrivial and fidelity-preserving in the minimal setting,
- one decision whether a larger forcing/counterexample campaign is scientifically justified.

Phase 2 kill conditions:

- If the minimal honest setting already shows the observable is too weak, too gauge-dependent, or too detached from transfer, stop with `model-level failure`.
- If the smallest-support audit already yields a faithful counterexample, promote that directly and stop the strategy after adversarial screening.

### Phase 3: Force-Or-Kill Test

Attack one concrete theorem candidate or one concrete counterexample candidate at
a time. Each exploration in this phase must begin with exactly one target:

- one lower bound on coherence during the transfer window,
- one incompatibility statement,
- one faithful low-coherence counterexample candidate,
- or one structural disconnect statement showing the observable cannot control the event strongly enough.

Acceptable outputs are:

- a lower bound or threshold on the preferred coherence observable during the event window,
- an incompatibility of delayed transfer with sustained low coherence,
- a faithful exact-NS or tightly justified reduced-model counterexample,
- or a sharp negative showing the observable is definable but not sufficiently coupled to delayed transfer.

### Phase 4: Mandatory Adversarial Screen

Any apparent positive firewall or counterexample must be stress-tested against:

- symmetry or gauge choices that trivialize the phase observable,
- changes in the intrinsic triad population definition,
- high-amplitude transfer without genuine phase locking,
- homochiral / heterochiral special regimes that create fake coherence,
- and the possibility that the same phenomenon survives Tao-style averaging.

This phase must answer explicitly whether the surviving claim is:

- exact-NS-specific,
- genuinely phase-sensitive,
- and free of hidden packet or role dependence.

### Phase 5: Final Verdict

Finish with exactly one named mission outcome:

1. positive phase-locking firewall,
2. counterexample: delayed transfer without strong phase locking,
3. definition-level failure,
4. model-level failure.

The final report must make the next-step implication unambiguous:

- pursue,
- kill,
- or reformulate.

## Cross-Phase Rules

1. By the end of Phase 0 there must be one preferred phase observable and at most one backup.
2. By the end of Phase 1 there must be one preferred delayed-transfer event and at most one backup.
3. Do not reintroduce packet roles, desired Tao edges, or hidden target-channel labels under new names.
4. The exact phase variable must stay explicit. If the argument becomes purely about support or amplitude size, the route has failed.
5. Run the smallest honest exact-support audit before any extended forcing or counterexample campaign.
6. One theorem target or one counterexample target per exploration.
7. Every substantial claim in the final report must be tagged `[VERIFIED]`, `[COMPUTED]`, `[CHECKED]`, or `[CONJECTURED]`.
8. Maintain a Direction Status Tracker in `REASONING.md` using `OPEN / PROMISING / CLOSED / EXHAUSTED`.
9. Early stopping is correct if the observable fails the intrinsic-definition gate, the event fails the fidelity gate, the minimal exact-support audit kills the route, or a sharp result survives adversarial review.

## Validation Criteria

This strategy succeeds if it delivers all of the following:

- one explicit preferred phase-coherence observable,
- one explicit preferred delayed-transfer event,
- one fidelity ledger for both objects,
- one sharp forcing statement, counterexample, or structural negative,
- one adversarial screen,
- and one unambiguous terminal verdict in the mission's named categories.

This strategy is exhausted if:

- no intrinsic phase object survives definition gating,
- the delayed-transfer event cannot be defined sharply without role labels,
- the preferred object survives definition but fails the minimal fidelity / coupling audit,
- or a positive or negative result survives adversarial review.
