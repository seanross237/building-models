## Finding

A trigger-focused exact-Navier-Stokes firewall does not produce a genuinely smaller faithful object than the earlier near-closed Tao-circuit definition. The distinguished trigger variable `a3` only becomes theorem-facing when it is embedded back into the full five-role delayed-transfer chain.

So the faithful exact trigger object is still the same sign-closed helical support and five-role monomial hypergraph as before, now read with `a3` singled out as the trigger.

## Why The Trigger Is Not Standalone

The trigger question needs the surrounding roles to make its key words meaningful:

- pre-threshold **smallness** needs a carrier scale, supplied by `a1`,
- **delay** needs an amplifier threshold, supplied by `a2 a3 -> a3`,
- post-threshold **decisiveness** needs downstream visibility, supplied by `a1 a3 -> a4` and `a4^2 -> a5`,
- precursor, leakage, and back-reaction claims need the full desired / internal-leakage / external-leakage ledger, not an isolated observable on `a3`.

In that sense, "trigger-only exact object" is a category error. The trigger is a causal role inside a five-role mechanism, not a self-contained theorem object.
