# Brief: Cross-Functional Navier-Stokes Millennium Prize Plan

## Goal

Produce a serious research plan for trying to solve the 3D incompressible
Navier-Stokes regularity problem at Millennium-Prize level, using the
authoritative repository status picture and a deliberately cross-functional
critique pass.

## Primary Source

- `current-status-of-navier-stokes.md`

## Secondary Input

- the previous refinement run may be used as a baseline draft, but the new run
  should not merely ratify it

## Constraints

- Treat `current-status-of-navier-stokes.md` as authoritative for current local
  status.
- Do not reopen routes the status file marks structurally closed unless a critic
  identifies a concrete reason that the closure is not binding.
- Optimize for theorem-bearing progress toward an accepted Navier-Stokes target,
  not for internal note-system elegance.
- Require all promoted objects to survive canonicity, reconstruction,
  relabeling/refinement stability, and exact comparison on one defined currency.

## Why This Run Exists

The first plan-debate-refinement pass improved the plan by making it
dependency-first and moving canonical extraction to the front. This second pass
should pressure-test that improved plan with critics who are more mathematically
cross-functional and domain-heavy.

The critics should not behave like generic planning reviewers. They should act
like domain experts with different mathematical instincts, for example:

- compactness-rigidity / critical spaces
- harmonic analysis / pressure / nonlinear estimates
- dynamical systems / triads / mechanism realism
- rigorous definitions / formal proof architecture
- prize-level strategy / theorem leverage on the PDE itself

## Required Output

The final plan should:

- keep the theorem-facing discipline of the first refined plan if it survives
  critique
- sharpen the main route and fallback route using domain objections
- say exactly where the plan touches accepted Navier-Stokes theorem targets
- preserve hard stop conditions
- make it obvious which domain critiques changed which parts of the plan
