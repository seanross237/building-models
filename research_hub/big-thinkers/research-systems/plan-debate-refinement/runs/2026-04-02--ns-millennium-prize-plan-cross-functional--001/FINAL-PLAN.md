# Final Plan: Cross-Functional Navier-Stokes Millennium Attempt

## Core Judgment

No prize-facing main route is currently justified until a short gating sprint is completed.

The authoritative status file and all five critiques agree on the same structural point: the draft over-promoted the least stabilized route. The critical-element / canonical one-bridge extraction program is still blocked at the object-definition stage, and the obstruction is not cosmetic canonicity work. It is the absence of an intrinsic, pressure-stable, compactness-compatible observable or extraction principle.

So the active main route is not a premature prize proof campaign. It is a bounded decision/no-go campaign whose job is to determine, quickly and theorem-first, whether the Tao-adjacent line can be lifted into accepted PDE language at all, or whether it should be formally demoted by a robust no-go statement.

## Accepted Theorem Targets

All active work must cash out into one of the following.

1. An exact packet-family theorem on the frozen near-circuit ledger:
   `Delta_tmpl(F_SL(rho)) = 1/4`,
   `Delta_spec(F_SL(rho)) = 3 rho + rho^2`,
   with sharpness at `rho = 1/16`.

2. A frozen-ledger leakage firewall lemma:
   any hostile tiny-trigger packet in the relevant frozen class must violate the repaired leakage sheet in the recorded coordinates `L_tot`, `L_mirror`, `L_companion`, and `L_feedback`, or else one explicit counterexample kills the route.

3. A compactness/pressure compatibility theorem or a no-go theorem:
   either produce an intrinsic observable or quotient object that survives critical compactness, pressure reconstruction, harmless refinement, relabeling, and same-currency comparison, or prove that the current one-bridge / packet family cannot supply such an object.

4. An intrinsic shell-crossing theorem:
   any quadrature-locked constructive transfer sector on exact shell-crossing triads must incur recursive companion/spectator growth too fast to support a Tao-like delayed window.

5. Only if written immediately as one exact statement in standard PDE language:
   a recognized conditional regularity theorem on the vorticity-direction side, CF/DHY-adjacent rather than vague alignment language, with an explicit reason it does not collapse back to the blocked `e_2`-alignment plus `s_2 > 0` picture.

## Active Main Route

### Active main route: bounded Tao-adjacent decision/no-go campaign

This is the correct current mainline because it is the only route ranking supported simultaneously by:

- the status file’s theorem-facing dependency graph,
- compactness-rigidity objections,
- pressure/nonlinearity objections,
- exact-triad mechanism objections,
- and prize-strategy objections.

The campaign has four tightly scoped work packages.

### Work Package A: close the frozen template-defect theorem exactly

Goal:
promote the repaired `Template-Defect Near-Closure` screen from a witness-local carried object into a genuine exact packet-family theorem on the frozen ledger.

Required output:

- one exact theorem statement on the `F_SL(rho)` family,
- one exact proof in one comparison currency,
- one sharpness statement at `rho = 1/16`.

Reason this stays active:
it is one of the only theorem-shaped next moves the status file actually supports.

### Work Package B: run the leakage firewall as a kill test, not as a slogan

Goal:
decide whether the repaired `Windowed Spectator-Leakage Budget` produces a real obstruction theorem on the same frozen ledger.

Required output:

- either a transfer lemma showing the hostile tiny-trigger shape forces the recorded leakage lower bounds in the same exact currency,
- or one explicit hostile same-ledger packet satisfying repaired leakage, which kills the route.

Interpretation:
positive output keeps the Tao-adjacent line alive only as a sharper theorem-bearing screen;
negative output is not failure of planning but a useful no-go result.

### Work Package C: compactness / pressure / quotient feasibility gate

Goal:
answer the actual upstream question the draft skipped:
is there any PDE-defined object carrying the intended mechanism information that survives critical limiting procedures and full pressure restoration?

Required output:

1. Freeze the exact PDE setting for any future bridge:
   ambient domain, solution class, blowup notion, symmetry group, and compactness statement.

2. Replace representative-first language by quotient-first language:
   define the candidate codomain as an intrinsic observable or equivalence class, not as a hand-frozen role-labeled ledger.

3. State one actual theorem-shaped bridge:
   `NS solution or concentration window -> quotient observable P`,
   with `P` stable under compactness, relabeling/refinement, and pressure reconstruction.

4. If that cannot be done, prove the negative version instead:
   no extraction into the current packet / one-bridge family can satisfy those stability requirements.

This package decides whether the critical-element route exists as mathematics or only as mechanism prose.

### Work Package D: keep one intrinsic non-packet moonshot alive, but on a strict leash

Goal:
test the strongest surviving intrinsic object class, namely exact shell-crossing triads with recursive spill, not another annotated packet model.

Required output:

- one theorem statement on shell-crossing triads that does not use desired-channel annotation,
- one recursive-spill or spectator-growth obstruction against sustained delayed-window locking,
- or one clean negative showing that the first genuinely intrinsic phase object remains too weak even after moving beyond tiny supports.

This stays active because it is the best remaining candidate for a mechanism-faithful exact object that is less representation-dependent than packet ledgers.

## Bounded Decision / No-Go Campaign

Duration:
`10-14` days of main-budget work.

Decision rule at the end of the sprint:

1. `Green`
   Work Packages A and B both survive exact proof scrutiny, and Work Package C produces an intrinsic or quotient object with pressure/compactness compatibility.
   Only then activate the prize-facing conditional route below.

2. `Amber`
   Work Packages A and/or B yield real frozen-ledger theorems, but Work Package C still fails to produce a PDE-stable bridge.
   Then record the packet theorems as local exact results, but do not call the branch a prize route. Keep only the shell-crossing intrinsic route alive.

3. `Red`
   Work Package A fails exact closure, or Work Package B admits a same-ledger hostile counterexample, or Work Package C yields a no-go theorem.
   Then demote the Tao-adjacent one-bridge program from prize-facing status and preserve only the theorem-shaped negative result.

This campaign is successful even in the red case if it produces a robust no-go theorem narrowing the global search space.

## Conditional Route

### Conditional route: critical-element rigidity only after the gate succeeds

This route is not active now. It activates only if the decision campaign delivers a PDE-stable intrinsic observable.

Preconditions:

1. The PDE setting is frozen exactly.
2. The output object is a quotient or intrinsic observable, not a witness-local representative.
3. The observable survives the chosen critical compactness framework.
4. Pressure reconstruction is restored in theorem form.
5. The contradiction property can be written directly on actual NS solutions, not in packet shorthand.

Only after that may the route take the form:

`minimal or almost-periodic critical element -> intrinsic property P on concentration windows -> rigidity theorem excluding P -> contradiction`

Notes:

- Do not default to `L^3` merely because it is standard.
- Do not run a critical-element campaign until the chosen host space is justified relative to the extraction codomain.
- Template-defect and leakage may appear only as downstream transfer targets or as evidence for impossibility, not as substitutes for the compactness-rigidity theorem itself.

## Parked External PDE Route

The non-Tao fallback is not active until it is written as one exact theorem statement.

The only presently credible class is the vorticity-direction route, CF/DHY-adjacent, based on curvature or `D_xi S` control in high-vorticity regions. Generic “alignment plus coherence” language is not allowed. Any reactivated theorem must say exactly:

- which recognized criterion it strengthens or verifies,
- which scale-critical quantity is controlled,
- why the hypothesis changes stretching magnitude rather than only direction,
- and why the known `s_2 > 0` obstruction does not simply reappear.

Until that statement exists, this route stays parked.

## Hard Stops

1. If the `F_SL(rho)` theorem still depends on hidden bookkeeping, witness-local conventions, or mixed currencies, stop treating it as theorem progress.

2. If one hostile same-ledger packet satisfies the repaired leakage sheet, the leakage firewall route is dead.

3. If no intrinsic or quotient observable carrying the intended mechanism can be written in accepted PDE language and shown pressure-stable under critical compactness, do not spend more prize budget on canonical one-bridge extraction.

4. If the shell-crossing route needs desired-channel annotation, arbitrary finite truncation, or witness-local closure conventions, stop calling it intrinsic.

5. If a proposed fallback theorem is only “alignment/coherence” rhetoric and does not explicitly escape the `s_2 > 0` obstruction, keep it parked.

6. Do not reopen the routes already marked structurally closed unless a genuinely new theorem-level ingredient is identified:
   enstrophy/BKM bypass,
   De Giorgi/Vasseur pressure improvement,
   epsilon-regularity bootstrapping,
   `H^1` pressure repair,
   harmonic far-field pressure repair,
   unconditional `e_2`-alignment regularity.

## What Counts as Real Progress

Real progress means one of:

- a theorem in accepted PDE language that strengthens a recognized regularity criterion,
- a rigidity theorem on an intrinsic PDE-defined class,
- an exact theorem that decisively bridges or kills the Tao-adjacent line,
- or a robust no-go theorem proving why an attractive extraction/object class cannot underlie an exact-NS singularity argument.

The following do not count:

- cleaner packet prose,
- prettier canonicalization language,
- representative-specific authority sheets without bridge value,
- mechanism stories that do not survive pressure, compactness, and quotient stability.
