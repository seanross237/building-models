# Draft Plan: Cross-Functional Navier-Stokes Millennium Prize Attempt

## Core Judgment

The current status file still supports the same top-level conclusion: the
classical estimate routes look largely exhausted, and any serious attempt now
has to be explicit about what accepted Navier-Stokes theorem target it is trying
to move.

The draft therefore starts from the strongest surviving discipline from the
first refinement:

- one main route,
- one fallback route,
- explicit gates,
- and no open-ended theorem-chasing on private artifacts.

## Accepted Targets

### Main target

Exclude a symmetry-normalized minimal blowup element in an accepted critical
host space, defaulting to `L^3`, by proving an exact contradiction chain:

`minimal blowup element => canonical extraction of one exact one-bridge object => canonical obstruction theorem on that object => contradiction`

### Fallback target

Prove a real conditional regularity theorem in standard PDE language via a
recognized direction/coherence criterion on vorticity and strain, not via packet
or ledger artifacts.

## Main Route

### Phase 0: Freeze the object layer

Before any theorem campaign, freeze:

- canonical one-bridge object
- comparison currency
- harmless refinement
- relabeling invariance
- reconstruction rule from source data
- meaning of canonical extraction

Required outputs:

- one definitions sheet in standard mathematical prose
- one provenance map back to the status-file frontier
- one reconstruction note showing the bookkeeping is not hand-maintained

Stop rule:

- if the object layer is not canonical or reconstructible, stop the route

### Phase 1: State the bridge to the PDE theorem target

Write the exact implication chain in accepted PDE language:

1. chosen critical host space
2. canonical extraction lemma
3. obstruction theorem on the extracted class
4. contradiction excluding the minimal blowup element

Stop rule:

- if that bridge cannot be written without circularity or private shorthand, the
  main route is blocked

### Phase 2: Prove the canonical extraction lemma

This is the main gate.

Target:

- canonical extraction from a minimal-element sequence, stable under compactness
  limits, harmless refinement, and relabeling

Stop rule:

- if extraction still depends on hand-frozen representatives, finite-support
  truncation choices, or witness-specific bookkeeping, stop the route

### Phase 3: Use template-defect and leakage only as subordinate obstruction tests

Only after extraction succeeds:

- test whether the repaired template-defect theorem transfers to the extracted
  canonical class
- test whether the repaired leakage firewall transfers in the same currency
- combine them only if they feed a contradiction theorem on the PDE side

Stop rule:

- if they remain frozen-ledger identities only, they do not justify continuing

### Phase 4: Assemble the contradiction

If the first three phases work, prove that no symmetry-normalized minimal blowup
element can satisfy the extracted obstruction conditions.

Stop rule:

- if the route yields only cleaner local artifacts without a contradiction
  theorem, close it as frontier mapping

## Fallback Route

Run exactly one fallback theorem program:

- combined strain alignment plus vorticity-direction coherence, aimed at a
  Constantin-Fefferman-type regularity consequence in standard PDE language

Required outputs:

- one precise theorem statement
- one exact accepted regularity criterion it is verifying or strengthening
- one statement of how it avoids merely repeating the blocked `e_2`-alignment
  route

Stop rule:

- if it collapses back to the known `s_2 > 0` obstruction or only reproduces
  existing conditionality, stop

## Parked Work

These do not receive primary time unless the active routes create a direct need:

- shell-crossing phase/coherence route
- helical-sign packet functionals
- broad sidecar portfolio
- detached packet-family theorem work

## Time Gates

### Short horizon

Within days to two weeks:

- freeze the object layer
- write the PDE bridge exactly
- decide whether extraction is formally well-posed

If that fails, switch to the fallback route.

### Medium horizon

Within one to two months:

- either make real progress on extraction
- or declare the main route blocked

### Theorem horizon

Only after extraction succeeds:

- spend serious effort on template-defect and leakage obstruction theorems

## What Counts As Real Progress

1. canonical extraction on an accepted critical-element framework
2. an obstruction theorem on that canonical class that feeds a minimal-element contradiction
3. a contradiction theorem excluding the critical element
4. failing that, a new conditional regularity theorem in accepted PDE language

The following do not count as main progress:

- private object cleanup
- witness-specific frozen formulas
- prettier packet language
- mechanism sketches without a PDE implication
