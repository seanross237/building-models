# Summary Of Revisions

## Main revisions

The draft was narrowed from a five-route research portfolio to one main route
and one fallback route.

The new main route is no longer "prove frozen-ledger theorems first." It is:

- define the objects cleanly,
- state the exact bridge to a standard critical-element contradiction,
- try to prove the canonical extraction lemma early,
- and use template-defect and leakage only as subordinate obstruction tests on
  the extracted canonical class.

The new fallback route is one direct PDE theorem program in accepted language:

- combined strain alignment plus vorticity-direction regularity aimed at a
  Constantin-Fefferman-type regularity consequence.

The sidecar portfolio, shell-crossing route, and helical-sign route were all
parked rather than kept as active branches.

Hard stop conditions were added at every gate:

- failure of canonical definitions,
- failure to state a non-circular bridge to a recognized Navier-Stokes theorem
  target,
- failure of the canonical extraction lemma,
- or failure to convert packet-screen results into a contradiction theorem on
  the PDE side.

## Which critiques caused which revisions

### Feasibility

This critique drove the front-loading of:

- the definition package,
- provenance and reconstruction requirements,
- the dependency-first structure,
- and the demand for a realistic short-horizon gate before any long theorem
  campaign.

It also caused template-defect and leakage work to be demoted from primary
routes to test cases for a formalism that must first prove itself canonical.

### Risks

This critique caused the main re-centering around the canonical extraction
problem. The revised plan treats extraction as the gate for whether the
ledger-based route has any chance of mattering, rather than as a late optional
bridge.

It also caused the explicit rule that frozen-ledger identities do not count as
main-route progress unless they transfer to the extracted canonical class and
feed a contradiction theorem.

### Dependencies

This critique caused the explicit Phase 0 and Phase 1 stack:

- freeze glossary and provenance,
- define comparison currency,
- write the exact implication chain to the accepted PDE theorem target,
- then attempt extraction.

It also caused the plan to anchor terminology back to the status-file frontier
instead of freely renaming it.

### Scope and simplicity

This critique caused the largest structural simplification:

- one main route,
- one fallback route,
- parked side routes,
- no standing sidecar search,
- and no separate top-level Route 5.

It also caused ledger-specific formulas to be reframed as subordinate checks
rather than as the plan’s headline milestones.

### Goal fit

This critique caused the plan to state accepted theorem targets explicitly:

- primary target: exclude a minimal blowup element in `L^3` or another accepted
  critical host space by a canonical extraction plus contradiction route;
- fallback target: prove a conditional regularity theorem in standard PDE
  language through a recognized direction-regularity criterion.

It also caused a hard stop if the work produces only internal object cleanup
without a theorem that changes the actual Navier-Stokes status picture.

## Residual risks

The largest residual risk remains the same one the critiques identified:

- the canonical extraction lemma may simply not exist in a usable form.

If that fails, the Tao-adjacent ledger program becomes frontier mapping rather
than a prize path, even if some local exact identities survive.

A second residual risk is that the fallback route may still prove only a
conditional theorem rather than a route to unconditional regularity. That is why
it is a fallback and not the main route.

A third residual risk is dependence on the status file’s frontier ranking. The
revised plan reduces that risk by tying the active work to accepted theorem
targets, but it does not remove it completely.
