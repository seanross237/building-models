# Draft Plan: Navier-Stokes Millennium Prize Attempt

## Core Judgment

The status file does not support a direct "push harder on classical estimates"
program. The only defensible plan is a theorem-first elimination program that
starts from the remaining live object-definition frontier and earns the right to
make stronger regularity claims only after canonical objects and exact transfer
laws are proved.

So this plan is not:

- "find one better bound"
- "retry De Giorgi with a different wrapper"
- "hope Tao-like language becomes a proof by accumulation"

It is:

- a ranked campaign for converting the remaining live objects into exact
  theorems,
- with hard kill conditions,
- and with a bounded sidecar search outside the Tao-adjacent family so the whole
  effort does not collapse into local optimization on a dead object class.

## Route Ranking

1. `Route 1`: exact template-defect family theorem on the frozen one-bridge ledger
2. `Route 2`: exact leakage firewall lemma on the same ledger
3. `Route 3`: intrinsic recursive spill-versus-locking theorem on shell-crossing exact triads
4. `Route 4`: non-Tao sidecar probes, capped and selective
5. `Route 5`: conditional critical-element bridge, only after a canonical extraction lemma exists

## Phase 0: Proof Standards And Route Discipline

Before any new theorem push, freeze the admissibility standard for all promoted
objects:

- exact-NS quantity, not an averaged or modeled surrogate
- canonical under harmless support refinement and representative relabeling
- same-currency comparison across friendly and hostile families
- finite-closure honest bookkeeping
- clear downstream theorem role: firewall, incompatibility, rigidity, or
  extraction

Immediate kill rule:

- if a candidate object fails canonicity or same-currency comparison, stop
  upgrading it and demote it to mechanism evidence only

Deliverable:

- one short "promotion checklist" used on every candidate theorem object before
  serious work continues

## Phase 1: Convert Route 1 Into A Sharp Packet-Family Theorem

Objective:

Promote the repaired `Template-Defect Near-Closure` line on the frozen
one-bridge ledger from an authority-sheet screen into a genuine exact theorem.

Concrete targets:

1. Prove on the frozen ledger that
   `Delta_tmpl(F_SL(rho)) = 1/4`.
2. Prove on the frozen ledger that
   `Delta_spec(F_SL(rho)) = 3 rho + rho^2`.
3. Prove sharpness at the carried witness `rho = 1/16`.
4. Verify that every step is exact role-projected coefficient/sign arithmetic on
   the frozen sheet, with no hidden packet bookkeeping.

Why first:

- the status file ranks this as one of the best concrete near-term theorem moves
- it has the clearest explicit formulas
- it is the fastest test of whether the frozen ledger really supports theorem
  promotion or only curated examples

Success condition:

- an exact family theorem on the frozen ledger with a sharp witness and a
  reusable proof template

Kill condition:

- if the defect identities cannot be closed without hidden witness choice,
  hidden desired-channel annotation, or ad hoc ledger repair, Route 1 drops from
  active theorem route to historical artifact

## Phase 2: Convert Route 2 Into A Leakage Firewall Lemma

Objective:

Prove a local frozen-ledger theorem that hostile tiny-trigger shape forces
repaired leakage-sheet violation in the recorded coordinates.

Concrete targets:

1. Freeze the precise hostile family statement in one exact interaction
   currency.
2. Prove the transfer lemma that carries hostile desired-channel shape into the
   leakage lower bounds.
3. Show forced violation in at least
   `L_tot`, `L_mirror`, `L_companion`, and `L_feedback`.
4. Check whether `F_SS(1/12)` and `F_SL(1/16)` can coexist as the full friendly
   reference pair without reopening semantics or comparison currency.

Why second:

- this is the other best concrete theorem move from the status file
- it pressure-tests the same frozen ledger from the hostile side
- success here would turn the ledger from a descriptive object into an
  obstruction object

Success condition:

- a theorem of the form:
  hostile near-degenerate trigger geometry implies violation of the repaired
  leakage sheet on the frozen ledger

Kill condition:

- a single exact packet on the same frozen ledger with hostile tiny-trigger
  behavior and repaired leakage survives

## Phase 3: Try To Earn One Intrinsic Object Class

Objective:

Pursue the cleanest remaining moonshot: an intrinsic phase/coherence theorem on
exact shell-crossing triads that does not depend on packet annotation.

This route must now use the stricter post-April-1 formulation:

- not first-branch phase frustration
- but recursive spill versus delayed locking

Concrete targets:

1. Freeze the phase/coherence observable in a form that is manifestly intrinsic
   on exact shell-crossing helical triads.
2. Write the recursive emission/closure law in the same language as the
   observable.
3. Prove or decisively refute a statement of the form:
   any quadrature-locked sector supporting sustained positive forward transfer
   triggers companion/spectator growth too quickly to support a Tao-like delayed
   window.
4. If the first theorem fails, identify the first exact mechanism that makes it
   fail:
   recursive reinforcement, phase-band persistence, or support-growth geometry.

Why third:

- it is the best remaining intrinsic route
- unlike the packet routes, it could create a genuinely new theorem object class
- but it is weaker than Routes 1 and 2 because the smallest honest supports
  already failed to obstruct

Success condition:

- an exact recursive-spill theorem or an equally sharp impossibility diagnosis

Kill condition:

- if later recursive layers continue to preserve constructive coherence without
  producing a theorem-facing incompatibility or burden explosion, Route 3 gets
  demoted behind non-Tao directions

## Phase 4: Run A Strictly Capped Non-Tao Sidecar Search

Objective:

Preserve exposure to non-Tao directions without letting the main program drift
 into generic novelty hunting.

Allowed sidecar directions:

1. alignment + vorticity-direction regularity
2. direct pressure-Hessian analysis of enstrophy balance
3. vorticity-direction Lipschitz regularity in high-vorticity regions

Rules:

- run these only as sharply scoped theorem probes, not broad literature tours
- cap each at one precise lemma target plus one explicit obstruction test
- stop if they collapse back into already-closed enstrophy or epsilon-regularity
  architectures

Why this phase exists:

- the status file identified three remaining live non-Tao directions
- the Tao-adjacent frontier may still be blocked at object-definition level
- a prize-facing program should not be monocultural when the main branch is this
  narrow

Success condition:

- one sidecar route produces a sharper theorem target than Route 3

Kill condition:

- the sidecar reduces to old closed mechanisms or produces only heuristic
  “interesting structure” with no theorem object

## Phase 5: Only Then Reopen The Critical-Element Bridge

Objective:

Attempt the compactness-rigidity bridge only if prior phases produce a canonical
ledger or extraction rule strong enough to survive compactness limits and
harmless refinement.

Required prerequisite:

- a canonical critical-element ledger extraction lemma

Only after that:

1. assume a symmetry-normalized minimal blowup element in a defensible critical
   host space, most likely `L^3`
2. canonically extract one exact Tao-style one-bridge ledger on concentration
   windows
3. try to prove `no-double-admissibility` for the repaired template-defect and
   repaired leakage gates

Reason for deferral:

- the status file explicitly says this branch is blocked earlier than before,
  at object-definition rather than estimate level

Success condition:

- a contradiction blueprint that survives the extraction step honestly

Kill condition:

- any dependence on non-canonical window choice, harmless refinement, or
  representative relabeling in the extracted ledger

## Cross-Phase Deliverables

Each active route must produce one of:

- a theorem statement with exact hypotheses and conclusion
- a clean counterexample or killer example
- a no-go memo that closes the route for a specific structural reason

Nothing counts as progress if it is only:

- a prettier formulation of an already-equivalent estimate
- a packet semantics rewrite without canonicity
- a broad “this still feels promising” memo

## Resource Allocation

Recommended near-term split:

- `45%` Route 1
- `30%` Route 2
- `15%` Route 3
- `10%` capped sidecars

Do not allocate primary time to Route 5 unless one of the first three phases
changes the extraction picture.

## What Would Actually Count As Prize-Relevant Progress

The plan should be judged by whether it earns one of these:

1. a canonical exact theorem object that survives all current objections
2. a firewall or incompatibility theorem that rules out a Tao-like delayed
   mechanism in exact NS on a non-artificial class
3. a canonical extraction rule that allows compactness-rigidity to use the new
   object honestly
4. a non-Tao theorem object that bypasses the closed estimate architectures

Absent one of those, the work is still frontier mapping, not a prize path.
