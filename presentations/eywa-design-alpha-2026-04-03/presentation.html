The Building Block

What is a node?

A directory on disk. Four files. That's it.

input.json

The task + context from parent

variables.json

Every knob for this node's behavior

output.json

What the node produced

metrics.json

Score, tokens, cost, time

+ optionally

children/

Sub-nodes (same structure, all the way down)

You can pick up any node from the middle of a tree, look at these four files, and understand everything about what happened.
No hidden state. No ambient context. The node is the atom.

The Heartbeat

How does a node live and die?

Five states. Linear path. No surprises.

Pending
→
Active
→
Working
→
Done
↓
Failed

PENDING

Directory created.
input.json + variables.json written.

ACTIVE

Orchestrator picks it up.
Assembles the prompt from variables.

WORKING

Model called. May spawn children
(which start back at PENDING).

DONE

output.json + metrics.json written.
Result bubbles up to parent.

The orchestrator is a simple loop: find deepest PENDING → activate → collect output → bubble up.

The Plan

What do we build first?

Smallest useful thing at each step. Each step unlocks the next.

1

The Node + Orchestrator

Node contract (4 files). Simple depth-first orchestrator loop. One run produces a tree of node directories.

↓
2

The Variable Schema + Prompt Assembly

Define the initial variable set. Build the prompt assembler that reads variables.json and constructs the model call.

↓
3

The Proving Ground

20-30 benchmark tasks with known answers. Auto-scoring. Run the suite, get a leaderboard.

↓
4

Bonsai L1: Best Average

Run benchmarks with variable sweeps. Pick the winner. Now every new run starts with proven-good defaults.

After step 4, the system is already useful and already learning.
L2 and L3 are unlocked by data, not by new architecture.
