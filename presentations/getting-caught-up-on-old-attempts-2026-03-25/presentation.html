The One Stumper: Blackboard Difference Game

A professor writes 1, 2, 3, ..., 25 on a board. Students repeatedly erase two numbers a, b and write |a−b|. After 24 operations, one number remains. What are ALL possible final values?
CORRECT
All odd numbers 1–25 (13 values)
CLAUDE SAID
"All odd numbers from 1 to 323"

Why It Failed

Claude correctly identified the parity invariant (result must be odd) and applied the signed-sum framework. But it missed a simple operational constraint: |a−b| ≤ max(a,b), meaning values on the board can never exceed 25. The math framework was right, but the process constraints were ignored.

Key Insight

Claude doesn't fail on hard math because of missing knowledge. It fails when there's a gap between what the abstract framework says and what the actual process allows. Constraint-checking is the blind spot.

What the Updater Learned About Sean's Taste

1
Sincerity over cleverness. Sean picks warm, direct declarations over poetic wordplay. "You're home to me" beats a clever metaphor every time.
2
Don't cram details. Listing every pet name + date + location in one couplet loses to a cleaner emotional line. There's a sweet spot.
3
Warm poetry is welcome. "Like the sunrise through the pines" isn't cleverness — it's tenderness. Sean loves warm imagery.
4
Gentle warmth over gut-punch. "Look at this beautiful life" beats "our baby will never know that pain." Celebrate, don't devastate.

Reflections · Mar 25

Building Iterative Systems

The goal wasn't the best couplet evaluator — it was learning how to build systems that improve themselves.

What Worked
The loop architecture is reusable. Guesser → Score → Updater → New Prompt is a clean, composable pattern. Stateless agents, single state file as source of truth, any agent can resume. This generalizes beyond couplets.
Blind evaluation with shuffling was real rigor. Stripping model names and randomizing positions eliminated position bias. The seeded Fisher-Yates shuffle made it deterministic and reproducible.
The dashboard made it an experiment, not a log dump. Live visualization turned raw JSON into something you can reason about and show to others.
The Regression Problem
Gen 3 cratered from 63% back to 43%. The updater saw 11 misses and said "don't do X, Y, Z" — the next guesser became paralyzed. Classic overcorrection. The updater needs a mutation rate: limit to 2-3 adjustments per generation, not full prompt rewrites.
The Self-Correction Gap
Gen 5's jump came from human intervention, not the loop. Someone recognized Gen 2 was the peak and went back to that base instead of continuing the degraded chain. A real self-improving system needs: regression detection, auto-revert to best-so-far, updater works from the best prompt not just the latest.

V2 Requirements

🧬
Mutation Rate
Max 2-3 changes per gen
🛡
Regression Guard
Auto-revert on score drop
🎯
Multi-Run Scoring
3x per prompt, use median
⚖
Analyze Hits Too
Preserve what's working
The updater only sees misses. It doesn't know WHY the 19 hits worked. So it inadvertently breaks the patterns that were already winning. Future updaters must explicitly say "preserve these."

Across All Experiments

Patterns

What ties these experiments together

"Get a source of validation. Set up a run/execute/validate feedback loop against that validation."
— thoughts-on-building-agents.md
1
Every experiment has ground truth
Actual purchases (predict_song_purchase_using_prompt_evolution), math answers (stumper), benchmark keys (BBEH), Sean's picks (rhyme-eval). No ground truth = no experiment.
2
Evolutionary prompt optimization works
predict_song_purchase_using_prompt_evolution and rhyme-eval both use the same loop: measure errors → analyze failure modes → generate variants → re-measure. Rhyme-eval went from 43% to ~63% in one generation.
3
Blind spots, not knowledge gaps
Stumper loop proved Claude doesn't lack knowledge — it misses constraints. BBEH showed the same: wrong answers come from rule-misapplication, not missing info.
4
Certainty signals are usable
BBEH showed 52% confidence = 0% accuracy, 95%+ confidence = 91% accuracy. When the model says it's unsure, believe it.
